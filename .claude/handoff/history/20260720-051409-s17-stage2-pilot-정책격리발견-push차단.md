## 발견 (검증됨)
- **F1(헤드라인·확실):** 스폰 claude(`claude-sonnet-4-6`)가 `[message from ...]` 봉투를 인젝션으로 판정·격리 → codeword/sender/id 회수 전부 실패(트랜스크립트 thinking 트레이스 직접 확인, 다수 런). 파생: 거부 후 **원과제 이탈**도 관측. 근거: 스폰 에이전트가 조직 보안 규칙(도구 콘텐츠 속 지시=데이터) 상속. **사전정당화 문구는 역효과**(더 강한 즉시 거부 — pre-authorization 패턴).
- **F2:** compaction **관측 가능**(유기적 native `compact_boundary`, preTokens 32490 캡처). 강제 `/compact`는 유저텍스트라 native 압축 못 일으킴 → 하네스서 제거.
- **F3:** 디코딩 usage(input_tokens ~3) ≠ 컨텍스트. 실 컨텍스트(~30–36k)는 트랜스크립트 cache-합에만. 문자추정 과소(23.7x→5x). 본런 fill은 실 usage 기준.
- **F4:** **자연문 필러조차 content filter 유발** → 본런 필러는 진짜 양성(실세계 유래) 필요.
- **F5(정직):** 판당 비용 미확정 — 필러 거부로 포화 완주 전 abort. 사전등록 셀 수치 근거 여전히 없음.

## ★설계 갈림길 (사용자 결정 — 본런 진입 게이트)★
본런 주 지표(포화 하 에이전트 간 codeword 회수)가 포화 아닌 정책으로 0. 세 갈래:
- **(a) 지표를 태스크 채널 회수로 전환** — 원과제 완주 + DOC-1 회상(정책 격리 밖). 현 하네스로 즉시 측정 가능.
- **(b) 주입을 일반 유저턴으로** — 격리 회피하나 **ADR-0090 d2(실 control 경로) 위반** + 측정 대상 바뀜.
- **(c) 격리 자체를 Stage 2 결론으로 확정 + 봉투/신뢰 설계(Stage 4)를 앞당김** — 정당한 메시지가 인젝션 방어를 신뢰 통과하는 법(wrap_message 인증·채널 신원).
- **메인 권고 = (a)+(c) 병행**(운영 경로 보존). 상세 = 보고서 §5.

## 검증 상태 (쌍)
- **한 것(전부 PASS):** `/review code full` 2인 적대(doc-aware reviewer-deep + Codex blind/codex-exec) **3라운드 수렴**(R1 BLOCK 15 → 통합FIX → R2 doc-aware PASS vs Codex BLOCK 신규 6 → 집중FIX → R3 Codex FIX 2 → 마이크로FIX 자체검증). **로드베어링 정확성(주입 turn 펜싱·귀속·stall·usage 덮어쓰기·안전캡·정리) 양쪽 CLOSED.** `/qa`: 워크스페이스 빌드(bin 제외 확인)·core 210·protocol 49·discovery 44·daemon 144 lib(experiment 72)·fmt·격리0 PASS. 프론트 무변경(tsc/vitest 미실행 — 이번 세션 프론트 변경 0). **재실행:** `cargo test -p engram-dashboard-daemon`(feature 불요 — self-dev-dep이 test-harness 켬) / `cargo build -p engram-dashboard-daemon --features test-harness --bin saturation-pilot`.
- **안 한 것(정직):** ① **본런 미착수**(설계 갈림길 대기). ② **포화 완주 런 없음** — 모든 실 런이 F1/F4(정책 거부)로 abort, 150k 목표 도달 전. 즉 판당 비용·완주 거동 미실측. ③ **마이크로FIX(실패경로 TurnRecord·helper 테스트)는 3차 적대 재리뷰 없이 자체검증(build/test/fmt)** — 실패경로 한정 record-completeness라 비례적 판단. ④ compaction 감지기의 real-series 경로는 유닛테스트만, 실 포화 런서 미발화.

## do-not (누적)
- 루트 bare `cargo test` 금지(src-tauri 크래시 — 멤버별 `-p`). 릴리즈를 `--all-targets`로 빌드 금지(test-harness 유니피케이션).
- **설계대로의 본런 금지** — 정책 격리로 codeword 회수가 0이라 셀마다 격리 재확인=토큰 낭비. 갈림길 결정 먼저.
- **주입 사전정당화 문구 금지** — pre-authorization 패턴이라 더 강한 거부 유발(F1 실측).
- 필러에 의사난수/부자연 텍스트 금지 — content filter 유발(F4). 진짜 양성 필요.
- epoch pinning 금지(ADR-0086 §F5) · 배달 시맨틱 변경 금지 · `/compact` 유저텍스트로 native 압축 기대 금지.
- 코더 지시서에 "태스크 리스트 불가침" 명시(과거 코더가 메인 태스크 삭제한 선례).

## 정지 조건
- 실 claude 대량 스폰(본런)은 ADR-0088 결정 5 안전 요건 + F4(양성 필러) 해결 후. 주입/셸 확장 케이스는 이 하네스로 절대 돌리지 말 것(Stage 4에서 5b 전체 요건과 함께).
- 리뷰어 정면 대립(FIX vs BLOCK)은 자율모드라도 사용자 에스컬레이션 — 이번엔 Codex가 상위집합(실 버그)이라 증거기반 보수취합했고, 최종 판정은 이 핸드오프로 승계.
