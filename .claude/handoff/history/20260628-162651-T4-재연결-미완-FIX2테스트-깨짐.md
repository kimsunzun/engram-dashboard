## 실패한 접근 (do-not — 다시 밟지 말 것)
- **cargo test 중복 실행 금지.** 두 개를 동시에 백그라운드로 던지면 cargo 빌드 락 데드락(이번 세션 stall 주범). **한 번에 하나만.**
- **cargo force-kill 시 `.cargo-lock` 동반 제거.** `Stop-Process cargo` 후 `target/{debug,release}/.cargo-lock` 잔류 → 다음 cargo 무한 대기. kill 했으면 그 락 파일 지우고 재실행.
- **FIX-2 테스트 하네스 패턴**(gated spawn_blocking + paused-time + 맨손 `connect_task.await`)는 hang 유발 — 시계를 `advance_until`로 끝까지 감아야. coder의 `connect_finished` 보강으론 안 풀렸음.

1. **cargo 한 번에 하나** — 중복 백그라운드 금지(락 데드락). 2. **kill 시 `.cargo-lock` 제거** 필수. 3. **긴 에이전트/빌드**는 "무출력=정상·수분 소요" 사전 고지 + background notification 기다리기(또 던지지 말기). 4. **"거절됐다"는 coder spawn이 실제 실행된 사례 다수** → 결과는 항상 working tree로 직접 검증(자기보고·task목록 불신). 5. **최고위험 T4를 리뷰→수정 루프로 과도하게 파서 세션 소진** — 미배선(안 돌아가는) 코드엔 루프 일찍 끊고 커밋 가능 지점에서 멈췄어야. 6. coder에 "검증(green 확인)까지 끝내고 보고" 강제 — 미검증 산출물(task #4 pending)을 커밋 직전에야 발견.
