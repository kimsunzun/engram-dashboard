## ★ /review prd BLOCK 결과 (다음 세션이 재발견 안 하게 박제)
opus(Tester) + Codex(User) 둘 다 BLOCK. 치명 3 + 갭 다수:
1. **자기모순(핵심):** "응답인식"을 비목표로 미뤘는데 **Handoff·Approval·command_rpc가 전부 응답 의존** → 정의만 있고 동작 못 하는 빈 껍데기. reply 인식 없이 request/response 메시징 설계 불가.
2. **경계 흐림:** command_rpc는 사실 **control-plane**(요청-응답)인데 PRD가 data-plane으로 묶음 → db2 protocol(control)↔내 messaging(data)이 같은 개념을 두 군데서 자름.
3. **프레이밍 결정 누락:** B가 A의 stdin 주입을 *사용자 입력*과 구분 못 함(둘 다 텍스트) → sentinel/프리앰블/별 채널 in-band 프레이밍이 D1보다 먼저 정할 핵심.
- 갭/놓친대안: 전달/읽음 상태 모델 없음 · broadcast/fanout 없음 · human-in-loop 없음 · 순서/causality/dedup 미정 · A→A·죽은 B·중복·B재시작(epoch) 경계 미정 · D1 4번째 대안(capability negotiation / 클래스 대신 보장-플래그 직교) · C2 zero-option(actor 없이 tokio task+mpsc) · C7 conversation-엔티티 keying(cascade가 B 재시작 시 메시지 잃음) · C3 enum retrofit 비용.
