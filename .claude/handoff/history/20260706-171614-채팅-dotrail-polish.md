## 남은 레일 polish 스펙 (다음 작업 = 이대로)
1. **연결선 clean-ends** — 현재 각 rail 행 선이 위 행과 잇느라 `top-[-12px]`로 위로 뻗어 **맨 위 dot(예: Thought) 위에 선 스텁이 허공으로 튀어나옴**(사용자 지적). 확장처럼 "첫 dot 시작 / 끝 dot 끝"으로 하려면 `StructuredTextView`에서 **연속 assistant-run 계산**(user 버블·separator = run 경계) → 각 rail 행에 위치(top/mid/bottom/single) 전달 → `ChatRow`가 위치별로 선 그림(top=dot↓, mid=full, bottom=↑dot, single=최소). ※`top-[-12px]`는 outer `pt-3`(12px)과 커플링 — pt 바꾸면 같이 조정.
2. **간격** — rail 행 `pt-3`, 유저버블 `pt-2.5`. 확장보다 빡빡함. Thought↔Read·유저↔클로드 여백 조정(연결선 오프셋과 커플링).
3. (소소) **툴 점 톤** — 현재 `bg-green-500`. 확장은 더 청록(teal)일 수 있음 — 스샷 대조 후.

## 벤치마크·진행 방식 (재론 금지)
- **벤치마크 = Claude Code VSCode 확장 룩**(소스 비공개 → **스샷 실측 벤치마킹**, 코드 포트 불가 — ADR-0050). CSS 값 베끼기도 금지(라이선스 스탠스). 사용자가 확장 스샷을 **중계**로 제공(내가 확장에 직접 문의 불가).
- **진행 = 1차 근사 후 사용자 스샷으로 반복 조정.** 시각 refine = inline 편집 + cdp 실측 + 게이트, 체크포인트에서 커밋(사용자 "커밋 푸쉬하고 진행" 방침). 비자명 구조 변경(레일 run-position 등) = `/implement`.

## 함정 (do-not)
- **레일 연결선 clean-ends = CSS 값만으론 불가** — run-position 로직 필요(위 스펙). 이미 `top-[-12px]` 커플링 있음.
- **opus thinking = 암호화(평문 빈 문자열) 업스트림 동작** — "Thought" 라벨만 뜨는 게 정상(펼칠 내용 없음). sonnet은 평문 → 스트리밍됐다 접힘. **재디버깅 금지**(ADR-0049 근거).
- **불변식:** `structuredAccumulator`/데이터 모델(ADR-0045)·dispatch FIX1(tool_result 흡수)·FIX2(InertCode 리터럴)·FIX3(`__streaming__` key)·usage 무렌더 — 손대지 말 것. `StructuredTextView`는 순수 렌더(state/effect 추가 금지).
- **ADR-0049 = 백엔드 thinking-token 주입(claude.rs)** ≠ dispatch(=0050). `ThoughtRow.tsx:6`의 0049 인용은 암호화 근거라 정당(유지).
- **BOM 회귀 전례** — 프론트 파일 편집 후 첫 3바이트 EF BB BF 검사.
- **앱 재기동:** `run-dashboard-clean.bat`은 데몬 재빌드(`cargo build -p daemon`)까지 함 → **미커밋 Rust echo(core)를 컴파일**(빌드 깨질 소지). 프론트만 볼 거면 데몬 재빌드 없이 `$env:WEBVIEW2_ADDITIONAL_BROWSER_ARGUMENTS="--remote-debugging-port=9223"; npm run tauri dev`. 포트 9223 고정.
