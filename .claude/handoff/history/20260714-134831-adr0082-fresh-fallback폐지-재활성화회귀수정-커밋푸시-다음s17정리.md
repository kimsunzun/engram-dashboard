## ★ 다음 과제 = 후속 3건 (ADR-0082 §영향에 기록됨) ★
1. **재활성화 가드 cross-connection TOCTOU** — 두 WS 커넥션이 동시에 같은 프로필 활성화 시 `get_session` pre-check·`spawn_agent` contains_key 가드를 둘 다 통과 → 이중 spawn(둘째 insert가 첫 Arc 덮음, 같은 epoch, 첫 child 누수). **HEAD부터 있던 기존 레이스**(이번 변경 무관). 닫으려면 contains_key→insert 를 단일 write-lock 원자화 or per-id spawn mutex. 데몬 명령은 **커넥션별로만** 직렬(각 read_task inline await), 크로스 커넥션 아님(`ws.rs:559`). 결정: 후속 ADR 감.
2. **code-0 조기종료 시 시체 미보존** — claude가 exit code 0으로 조기종료하면 reaper `decide`가 `DeleteProfile`→프로필 삭제(시체 안 남음). 문서화된 "No conversation found"는 code 1이라 보존(`KeepDisableAutoRestore`). code-0 서브케이스 = 잔여 갭. "시체로라도 남겨"를 code-0까지 보장하려면 reaper disposition 조정 필요(사용자 결정 대상).
3. **시체 시각 표시** — 죽은 프로필이 현재 "예약(Reserved)" 노드로 렌더(한 번도 안 켠 것과 시각 구분 안 됨). 별도 "죽음/실패" 마커 띄울지 = **사용자 추후 판단**(실물 보고). 하려면 protocol에 마지막 종료상태 실어 프론트 `mergeTreeNodes` 확장(S17 프론트/protocol 인접 — 겹침 주의).
