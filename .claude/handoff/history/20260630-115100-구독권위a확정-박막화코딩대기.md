## ★이번 세션 결정 (확정)★

### 갈림길 ① (마이너) — truncated 알림 = 제거
- 출력 버퍼(데몬 ring 2MB·4096조각, ring evict) 초과로 앞부분 잘리면 화면을 **clear 후 최근부터(tail) 다시 그림**. 이 **처리(clear+tail)는 유지**.
- 단 **사용자 알림(로그/UI)은 제거**. 근거: 출력 히스토리 권위는 우리(대시보드)가 아니라 **claude 에이전트 쪽**(--session-id/--resume, ADR-0008). 우리 버퍼는 "재연결/늦은 attach 시 최근 화면 복원"용 단기 캐시지 아카이브가 아님. 프론트 xterm scrollback도 기본 1000줄(명시 설정 없음) = 단기 뷰어.
- 박막화로 기존 truncated 로그 경로 사라지는 김에 알림 제거.

### 갈림길 ② (메이저·ADR 관련) — 출력 구독 권위 = (a) 레이아웃(src-tauri 클라) 단독
- **(a) 확정:** 출력 구독 신청은 **src-tauri Rust 클라(레이아웃 권위)가 단독**. 슬롯 배정(layout assign)→router→client.subscribe로 구독 발생. **WebView(프론트)는 구독 신청 안 함** — 받아 그리기만(순수 렌더러).
- 현재 버그 = WebView도 Subscribe를 직접 보냄(forward_daemon_command에서 request_id 없어 Fire로 새 SubState 우회→항상 FromOldest) → layout 경로와 **이중**. (a) = 이 WebView 구독을 떼서 src-tauri 단독화.
- **근거 (이번에 ADR 실제 읽고 확정 — 추정 아님):**
  - ADR-0035(레이아웃 권위=src-tauri, 데몬 View 불가지론, **프론트=순수 렌더러**) 정합. (b)는 "프론트=순수 렌더러" 위반 → (b) 채택 시 ADR-0035 번복(새 ADR) 필요.
  - ADR-0037(가드 Rust 단독, ProtocolClient 박막화)과도 일치.
- **순서 안전 검증됨(general-purpose opus 추적):** "창 mount ↔ 레이아웃 배정" 순서 어긋나도 출력 **유실 안 됨** — 백엔드 3중 안전망(① 데몬 ReplayBuffer ② src-tauri render_seqs 미배달 재시도 ③ epoch reset). 가드가 전부 백엔드라 박막화(프론트 가드 제거) 후에도 안전성 유지. **단 그 race를 직접 때리는 회귀 테스트가 없음 → Stage 1~3에서 fan_out_per_window 미배달 재시도 단위 테스트 추가 권고.**

### 사용자 방향 (철학 재확인)
"WebView는 클라하고만 통신, 역할은 유저 입력 + 출력 표시 정도, 나머지(구독·판단·제어)는 다 클라." = CLAUDE.md §5 손발/두뇌 분리. **단 "WebView 역할을 입력/출력만으로 완전히 좁히기(레이아웃 제어 등 포함)"는 더 넓은 작업 → 별도 후속(나중).** 지금 박막화는 "출력 구독 가드"에 한정.

- 각 Stage: 코더(opus/sonnet) → 게이트(cargo test 가능분 + vitest + tsc) → Stage 2·3은 GUI 실측. 커밋은 게이트 통과 후.
