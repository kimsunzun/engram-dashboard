## 2. 현재 동작 상태 (QA 실측 / 주의)
- **cdp PASS:** embedded·daemon 두 모드 기동 → `__ENGRAM_MODE__` 주입 정확(embedded/'daemon'), `get_autostart` invoke 동작(ACL 통과), `daemon_status` 회귀 0, UI 정상 렌더.
- **현재 기본 = embedded**(인자 없음). daemon 모드는 `ENGRAM_MODE=daemon` 또는 `--mode=daemon` 로 진입. dev(`npm run tauri dev`)는 인자 전달이 까다로워 **`ENGRAM_MODE=daemon` env 로 dev 모드 테스트**(우선순위 2번이라 동작).
- **dev 빌드는 data_dir 모드 무관**(release 에서만 폴더 갈림) — embedded/daemon data_dir 분기는 release 빌드로만 실측 가능.
- daemon 모드에서 프론트가 부팅 시 WS daemon 연결 시도 → `connection superseded` 경고(QA 발견). pre-flip 기지 경로(데몬이 아직 에이전트 호스트 아님)라 스텝3 무관, phase4(flip) 영역.

## 3. ★사용자 수동검증 필요 (cdp 원천 불가 — 네이티브)★
다음은 cdp 가 못 봐(트레이=네이티브 DOM 밖, 창 close/restart=cdp 세션 끊김) **사용자가 직접 확인**해야 함:
1. **embedded(기본 실행):** 트레이 아이콘 **없음**, 창 X → 프로세스 **종료**(작업표시줄·트레이 모두 사라짐).
2. **daemon(`ENGRAM_MODE=daemon`):** 트레이 아이콘 **상주**, 창 X → **hide**(트레이로 최소화, 프로세스 생존), 트레이 "UI 보이기" 로 복귀.
3. **트레이 자동시작 토글(daemon):** 메뉴 "부팅 시 자동 시작" 체크 → 레지스트리 `HKCU\...\Run` 등록 확인, 해제 시 제거.
4. **set_mode self-relaunch:** embedded 에서 `invoke('set_mode',{mode:'daemon'})` → 앱 재기동 후 트레이 떠야 + `__ENGRAM_MODE__=='daemon'`. 반대(daemon→embedded)도. (cdp 로 invoke 는 가능하나 재기동이 세션 끊음 → 재기동 후 새로 cdp 붙어 확인.)
5. **single-instance:** 같은 폴더에서 embedded 2번 실행 → 2nd 즉시 종료(현재 raise 없음). 다른 폴더 → 각자 실행됨. daemon 2번 → 2nd 가 1st 창 raise.

## 4. 핵심 불변식 (이번 세션 추가/유지 — 변경 금지)
- **모드 결정 = run() 최상단 1회**(`discovery::resolve_mode()`), `let data_dir=default_data_dir(mode)` 도 1회(가드키·store 단일출처). 재계산 금지. 전환=self-relaunch(새 프로세스), 런타임 가변 아님. (ADR-0027 보강 §63)
- **`__ENGRAM_MODE__` 주입 = Rust source of truth.** 프론트 localStorage 'engram_client_mode' 는 **Tauri 밖(vitest/브라우저) fallback** 일 뿐(Tauri 안에선 주입이 항상 우선). dev 모드 스위칭은 `--mode`/`ENGRAM_MODE` 로(localStorage 아님).
- **single-instance: daemon=플러그인(전역, 키 커스터마이즈 불가) / embedded=자체 named-mutex(폴더해시).** 두 메커니즘 키 공간이 달라 self-relaunch 모드전환 시 핸드오프 충돌 없음.
- **set_mode = 명시 self-relaunch**(current_exe 깨끗한 argv spawn+exit). `app.restart()` 안 씀(argv 재전달이 모드전환 막음). 데몬 직접 spawn 금지(ADR-0024 C1)는 **데몬 exe 한정** — 앱 자신 재기동은 무관.
- **embedded X=app.exit(0)**(보조 hidden 창 때문에 자동 종료 안 됨). daemon X=prevent_close+hide. (커밋2)
- 기존 불변식(kill 2동사·finalize 1회·StopOutcome→IconState 분기·코어 tauri import 0·tray/core.rs 순수) 전부 유지.

## 5. 이연/미결 (메인이 정하고 보고 — 사용자 확인 권장)
- **embedded-release data_dir = 현 exe폴더 유지.** ADR-0027 본문은 "실행한 **작업 폴더**(cwd)" 라 했으나 현 코드는 exe폴더. release-only라 dev 무영향이라 보수적으로 현행 유지 — **cwd 로 바꿀지 사용자 확인 대기**(`discovery/src/lib.rs` release Embedded 분기 + appdata_data_dir 주석에 명시).
- **앱-in-daemon-mode in-proc store 이중소유** = flip 이연(ADR-0027 보강 §64, pre-flip 잠복).
- **embedded single-instance "기존 창 raise"** = 이연(2nd 는 exit 양보). 폴더별 raise IPC 필요.
- **--hidden 깜빡임**(main conf 기본 visible=true) = 이연(conf visible:false + 비-hidden show).
- **set_mode autostart-부팅 케이스** = 명시 self-relaunch 로 해결됨(argv 재작성). 단 self-relaunch 핸드오프 중 agents.json 경합(atomic write 완화)은 flip 후 재검토.
