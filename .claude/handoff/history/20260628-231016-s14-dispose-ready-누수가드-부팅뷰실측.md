## 결정된 것 (이번 세션)
- **ADR-0039:** 프론트 이벤트 구독 teardown = 동기 `{dispose, ready}` 반환(ADR-0035 후속). dispose 즉시 확보 + 등록완료 ready. 거부 대안: ① async+`if(disposed)`(dead branch — disposed setter가 반환 disposer뿐이라 await 중 set 불가) ② AbortController(Tauri `listen()`이 signal 미지원) ③ async dispose(즉시 취소핸들 요구 미충족). 근거 = cross-family OSS 리서치(보고서).
- **eventBus 통합 불변식(ADR-0039 영향):** dispose push·`import.meta.hot.dispose` 콜백 등록 **둘 다 `await ready` 전**에 끝낸다(ready-pending-중-HMR 누수 방지) + `await ready` try/catch 격리(layout 구독 실패가 agentClient 구독을 막지 않게).
- **메일박스 메시징 단순화(미착수, 방향만):** 목표 ⑤(1→3 에이전트 메시지)을 **fire-and-forget 메일박스 1개**로 좁히면 a1의 경계 BLOCK(응답인식 소유·command_rpc control/data)을 **상당부분 우회**(단방향이라 응답인식 불필요). 남는 정의 = 수신 에이전트가 메시지를 어떻게 소비하나(stdin 주입 vs 화면 표시만 — 데모면 표시만이 제일 쌈).

## 전체 그림 — 목표 데모 시나리오 & 진행도
**목표(오너):** "첫 화면에서 명령 → 에이전트 spawn → 특정 에이전트 화면 표시 → 화면 분리 → 1사분면이 3사분면에 메시지 1개."

```
①첫화면→spawn  ②spawn      ③슬롯에 에이전트화면   ④화면분리   ⑤1→3 메시지
  거의 됨        됨(invoke)   부분→미완(텍스트만)     됨 ✓        미착수
                             =★남은 핵심 덩어리★               =메일박스
```
- 3계층(공용 용어): **webview(React)** ↔ **Client(src-tauri, 레이아웃 권위+invoke)** ↔ **Server(데몬/에이전트)**.
- **webview↔Client(레이아웃): ✓ 완료(이번).** Client↔Server 전송배선(모듈① T5~T8): 남음. 새 슬롯 터미널 렌더(모듈③): 남음.
- ⚠️ 에이전트 출력은 **이미 흐름**(레거시: 각 창이 데몬 WS 직결, `agentClient`/`wsTransport`). 모듈①은 그 carrier를 **Client 단일 choke point 경유로 재배치**(ADR-0036)지 *새로 만드는 게 아님*.

**모듈 진행도:** 백엔드코어·데몬화 = 완료. 모듈① 전송재배치 = **T1~T4✓ / T5(OutputRouter)·T6(invoke)·T7(TauriTransport)·T8 남음**. 모듈② 레이아웃 = 완료(이번). 모듈③ slotStore→viewStore 전체이주(PopupPage/TreePage/SlotContextMenu) = 미착수.

## 실패한 접근 / do-not (carry-forward + 신규)
- **green ≠ correct (신규 재확인):** dispose 가드가 dead-branch인데 테스트는 green이었다(가드가 안 타도 통과 = vacuous). 누수 가드 테스트는 *가드 무력화 시 red 되는지* 자체검증 필수.
- **cross-family 리뷰 가치(신규):** Codex(blind)가 "HMR 콜백이 ready await 뒤라 누수 미완"을 적출(opus는 못 봄). 동시성 변경은 `/review code` 2인 적대 유지.
- **Codex 모델명:** `mcp__codex__codex` 호출 시 `model` 생략(ChatGPT 계정서 `gpt-5.2-codex` 미지원). config는 `model_reasoning_effort`만.
- **cargo 동시 실행 금지:** 중복 백그라운드 cargo/test → 빌드락 데드락. 한 번에 하나. tauri dev도 cargo 빌드라 cargo test와 겹치면 안 됨(순차).
- **coder spawn "거절"이 실제 실행된 사례 있음** → 결과는 working tree 직접 검증, 자기보고 불신(이번에도 핸드오프 stale로 재확인됨).
- **over-loop 경계:** 잔여 좁으면 마지막 라운드로 닫고 커밋. 이번 슬라이스도 코더 2 + 리뷰 1라운드로 닫음.
