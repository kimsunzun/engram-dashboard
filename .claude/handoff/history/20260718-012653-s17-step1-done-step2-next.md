## 스텝 1 구현 구조 (스텝 2가 올라탈 지반 — 코드 앵커)
- **core seam:** `ControlChannel` trait(`crates/engram-dashboard-core/src/agent/types.rs`) + `NoopControlChannel`. `provision -> Result<Option<ControlEndpoint>, ProvisionError>`(순수 std, rmcp/io 누출 없음). 코어엔 rmcp/axum/http **의존 0**(격리 유지).
- **daemon impl:** `crates/engram-dashboard-daemon/src/control/` — `mod.rs`(DaemonControlChannel) · `registry.rs`(ControlRegistry: issue/validate/revoke/`bind_session_if_absent`(정확 토큰 재확인)/`unbind_session` · BoundIdentity) · `mcp_config.rs`(생성/삭제/부팅 sweep) · `mcp_server.rs`(axum+rmcp StreamableHttpService + `bearer_auth` 미들웨어 + engram_ping + McpServerHandle Drop=cancel).
- **provision 지점:** `manager.rs::spawn_agent`(epoch 확정 후, ProvisionGuard RAII로 등록 전 실패 시 revoke). **revoke 지점:** reaper 단일 terminal 수렴(ADR-0019) + `kill_agent` 선제(멱등). **백엔드 조건부:** `backend/mod.rs::supports_control_channel()`(claude=true, shell/codex/gemini=false) — 코어 backend dispatch(ADR-0004, manager-side matches! 금지).
- **auth 상태기계(mcp_server.rs `bearer_auth`):** 토큰 검증 먼저 → 무/틀린/폐기 = **401** → 세션id 있으면: 바인딩+신원일치=통과 / 불일치=**403** / 바인딩없음=**404**(고아 세션 차단) → 세션id 없는 GET/DELETE=**400** / 비-UTF8 세션헤더=**400**. POST(initialize)는 세션id 없이 통과.
- **주의(스텝 2에 직결):** ① 데몬 MCP 서버는 **OAuth 메타데이터 광고 금지**(#59467 — 광고 시 claude가 정적 Bearer 무시). ② `from`은 **토큰에서만 파생**(payload from 무시 — 사칭 차단). send_message가 이 규약을 이어야 함. ③ **워커엔 send_message만 노출**(least-privilege). ④ body limit = `tower_http::limit::RequestBodyLimitLayer`(axum DefaultBodyLimit은 중첩 rmcp 서비스에 미적용 — 이미 교체됨).

- **재실행 명령:** 위 4개 cargo/npm 명령 그대로. GUI 실측은 `WEBVIEW2_ADDITIONAL_BROWSER_ARGUMENTS="--remote-debugging-port=9223" RUST_LOG=info npm run tauri dev` 후 `node scripts/cdp.mjs eval "..."` — spawn은 `window.__TAURI__.core.invoke('forward_daemon_command',{cmd:{CreateProfile:{...output_format:'StreamJson'...}}})` → `{SpawnProfile:{profile_id,resume:false,...}}`. 라이브 MCP 프로빙은 mcp-config의 url+token으로 직접 HTTP.
- **안 한 것(미검·정직 note):** ① **claude 자체 MCP 연결의 라이브 관찰은 간접** — StreamJson claude가 stream-json 입력 대기라 앱 출력으로 system:init 직독 못 함. 데몬측을 *실 생성 토큰*으로 프로빙해 등가 확인 + claude가 Bearer 보내는 건 스파이크 ⑤(동일 config shape, claude 2.1.170)로 증명. ② **"provision 후 transport 실패" 경로는 통합테스트 미커버** — Windows에서 claude ConPTY 스폰 실패를 결정적으로 못 만들어 RAII 논증 + revoke-reclaim 테스트로 대체(코더 명시). ③ 실측 1회 통과 = smoke, race-free 증명 아님(핫패스 정직 note).

## do-not (이번 세션 갱신)
- **스텝 2에서 배달/Mailbox 넣지 마** — 스텝 2 = send_message 툴 + Validator + enqueue-ACK까지. Dispatcher push=스텝 4, SQLite=스텝 3.
- **`agent.spawn` command로 claude 테스트하지 마** — 그건 shell(cmd.exe)을 띄워 supports_control_channel=false라 MCP 경로를 안 탄다. claude 스폰 = CreateProfile(output_format 'StreamJson'/'Terminal') → SpawnProfile.
- **데몬 로그로 검증 시도하지 마** — 데몬은 detached(CREATE_NO_WINDOW)라 stdout이 콘솔 없이 소실. 검증은 mcp-config 파일 + 라이브 엔드포인트 프로빙 + agent output(GetSnapshot)으로.
- **루트 bare `cargo test` 금지** — src-tauri WebView2 크래시(정본 do-not). 멤버별 `-p`로.
- **여전히:** 마커(M3)·Bash 센티널·PreToolUse 훅 주채널 다시 꺼내지 마(ADR-0086이 폐기). engram-ctl 이름 재사용 금지.
