## 2. ★사용자가 원하는 그림 (멘탈모델 — 이걸 사용자에게 먼저 설명)★
tmux 모델의 완성형:
- **데몬 = 영속**(에이전트=claude 등 PTY 자식을 보유). UI 닫혀도 살아있음.
- **UI = 일회성**(붙었다 떨어졌다). 닫아도 데몬 생존 → 다시 열면 재연결(무손실).
- **트레이 = 항상 상주**, 여기서 데몬/UI를 켜고/끄고. 진짜 종료 담당.
- 외부에서 데몬 kill → 자동재시작 없음(명시 재시작만). ← 사용자가 겪은 "막아도 리스타트" 문제 해결.

**용어 주의(사용자가 헷갈려 한 지점 — 설명 시 명확히):**
- "백엔드" ≠ "데몬". 백엔드(에이전트 코어)는 **한 벌**이고, embedded(앱 안 in-process) 또는 daemon(별 프로세스)로 **호스팅**될 뿐. 데몬 = "그 코어를 별 프로세스로 띄운 것".
- "앱" = 프론트(React/WebView) + 얇은 Rust(트레이·invoke)를 담은 **한 프로세스**. 순수 프론트엔드가 아님.
- 패킷(명령 AgentCommand / 출력 OutputChunk / 이벤트 AgentEvent)은 **단일 프로토콜로 공용화**, carrier(in-proc Channel ↔ WS)만 교체(ADR-0020). 그래서 embedded↔daemon 전환은 UI 코드 변경 0.

## 3. ★확정 설계 (ADR-0023/0024) — 재론 금지, 근거는 ADR 본문★
**3프로세스 토폴로지:**
1. **tray-host**(신규) — **순수 Rust(tao + tray-icon, WebView 없음)**. 시작프로그램 상주 런처 + lifecycle owner. 트레이 메뉴 `[UI 열기][데몬 시작][전체 종료]`.
2. **daemon** — 헤드리스, 에이전트 보유, **detached self-owned**(누구의 자식도 아님).
3. **UI 앱** — Tauri/React, 순수 I/O. **X 버튼 = 최소화(hide)**, 단일 인스턴스.

**핵심 결정(C1~C4, judge 검증):**
- **C1** 데몬은 누구의 Job 자식도 아님(detached). Windows JobObject KILL_ON_JOB_CLOSE에 넣으면 부모 죽을 때 데몬도 죽어 생존 위반. 고아는 idle self-shutdown(연결0∧에이전트0).
- **C2** 생사 = WS connection liveness(1차) + lockfile{PID,port,generation} 재발견(stale면 정리 후 fresh).
- **C3** desired-state는 owner(tray-host)가 소유, **UI는 직접 ensure 금지, 연결만**(현 UI bootstrap의 ensure 제거).
- **C4** 전체 종료 = ensure 차단 → 데몬 graceful drain(ack, 타임아웃+Job 강제 폴백) → UI → tray-host 순(재spawn race 차단 = 자동재시작 폐기 사유 재발 방지).
- **데이터** = `.engram-data/`(repo 루트, gitignored, **영속**, swappable). embedded·daemon 공유. tray-host가 경로 결정해 주입.
- **제어 "허락" 모델** = 데몬이 lockfile에 발급한 토큰으로 인증(graceful StopDaemon), force=taskkill 폴백. WS는 `127.0.0.1` 바인드(외부 노출 0). 원격/모바일 노출은 미래 별도 보안작업(보안 1급, 사내 검토).
- commit `664d629`(X=앱종료) → **X=hide로 조정**(단계2에서).

**거부한 대안(다시 꺼내지 말 것):**
- 2프로세스(앱=트레이셸): 교차검증 3종이 "일반적으론 더 단순"이라 했으나, 사용자 의도가 "로그인 상주 런처 + WebView 비용 회피"라 거부. **단 tray-host가 WebView를 쓰면 2프로세스가 맞다는 게 교차검증 결론 — 그래서 tray-host는 반드시 순수 Rust(no WebView).**
- 데몬=앱의 자식+OS wait 감지(Gemini안): detached와 모순 → 기각(judge).
- embedded 완전 휘발: 사용자가 거부(복원 유지 원함). embedded도 영속.

**교차검증 산출물:** `I:\Engram\agents\web-runner\shared\20260618-151957-consult-tray-daemon-topology\`(question/gpt/gemini/claude/blind-*). score-log에 claude 최신뢰 기록.

## 4. ★단계1 구현 스펙 (Plan 에이전트가 코드 실측+기술검증으로 작성 — 바로 코더 투입 가능)★
**목표:** "트레이 떠 있고 → UI 열면 데몬 모드 연결 → 전체 종료 동작 → 데이터 `.engram-data/`." 골격이 도는 것.

**재사용(이미 있음, 새로 안 짬):** 데몬 spawn은 **이미 detached**(WMI `Win32_Process.Create`, WmiPrvSE가 부모라 Job 상속 안 함 → C1 자동 충족). graceful stop·connect·daemon_status 경로 존재.

**신규:** `crates/engram-tray-host/`(tao 0.31 + tray-icon 0.24.1 + image[ico]). 메뉴 3개. 아이콘 `src-tauri/icons/icon.ico` 임베드. Windows: 이벤트 루프 메인 스레드.

**주목 발견 2개:**
1. **WMI는 env 주입 불가** → data_dir은 **CLI arg `--data-dir`** 로 넘김(env 아님). `wmi_spawn` CommandLine + daemon `resolve_data_dir()` 우선순위(CLI arg > ENGRAM_DATA_DIR env > 기본)에 추가, 기본값 `.engram-data/`로 승격.
2. **`discovery.rs`(데몬 spawn 순수 로직)를 공유 crate로 이동**(현재 src-tauri 안) → tray-host가 재사용. `commands/discovery.rs`(tauri wrapper)는 src-tauri 잔류. 저위험 리팩터(discovery.rs는 tauri import 0).

**기존 파일 변경:** 루트 `Cargo.toml`(멤버 추가) · `discovery.rs`(이동+`--data-dir`) · daemon `lib.rs`(resolve_data_dir arg+기본) · `src/api/clientFactory.ts`(기본 embedded→daemon flip, bootstrap connect-only) · `src/App.tsx`(ensure 책임 제거) · `daemonControl.ts`(connect-only로 좁힘). **`src-tauri/src/lib.rs` CloseRequested는 단계1에서 건드리지 않음(X=hide는 단계2).**

**검증 5개:** ①트레이 뜸 ②UI 열기→데몬 연결(cdp eval) ③창닫고 재열기 시 에이전트 생존 ④`.engram-data/`에 daemon.json·agents.json(헤드리스 단언 가능) ⑤전체종료 좀비0(tasklist). headless: data_dir CLI arg 우선순위·discovery 이동 회귀.

**단계1 스코프 밖(명시적으로 안 건드림):** X=hide·UI show/focus·단일인스턴스(→단계2) / 정밀 graceful C4·재입양 generation·idle shutdown(→단계3) / lifecycle 커맨드 표면(→단계4). **단계1 전체종료 = `daemon_stop`(taskkill /T)로 좀비0**(데몬 Job이 PTY 정리), 정밀 graceful은 단계3.
