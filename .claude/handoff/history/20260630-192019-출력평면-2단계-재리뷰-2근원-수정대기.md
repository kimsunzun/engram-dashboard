## 한 줄 상태 + 다음 첫 액션
멀티뷰어 출력 재설계 **2단계(공유 버퍼 통합)** 구현 + 1차 리뷰 BLOCK 2건 수정까지 마쳤고, **재리뷰(code deep, 2라운드)에서 그 수정이 만든 새 결함 2근원이 적출**됨. 게이트(cargo/npm)는 green이나 cdp 실측 0 + 재리뷰 2근원 미수정.

**다음 첫 액션: 재리뷰 2근원 수정(코더 opus).** ① **enqueue 순서 보존** — `send_subscription_delta`가 layout 락 *밖*에서 `ReplaySlots`/`DropSlots`를 enqueue해 enqueue 순서 ≠ rebuild 순서 → 빠른 창 조작 시 drop이 replay 덮음(좀비 cursor/빈 화면). 해법: delta를 락 안 enqueue(try_send는 동기·non-blocking이라 ADR-0006 위반 아님 — layout.rs:50 주석 재검토) **또는** replay+drop 한 cmd로 묶어 순서 고정 **또는** ReplaySlots 처리 시 router 재검증. ② **cursor 생명주기 ↔ Channel 등록 정합** — cursor가 router(layout) 파생이라 Channel 미등록/reload 시 cursor advance가 delivery 없이 일어나 이후 replay no-op(빈 화면). 해법: cursor 생성을 Channel 등록(delivery 가능)에 묶거나 registry miss 시 re-replay. **단 코더 전 갈림길 1건 사용자 결정 필요(↓미결정).** 수정 후 3차 재리뷰 → `/qa full` cdp 멀티뷰 → 커밋.

## 핵심 설계 (정본=문서, 여기는 포인터)
- **ADR-0040 그대로 유효** — 공유 콘텐츠 버퍼 1벌 + per-view cursor + **두 축 분리**(데몬 재구독 `after_seq`=버퍼 최신 seq ↔ 창 read=per-view cursor). 재설계 아님 — 새 machinery의 국소 결함만.
- **이번 세션 사용자 결정:** 2단계 통합부 순수 로직을 **core로 분리해 단위테스트 가능하게**(방향성 변경 아님, 검증 가능성만). → core `OutputViewStore<S>`(Channel 안 듦, "어느 slot에 어떤 bytes" snapshot 반환; send는 src-tauri가 락 밖). headless 테스트로 동시성-치명 로직 박제.
- **데몬 구독 소유 = Rust(src-tauri)** — BLOCK-1 수정으로 프론트 직접 구독 차단(ADR-0035/0037 이행). 구독은 layout 구독 델타가 `after_seq`=버퍼 최신으로 소유.

## 재리뷰 2근원 (다음 세션 수정 대상 — opus동시성+Codex 수렴, opus불변식은 FIX 관대)
- **근원1 [BLOCK급] enqueue 순서 ≠ rebuild 순서.** `layout.rs send_subscription_delta`가 ViewManager 락 밖에서 `replay_slots`/`drop_slots` try_send → 동시 layout invoke 둘의 enqueue 인터리브로 `ReplaySlots`(재배정)가 `DropSlots`(닫힘)보다 actor 큐에 먼저 들어가면 drop이 replay 덮음 → 재배정 창 영구 빈 화면(다음 frame까지). 빠른 toggle/drag 재현.
- **근원2 [BLOCK급] cursor 생명주기(router 파생) ≠ Channel 생명주기(mount).** `sync_viewers`가 router.targets 기준 cursor 신설 → Channel 미등록 창도 cursor advance, `flush_snapshot` registry miss로 유실 + 이후 `subscribe`가 cursor 있다고 불가침(빈 replay). webview reload(Channel 교체) 시 stale cursor → 빈 화면. (Codex BLOCK connection.rs:939 + opus동시성 F3/F4 수렴.)
- **FIX:** `drop_slots` bounded(64) mpsc full 시 silent drop → cursor 영구 누수(ReplaySlots는 자가복구, DropSlots는 복구 트리거 없음). / connect handoff race — `cmd_tx` publish 순서 vs `current_agents` 스냅샷(Codex mod.rs:464).
- **정리:** core `subscribe_agent`/`drop_agent`(agent 단위)는 운영 미사용 데드코드 — 제거 또는 "미사용" 주석(다음 세션이 정본 오인→이중 트리거 위험).

## 이번 세션 서브에이전트 이력
- **코더 opus ×3:** 2단계 1차 구현 / mount-replay·생명주기 갭 배선(실측: 1차가 layout 델타로 이미 배선해둔 것 검증) / BLOCK·FIX 수정(데몬 구독 Rust 이전 + slot 델타 + actor arm).
- **`/review code deep` ×2 라운드, 각 3인**(opus 불변식 breaker + opus 동시성 전용 + Codex blind). **cross-family 2회 실증:** 1차=Codex가 프론트 직접 구독 BLOCK 적출(opus 2인 놓침, Rust 버퍼만 봄) / 2차=opus동시성+Codex가 enqueue 순서·cursor 생명주기 BLOCK 수렴(opus불변식은 FIX 관대). 메인이 1차 BLOCK-1을 핀포인트 코드 검증(protocolClient.ts:299→forward_daemon_command:180)으로 사실 확정 후 사용자 에스컬레이션.
