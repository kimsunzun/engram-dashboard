## M3 설계 산출물 (다음 세션이 만들 것)
1. **마커 프로토콜** — 단일라인 센티넬 + JSON(예: `@@ENGRAM_SEND {"to":"<uuid>","text":"..."}`). 필드·이스케이프·상한 확정. (펜스블록 금지 — 아래 do-not.)
2. **펜스스킵 파서** — ```코드펜스 안 라인은 무시```(실측 false-trigger 방어, 필수).
3. **`OutputEvent::ControlSignal{target, message}`** 신설(`types.rs`) — 디코더가 마커 검출 시 방출.
4. **데몬 라우팅** — 데몬이 ControlSignal 가로채 `write_stdin(target, message)` 호출.
5. **표시 억제** — 마커 라인은 화면(RichSlot)에 안 보이게.
6. **데몬 mail 범위** — **단방향 fire-and-forget 확정**(A→데몬만; 배달·보관은 데몬 소관, 반환 없음). 받은편지함·ACK·영속은 별도 후속.
7. 설계 확정 후 → `/implement` + TDD.

## 핵심 실측 (근거 — claude 2.1.170, `.spike-control/`는 삭제됨·결론은 결정 노트에 박제)
- **M3 마커 방출 near-perfect**: 엄격 단일라인 지시 시 마커 라인 **38/38 완벽**(유효 JSON·수신자/본문·이스케이프). 유일 함정 = 포맷을 *설명*할 때 4/5로 펜스 예시 안에 마커 찍음 → **펜스스킵 파서 필수**. 단일라인 센티넬 > 펜스블록(펜스는 실물/예시 구분 불가).
- **라이브/리플레이 경계 코드 확인**: resume seed = `output_core.seed()`로 **fanout 없이** 적재, 라이브만 `emit()` 경유 → **마커 파서는 리플레이/transcript seed를 절대 안 봄**(부작용 재실행 없음). ADR-0079 정합.
- **M4(hook)도 발화 확인**(우리 헤드리스 호출서 PreToolUse 발화·command 캡처·`--settings` 주입·deny 차단) — 하지만 "우리가 콜백받는" 게 아니라 간접 릴레이 + 데몬 인그레스 없음(WS만) → **문서화 폴백으로 보류**(터미널모드·포맷붕괴 헤지용).

## 구현 seam (실측 매핑 — 참고)
- **공통 배달**: `AgentManager::write_stdin(agent_id, data)` (`manager.rs:638`) → `session.write_input` → 대상 stdin. 데몬 호출부 `connection_core.rs:714`. 에이전트 lookup `manager.rs:740`.
- **M3**: `OutputEvent` 변형 추가(`types.rs`) · 디코더 마커검출(`backend/claude.rs`, `ClaudeStreamDecoder::decode` ~318-364, 라이브 `TextDelta`) · 데몬이 ControlSignal 가로채 라우팅(`ws.rs`/`connection_core.rs`). **core 격리(ADR-0003)**: 마커 검출은 core에서 **이벤트 방출까지만**, 라우팅은 데몬(core에 tauri/데몬 의존 금지).

## do-not (반복 방지)
- ★**펜스스킵 없는 파서 금지**★ — 에이전트가 포맷 설명할 때 오발화(실측 4/5).
- ★**펜스블록 마커 금지**★ — 실물 마커와 예시 마커 구분 불가 → **단일라인 센티넬** 채택.
- ★**마커 파싱을 라이브 stdout에만**★ — 리플레이/transcript seed 경로엔 걸지 마라(재실행 방지, 근데 seed는 애초에 fanout 안 함).
- ★**engram-ctl 다시 만들지 마라**★ — 폐기 결정. (API 백엔드는 직접 tool 콜백=자명, CLI는 M3 — capability matrix ADR-0002 백엔드별 구현.)
- core에 tauri/데몬 import 금지(ADR-0003).

## 열린 항목 / 미결
- **정식 ADR 미승격** — engram-ctl(ADR-0014/0080) supersede + M3 결정을 `/adr`로 박제해야 함(현재 결정 노트만).
- **PRD/TRD 갱신** — TRD `spec/trd.md` §6은 engram-ctl 전제(폐기 대상, 단 에러 시맨틱·계약 의도는 참고 가능). Unit 재설계.
- ★**병렬 세션 조율**★ — 다른 세션이 engram-ctl 크레이트를 만들고 있었음(이 세션이 폐기). 그 세션 살아있으면 M3 피벗 알려 중복작업 막을 것.
