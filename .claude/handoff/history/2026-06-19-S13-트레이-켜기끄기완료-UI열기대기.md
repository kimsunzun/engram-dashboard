**작업 크기(사실 기반):**

## 4. 이번 세션 핵심 결정/주의
- **끄기 = graceful 일방 발사**(사용자 결정): "꺼라" 보내고 ack 안 기다림. 강제 종료(taskkill) 폴백·ack 대기는 **나중**(`daemon_stop`이 폴백 자리로 보존됨). 응답 없으면 데몬 활성 유지, 다시 누르면 재발사.
- **아이콘 갱신 = 액션 직후 + 끄기 시 "연결 닫힘" 신호**(옵저버 경량형). **외부/크래시 죽음은 아직 미감지**(주기 probe/상시 연결 = C2 완전형, 다음). 사용자가 이 한계 인지·수용.
- **데몬 콘솔:** 디버그=콘솔 창(로그, `RUST_LOG=debug`로 데몬 로그 가시), 릴리즈=무창. 사용자가 "디버그 창 있이" 결정.
- **사용자 협업 방식(신규 CLAUDE.md 섹션):** 사용자는 코드 용어 학습 중. 결정은 동작·정책으로 번역, 처음 용어는 한 줄 풀이, 개념 합의 후 용어 풀 브리핑. 작은 기능은 PRD/TRD 묶기.

## 5. ★rot 방지 메모 (다음 세션 필독 — reviewer-deep 적출)★
- **끄기 진입점 비대칭:** 운영 끄기는 트레이 워커가 `discovery::send_stop`을 **직접** 호출(StopOutcome을 LaunchError가 못 올려서). `RealLauncher::stop_daemon`·`core::dispatch(StopDaemon)`은 운영 경로 미사용(테스트·향후 LLM 제어 표면용 존속). **C4 owner 상태머신(sub-step 3)에서 단일 진입점 통합 예정** — 두 경로 동작 차이 모르고 한쪽만 고치지 말 것(§5 단일 control surface와 임시로 어긋남).
- **S13 끄기 변경 시 필수:** `cargo test -p engram-dashboard-discovery --test stop_smoke -- --ignored --test-threads=1`(분류 정확성 end-to-end 검증이 #[ignore]라 자동 회귀망 밖. `--test-threads=1` 필수 — 실 데몬 mutex 공유).
- **`daemon_stop`(taskkill)·`StopOutcome::DaemonClosed` 주석:** 보존·의미 한정 주석 있음(지우거나 과신 말 것).
