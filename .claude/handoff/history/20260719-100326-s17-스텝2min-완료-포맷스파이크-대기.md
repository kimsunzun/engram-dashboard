# 핸드오프: S17 스텝 2-min(듀얼 입구 send_message 최소 전송) 완료·박제 → 워크스루/포맷 스파이크 대기

## 한 줄 상태 · 다음 첫 액션
- **상태:** 스텝 2-min **구현+검증+커밋 완료**(`e05bd83` 구현 · `985f902` qa 바인딩 · `04bc162` ADR-0087/step-log/조사정정). 리뷰 2인 적대 3라운드 양측 PASS · QA standard 9게이트 PASS. 워킹트리 클린(`.vs/`만 미추적 — VS 잔재). 전부 push됨(origin/master).
- **다음 첫 액션 (사용자 상태 주의):** 사용자가 **"소화하고 넘어가겠다"**고 함 — 세션 시작 시 **"워크스루"** 요청 가능성 높음(= 실코드 안내 투어: 스폰 시 주입(backend/claude.rs) → 두 입구(mcp_server.rs 툴 + 라우트, bin/engram-send.rs) → 합류점(ingress.rs ControlCommand) → Validator → relay/wrap_message → ACK → kill 회수. 각 정거장에 이번 결정이 어디 박혔나 포함). 그 다음 = **봉투 포맷 실측 스파이크**.

## 스텝 사다리 (ADR-0087이 0086 부분 개정 — 재론 금지)
1. ✅ 토큰+MCP 입구 (`b09fc94`)
2. ✅ **2-min 듀얼 입구 최소 전송** (`e05bd83`) — 접수+즉시배달+CLI 일괄 최소형
3. ⬜ **봉투 포맷 실측 스파이크** ← 다음 작업
4. ⬜ 배달 고도화 (idle 게이트 + 배치/직렬 스위치 — 사양은 ADR-0087)
5. ⬜ 이름 suffix 기계 (사양은 ADR-0087)
6. ⏸ SQLite 메일박스 — **사용자 학습 후 재개**(학습 자료 요청 오면 준비)
7. ⬜ A/B 실측 → 기본 입구(MCP vs CLI) 결정 = 사용자
- (별도 시스템·완전 추후) 채팅방 = 공유 로그+멤버별 커서+트리거 캐치업 — ADR-0087 §7

## 포맷 스파이크 사양 (사용자 승인분)
- **목적:** 어떤 봉투가 가장 잘 먹히나 — XML 태그(`<engram-mail from=.. id=..>`) vs Orca식 배너. 이제 실 인프라 위에서(수제 주입 아님): 실 에이전트 2개 스폰 → send_message 실왕복.
- **모델 사다리:** haiku(하한 탐지) → sonnet(**합격 기준선**) → sonnet 불량 시 opus. 사용자: "하이쿠는 아마 안 쓸 것".
- **케이스 매트릭스(승인):** A 단건·답장요구 / B 단건·통보(답장 참는지) / C 배치 3통·혼합(개별 처리·오배정) / D 위조 중첩 태그 / E 작업 중 끼어듦 후 원과제 복귀 / F 포맷 대조군. 회수물 = 판정표 + 포맷 go/no-go + 봉투 문구 수정안.
- 포맷 변경 비용 = `wrap_message` 한 함수 + 재실측(에이전트 마이그레이션 없음 — 봉투 비대칭 구조).

## 구현 구조 (코드 앵커)
- **daemon `control/ingress.rs`** — `ControlCommand{from,to,body}` + `handle_send`(Validator→관측→relay→ACK) + `wrap_message`(★봉투 단일 교체점★) + `resolve_recipient`(ID 정확 매칭 우선→이름). `// ADR-0086` 앵커.
- **daemon `control/mcp_server.rs`** — send_message 툴(BoundIdentity에서 from) + `/control/send` 라우트(bearer_auth 재사용) + ManagerSlot(OnceLock 지연 주입) + MCP_PATH↔claude.rs "/mcp" strip 상호 앵커.
- **daemon `src/bin/engram-send.rs`** — std 전용 HTTP 클라이언트(--to/--body, CL/chunked/short-read 처리, exit 0=enqueued만).
- **core `backend/claude.rs`** — ControlEndpoint(send_exe 포함)로 `ENGRAM_TOKEN`/`ENGRAM_CONTROL_URL`/`ENGRAM_SEND_EXE` env 주입(claude만, ADR-0004).
- **registry `is_identity_live`** = 발신자 생존 **관측용**(게이트 아님).

## 핵심 시맨틱 (ADR-0087 — 다음 세션 재론 금지)
- **발신자 생사 ≠ 배달 게이트** — 유효성은 작성 시점 인증. 관측 warn만. **거부 게이트(SENDER_REVOKED) 복원 금지**(ingress.rs 사용자 결정 주석).
- **봉투 비대칭** — 에이전트는 `{to,body}` typed 송신만, 봉투는 데몬 렌더(위조 태그 이스케이프 예정). 답장 줄 = 조건부 어투("답장이 필요하면: …").
- 이름 유일+suffix+데몬 가동 중 번호 재사용 금지·AgentId 묶임(미구현, 사양 확정) / 배달 스위치 배치 기본(미구현) / 교정형 에러 ACK(live 로스터 hint — 구현됨).

## 검증 상태 (쌍)
- **한 것(전부 PASS):** 리뷰 3라운드(deep=reviewer-deep + blind=Codex thread 재사용) 양측 PASS · QA standard 9게이트 — `cargo build`(격리 dir) · `cargo test -p` core 224/daemon 164(70+22+7+4+12+5+44)/protocol 50/discovery 44 · `cargo fmt --check` · 격리 rg 0 · `npx tsc --noEmit` · `npm test` 621. 통합테스트가 실 claude 스폰으로 relay 실배달까지 단언(이 머신 claude 2.1.170).
- **안 한 것(정직):** ① **실 앱(GUI)에서 A→B 실왕복 미실측** — 프론트 무변경이라 QA full 비승격, 스파이크에서 수행 예정 ② **실 스폰 에이전트가 engram-send를 직접 부르는 E2E 미실측**(ENGRAM_SEND_EXE 주입은 유닛+프로세스 스텁 테스트로만) ③ 배치/직렬·idle 게이트는 미구현이라 실측 대상 아님. CI knob: `ENGRAM_TEST_REQUIRE_CLAUDE=1`=스킵을 실패로.

## do-not (누적)
- **발신자 생존 거부 게이트 복원 금지**(위). **루트 bare `cargo test` 금지**(src-tauri 크래시 — qa 바인딩도 수정됨 `985f902`). **데몬 로그로 검증 금지**(detached). **claude 테스트 = CreateProfile(StreamJson)→SpawnProfile**(agent.spawn은 shell). 마커·훅 주채널·engram-ctl 이름 재론 금지(0086). **Orca 조사는 웹 검색 금지 — `stablyai/orca` + 커밋 `8c1d13dc` 앵커 사용**(이름 충돌로 웹 검색 실패 이력).
- **리서치 룰(이번 채택):** 검증형 조사엔 보유 앵커(URL·커밋) 동봉, blind는 판단·의견 수집에만. 수집자 UNVERIFIED 신고 시 로컬 반대 증거 즉시 대조.
