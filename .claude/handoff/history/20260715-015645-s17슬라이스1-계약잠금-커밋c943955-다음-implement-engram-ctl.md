## 구현 스코프 (슬라이스 1 = 메시지 길)
1. **engram-ctl Rust CLI 신설**(workspace 새 멤버 crate 권장 — 배치는 내부 결정): portfile(`daemon.json`) discovery(discovery crate 재사용) → WS → Auth(`ENGRAM_CONTROL_TOKEN`) → 기존 AgentCommand 직송 → **v1 봉투 stdout**(`{v,ok,requestId,result}`/`error{code,message,retryMode}`, exit 0/1). 서브커맨드 = `agent list` · `agent write <uuid>` · `list-commands`. node 스파이크 `scripts/engram.mjs`가 경로 증명(Rust 재작성 대상).
2. **spawn env 오버레이**(`manager.rs:254` seam, 비영속 — profile.env에 안 넣음): `ENGRAM_CONTROL_TOKEN` + PATH prepend(ctl 번들 dir) + `ENGRAM_AGENT_ID`/`ENGRAM_AGENT_NAME`(발신자 표기 원천).
3. **데몬 백엔드 명령 dedup store**(net-new, §5 잠금 설계): 데몬-전역 `request_id→캐시 결과`, TTL=`retryWindowMs`(기본 300000), 같은 id+다른 페이로드=`REQUEST_ID_CONFLICT`, in-flight=coalesce, 완료분=캐시 봉투 재생.
4. **에러코드 합성**: 데몬은 자유 문자열 `Error{message}`뿐(typed 없음) → ctl이 패턴매칭으로 code 합성 + **데몬 실문자열 golden 테스트로 고정**.
5. **발신자 표기**: `[from: <이름(한줄 escape·64자)> (<풀 UUID>)] ` 자동 프리픽스, env 정체 없으면(사람 직접 호출) 무표기.

## do-not (리뷰가 적출한 구현 함정 — 어기면 재리뷰서 또 잡힘)
- ★**lease acquire→write→release 3커맨드 구현 금지**★ — 실물은 **빈 lease면 `WriteStdin` 통과**(`check_input` Allow, `connection_core.rs`), 타 연결 보유 시만 LEASE_DENIED. 댄스를 만들면 중간 크래시 시 lease 고아(사람 입력 잠김) 창을 신설하는 것.
- `mode` 필드는 wire `AgentInfo`에 없음 — `structured`는 `capabilities.output`에서 파생(`domain.rs`).
- 에러코드 typed 아님 — 문자열 매칭 + golden. status enum 값도 `messages.rs` 대조로 나열(날조 금지).
- 터미널 대상 자동 `\r` 금지(raw 전달·자동 제출 없음 — TRD §6-2).
- member-scoped 테스트만(bare `cargo test` = WebView2 크래시) · 커밋 메시지 `-F <file>`(win32) · GUI 실측 전 데몬 kill→`cargo build`→재기동(옛 바이너리 트랩).
- `docs/reference/architecture-overview.md`(미커밋 잔여) = S17 병렬 세션 것 — 건드리지·커밋하지 마라. repo 루트 PNG들 = throwaway(삭제는 사용자 확인).
