## ★ 확정 아키텍처 (사용자 승인 — 풀 브리핑 완료) ★
```
[스폰 시 1회] 데몬: (AgentId, epoch)별 토큰 발급
   ├→ MCP용: 에이전트별 mcp-config 생성 (127.0.0.1:PORT + Bearer 토큰 헤더)
   └→ CLI용: PTY env 주입 (ENGRAM_TOKEN)   ※epoch 회전=구토큰 폐기, kill=즉시 폐기

[런타임] 에이전트
   ├─[입구① McpIngress] send_message tool_use → 웜 연결(시작 시 1회 바인딩) — 기본 후보
   └─[입구② CliIngress] Bash("engram send --to B ...") → 콜마다 접속+env토큰 (스텝5)
        ▼ ControlIngress seam — 둘 다 정규화
   ControlCommand { from*, cmd, args }   *from = 페이로드 아님, 토큰에서 파생(사칭 차단)
        ▼
   Validator (수신자 실존·스키마·상한 · 후일 authz)
        ▼
   Mailbox: SQLite WAL append = ★source of truth★
     (id·from·to·sequence·body·created_at·delivered_at·read_at)
        ▼
   Dispatcher: 수신자 온라인→ write_stdin push (manager.rs:638 재사용)
               오프라인→ 장부 대기, 복귀 시 배달. delivered_at = 재주입 방지
        ▼ 동기 enqueue-ACK (입구 무관 동일 JSON)
   성공 {"status":"enqueued","message_id","sequence"} / 실패 unknown_recipient·invalid_schema·size_limit
```
- **two-level ACK:** 동기 = "장부에 실렸다"까지만("읽었다" ACK 없음). 의미적 응답 = 별도 메시지/스레드. (생태계 지배 패턴)
- **커맨드 = 의도별 전용 툴 분리**(제네릭 메가툴 사례 0). 이번 스코프 = **send 하나만**, spawn/창이동 등은 additive로 나중에. **워커에겐 send_message만 노출**(least-privilege = authz + MCP 컨텍스트 세금 동시 해결).
- **TUI 모드 = 제어채널에서 제외**(사용자 결정 — 별도 처리 없이 "파싱 안 되니 자연 제외").
- **데몬 = 리스너.** 입구 연결은 인바운드로 들어와 핸들러 발화(콜백) — claude.exe는 MCP에 **시작 시점에** 붙음(system:init에 툴 목록 뜨는 이유) → "연결 성공"이 스폰 직후 검증 가능한 마일스톤.

## 스텝 사다리 (스텝 1 승인됨 — 한 스텝 = 한 검증 게이트)
1. **토큰 + MCP 입구 연결** ← 승인·착수 대상. 수용 기준: 실 에이전트 스폰→데몬에 (AgentId,epoch) 세션 바인딩 확인 / 틀린·구(舊) 토큰 거부 / 에이전트 `system:init`에 우리 툴 노출 확인.

## 인증 설계 (확정)
- 스폰 시 (AgentId, epoch)별 토큰 발급 → 레지스트리. **MCP = mcp-config에 박아 연결 시 1회 바인딩**(토큰이 모델 셸 밖 — claude.exe 전송층이 자동 처리) / **CLI = env로 콜마다 제시**(콜=프로세스라 신원 고정 불가; 비용 주범은 인증이 아니라 프로세스+연결 재수립).
- `from`은 항상 토큰에서 파생(페이로드 from 무시). 위협 모델 = 프롬프트 주입/오작동 에이전트의 사칭·오귀속(같은 OS 유저라 하드 격리는 원래 불가 — 최종 방어는 Validator).

## 이 세션 스파이크 실측 (재사용 가치)
1. **stream-json 봉투** (`claude -p --output-format stream-json --verbose`): tool_use = assistant content 블록, **complete 봉투에선 input fully-formed**(델타 재조립 불요 — `--include-partial-messages`일 때만 partial_json). 봉투 타입이 가설보다 많음(user/rate_limit_event/system:thinking_tokens/system:status/stream_event) → **방어적 파싱 필수**. `result` = 프로세스 완료(원샷), stop_reason 확정값은 message_delta에만.
2. **Bash 센티널 컴플라이언스** (haiku): 6회 중 4회 clean-parse(66%), 따옴표 body에서 깨짐, nonce 오기재 1회, 모호한 수신자면 되묻기 — **이 접근 자체가 폐기됨**(조사 결과 채택 0건 패턴).
3. **훅 기제** (문서): PreToolUse는 headless에서 발화 OK, deny에 **idle 버그 #24327**(간헐) — 훅 주채널 접근 폐기.
4. **MCP 동적 로드 불가** (문서): 서버는 시작 선언 필수(system:init 후 불변). `tools/list_changed`는 지원(연결된 서버의 툴셋 변경). dormant 비용 미미(tool search deferred, v2.1.191+ 기본 — 이름만 로드).

## do-not (이 세션 갱신분 — 이전 것 일부 번복 주의)
- ★**Bash-true 센티널·텍스트 마커 다시 꺼내지 마**★ — 최종 폐기(두 독립 조사 채택 0건 + 스파이크 66%). ADR-0085의 "마커=주채널"은 죽은 결정.
- ★**PreToolUse 훅을 주 전송로로 쓰지 마**★ — 수제 MCP(콜당 스폰 + idle 버그 + 부품 과다).
- ★**"engram-ctl 금지" do-not은 공식 번복됨**★ — typed CLI 입구(스텝 5)로 부활(Orca가 실증). 단 이름은 engram-ctl 재사용 금지(혼동) — `engram-send` 또는 데몬 서브커맨드, 메인 재량+보고.
- **claude-flow(ruflo) 참조하지 마** — 64k⭐에도 placeholder 다수, retry 로직 결함 추정(Codex 코드 확인).
- **Codex web_search 미노출 지속** — 영구설정(`~/.codex/config.toml` web_search="live")+재시작 후에도 `NO_NATIVE_WEBSEARCH`. 단 **"sol 빌드 회귀냐 원래 안 됐냐" 원인 미판정 — 단정하지 말 것**(사용자 지시). Codex 리뷰는 web_search disabled 순수추론으로.
- 사용자 협업 교훈: **실행 전 보고·승인**("보고를해 결정되면 진행을하지말고") — 스파이크·조사 돌리기 전 계획 먼저 확인받기. 스텝 단위로 쪼개서 진행.
