## 핵심 설계 결정 (다음 세션이 반드시 알 것)
- **아키텍처 = ADR-0081.** engram-ctl → 데몬 WS → 백엔드=`AgentManager` 직행 / UI=데몬 **opaque-relay** → 앱. 앱은 relay 받은 UI 명령을 **사람 Tauri 경로와 공유하는 transport-중립 `ViewCommand` 적용 서비스**로 ViewManager에 적용.
- **★재진입 데드락 회피(핵심 함정):** relay는 `daemon_client` 액터 **밖(spawn 태스크)에서** 적용하고 `request_id→oneshot`로 상관한다. 액터에서 인라인 await하면 compound 명령(`spawn_into` — 자기 안에서 `client.send_command().await`)이 self-deadlock(코드 확인: `src-tauri/src/commands/layout.rs:483`, 액터 `daemon_client/connection.rs`; 데몬 head-of-line은 `ws.rs:763`).
- **request_id end-to-end**(별도 relay_id 없음). engram-ctl `--request-id`가 상관+dedup **통합**(옛 별도 --idempotency-key 제거). 없으면 fresh uuid=at-least-once.
- **dedup = 데몬-전역**(per-call 스폰이라 per-connection 불가), **바운드(TTL/LRU — 데몬 lifetime 무한 아님)**, **UiResult(적용) 후에만 커밋**(사전 캐싱=거짓 성공 금지), in-flight 중복=coalesce.
- **lease = per-call 자동획득(a).** engram-ctl은 호출마다 새 연결이라 per-connection lease가 증발 → `write`가 그 호출 범위로 acquire→write→release, 사람 보유 시 LEASE_DENIED.
- **RegisterRole 재연결 재전송 + last-wins**(안 그러면 재연결 후 UI relay 전부 APP_OFFLINE).
- **토큰/discoverability = spawn env 오버레이**(`manager.rs:254` seam, env 인프라 기존). 토큰=env var(`ENGRAM_CONTROL_TOKEN`), engram-ctl=**PATH prepend**.
- **순서 = ViewManager 단일 Mutex 직렬화**(`layout/mod.rs:29`, 실측 확인) — 재진입 안전은 위 async 상관이 담당.
- **error 확실성 코드:** APP_OFFLINE=미적용 확정 / APP_DISCONNECTED·TIMEOUT=불명(read로 재조정). STALE_REF·LEASE_DENIED·UNSUPPORTED·ALREADY_APPLIED.
- **opaque 결합 가드:** relay 봉투는 protocol crate에 opaque payload만. UI enum·dispatch 맵은 src-tauri에만. engram-ctl·데몬·core는 UI enum import 금지(core tauri-free 유지).
