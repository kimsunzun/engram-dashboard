## ⭐ 규약 — 커밋 금지 (사용자 결정, 이 세션) ⭐
**자율 모드에서도 자동 커밋 금지.** 커밋은 사용자가 명시 지시할 때만. "계속 진행"은 파이프라인 진행 지시이지 커밋 승인 아님. 게이트 통과해도 메시지 초안 + 스테이징 대상만 제시하고 승인 대기.
- `implement` 스킬 SKILL.md + flow.md §5에 반영(미커밋 — `/review doc` 후 커밋). 로드된 스킬 hot-reload 안 되므로 다음 세션부터 자동 적용.
- 이유: 자동 커밋이 워킹트리를 clean으로 만들어 커밋 전 diff 검토 기회를 뺏음.

## ⭐⭐ F1 — 순서 위험 (다음 세션 필독, 리뷰 적출) ⭐⭐
tag1 frame은 클라 relay·공유버퍼는 통과하지만(`connection.rs:1008` decode_frame이 tag1 OK, cursor 전진), **프론트 `src/api/wsFrame.ts:20`이 `tag !== FRAME_TAG_TERMINAL_BYTES`면 null 반환 → `wsTransport.ts:256`이 조용히 버림.** 그래서:
- **B3(decoder→pump 배선 = tag1 생산자)를 프론트 tag1 소비(모듈6) 없이 하면**, tag1이 흐르기 시작하는 순간 프론트가 매 frame을 null-drop → **그 seq는 cursor가 이미 전진해 replay로도 복구 불가 = 영구 무음 유실.** "왜 구조화 출력이 안 뜨지"를 데몬·codec 다 뒤진 뒤에야 프론트 tag 가드에서 찾게 됨(디버깅 비쌈).
- **해소:** B3 배선 커밋에 프론트 tag1 소비(wsFrame tag 분기 + wsTransport/protocolClient/RichSlot)를 **묶거나**, 최소한 wsFrame.ts를 tag1 통과+상위 미처리 명시 seam으로 먼저 열어둔다. 즉 TRD §6 순서(⑤B3 → ⑥F)를 그대로 따르면 위험 — ⑥의 프론트 tag 관용을 ⑤와 함께/먼저.
- 지금은 tag1 생산자 미배선이라 무해(잠복).

## ⭐ 리뷰가 잡은 버그 / do-not (재도입 금지) ⭐
1. **Ring이 최신 이벤트 evict(모듈3 Codex High, 교정됨).** cost_bytes>2MB 단일 이벤트가 최신 seq까지 빼냄 → `while items.len()>1 && (over budget)`로 최신 1건 보존.
2. **result.is_error 은닉 + interrupt 오분류(B2, 교정됨).** 오류 판정은 error allowlist(`starts_with("error")` OR is_error). `!= "success"` denylist는 정상 interrupted를 실패 위장 → 금지.
3. **malformed 블록 가짜 정형 이벤트(B2, 교정됨).** name 없는 tool_use는 Structured 보존, text 없으면 skip.
4. **오버플로 resync 부재(B2, 교정됨).** `discarding` 상태로 다음 개행까지 폐기.
5. **snapshot() 구조화 무음 유실(모듈3, warn 부착).** `snapshot()->Vec<OutputChunk>`가 구조화 drop(TerminalBytes만) — `_ => None`에 tracing::warn 있음. **get_snapshot 구조화 wire 매핑은 후속**(core snapshot() 변경 필요, 이번 스코프 밖). B3 배선 시 이 경로도 주의.
6. **F1 프론트 tag1 null-drop(위 ⭐⭐).**
7. **estimate_cost_bytes 과소평가(모듈3·4 재지적, won't-fix 결정).** Usage(turn_id=None) cost=0 → byte 예산 헐거움. **event-count 상한(4096)이 하드 백스톱**이라 무한성장 없음 → 근사 설계 유지. 향후 backend가 Usage에 큰 문자열 추가 시 재검토.
8. **wire=StructuredEvent 신설(OutputChunk 확장 아님).** OutputChunk는 AgentEvent::Output/Snapshot 계약에 묶여 확장 시 파손 → 새 타입. OutputChunk는 GetSnapshot 전용으로 남김. 되돌리지 말 것.

## 확정 설계 (정본 = ADR-0045 + S15 TRD)
흐름: claude NDJSON → StdioTransport pump(바이트) → **decoder(B2✅)** → OutputEvent → **OutputCore Ring<StoredOutput>(B4✅, ADR-0006 락)** → **OutputPayload{Bytes|Event}(B5✅)** → **daemon adapter output_event_to_wire(B7✅)** → WsOutputSink(Bytes→tag0 / **Event→StructuredEvent JSON→tag1**, B7✅) → codec decode_frame(헤더-only) → 클라 공유버퍼 append/replay 재사용(B8✅ opaque 확인) → **[⑤B3 생산자 배선] → [⑥ 프론트 wsFrame tag 분기]**.
- wire tag1 payload = `StructuredEvent`(protocol, `#[serde(tag="type")]` self-describing JSON, OutputEvent 충실 미러, ts-rs export). 필드 유실 0(turn_id/message_id/id 보존, MessageDone/Error/Structured 포함).
- 불변식: ADR-0006(emit lock 밖 send) · ADR-0003(core는 wire 모름, 변환 daemon adapter, core Serialize 미부착·tauri import 0) · ADR-0004(claude 스키마 decoder만) · ADR-0002(출력 종류 비가정).
- WsOutputSink Event 직렬화 실패/None = warn+drop+Ok(연결 안 죽임) — 현 필드(String/u64/Option)로는 실패 unreachable.
