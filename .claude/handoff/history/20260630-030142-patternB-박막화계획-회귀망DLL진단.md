## 이번 세션 성과 — 결정·계획 (휘발 산출물 영속화)

### 1. 패턴 B 확정 (리서치 light로 검증)
출력 재동기화 = **패턴 B(Kafka consumer offset / SSE Last-Event-ID 식)**:
- **데몬(에이전트 호스트)** = dumb provider, seq 붙인 출력을 요청 범위만 제공.
- **src-tauri DaemonClient/OutputRouter** = "Client" 레이어 — 창별(per-window) 커서·세대(epoch)·중복제거 가드 **단독 소유**, 평소 tail resume, **어긋남 감지(세대변경·seq gap·버퍼초과) 시 full reset+replay 폴백**.
- **프론트 ProtocolClient** = 박막화(받은 청크 그냥 렌더, JS 가드 제거). agentClient 인터페이스(ADR-0011)는 불변.
- = **ADR-0037 완성**. 리서치(Sonnet+Codex BLIND, 불일치0 수렴)가 지지. 사용자 직관("UI 사정은 Client가, 데몬은 달라는 것만")과 정확히 일치. 출처: Kafka offset·SSE Last-Event-ID·mosh SSP·tmux·Zellij.
- ⚠️ 새 ADR 불필요(ADR-0037이 이미 이 방향). 단 "어긋남→full 폴백" 세부정책은 TRD/ADR 한 줄 박을 가치.

### 2. Plan 핵심 발견 (opus, 박막화 계획서)
- **Rust는 이미 패턴 B 거의 완성:** 창별 커서(`output_channel.rs WindowEntry.render_seqs`)·epoch가드(`protocol_state.rs decide_epoch`)·dedup(`output_channel.rs should_deliver`)·어긋남감지(`apply_subscribe_ack`)·**full폴백(`reset_all_windows_for_agent` + 데몬 FromOldest)** 전부 구현됨.
- **프론트 ProtocolClient 가드 = 죽은 2차 방어선.** `handleOutput`의 dedup/epoch, `resubscribeAll` = ADR-0037이 거부한 이중화 잔재.
- **SubscribeAck는 Rust가 내부 소비, 프론트로 안 올라옴**(리뷰 지적 확인) → `protocolClient.ts`의 SubscribeAck 분기는 **wire상 도달 불가능한 죽은 코드.** 박막화 후 프론트에 불필요(확실).
- **박막화 = 프론트 잔재 제거가 주작업, Rust 추가구현 거의 불필요.** 작업량 작음.

### 3. 단계 분해 (Plan)
- **Stage 0** — 갈림길①② 사용자 결정(코딩 전 게이트). **통과 전 코더 진입 금지.**
- **Stage 1** — 프론트 박막화: `handleOutput` dedup/epoch 가드 제거 + `SubState` 축소(onChunk만) + SubscribeAck 분기 제거. 위험 中, **deep 불필요**. `protocolClient.test.ts`의 dedup/epoch 케이스 삭제/이전 필요(안 그러면 깨짐).
- **Stage 2** — `resubscribeAll` 제거(+호출처). ★단 `onConnectionStateChange`의 pending reject는 보존(promise lifecycle, 가드 아님)★. 위험 中, **GUI 실측 필수**(데몬 재시작→재연결→출력 무손실).
- **Stage 3** — 구독 이중경로 해소(갈림길② 결정 반영). 위험 **높음(동시성 치명)**, **`/review code deep` 대상**.
- **Stage 4** — `TerminalSlot.tsx` 컴포넌트 dedup 제거. ★`terminal.reset()`·epoch deps `[agentId,epoch]`는 보존(렌더 책임)★. 위험 낮음.

## 리뷰 findings (3인 종합 BLOCK — 결정 후 코더가 닫을 것)

스폰: reviewer-deep(opus)×2 + Codex blind. 종합:
- **① 구독/epoch 이중권위 → epoch 재시작 시 출력 전멸 + 이중구독** (BLOCK 핵심) → **패턴B(박막화)로 해소.** ※전멸 트리거는 "재구독 없는 재시작+재연결 겹침"에 한정(빈도 낮음). UI 단순 재구독은 안전(새 SubState가 기준 -1 리셋).
- **② 출력 Channel 등록 race** (high) — (a)등록 전 connected 노출→초기/재연결 유실 (b)비원자 발사 좀비 Channel (c)이미 connected인 데몬에 새 창 attach 시 전이 못 봐 등록 안 됨=멀티창 2번째+창 출력0건.
- **③ 30s reply 타임아웃** (med) — Rust pending 누수 + 부작용명령(WriteStdin) 재시도 시 입력중복 위험 → 조회명령 한정 또는 "재시도 금지" 명시.
- **④ 부팅 listener 등록 race** (med, 호출순서 미확인).
- **⑤ app:None 게이트 위장** (high) — 통합테스트 위양성인데 STATUS_ENTRYPOINT_NOT_FOUND로 가려짐. 정공=connection task가 `Option<AppHandle>` 받기. + 정식 ADR 채번 필요.
