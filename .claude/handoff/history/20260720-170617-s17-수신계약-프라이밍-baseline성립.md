# 핸드오프: S17 수신 계약 첫 슬라이스 완료 — 프라이밍 baseline 성립 (파일럿 "격리" confound 정정)

## 한 줄 상태 · 다음 첫 액션
- **상태:** S17 에이전트 간 메시지 기능이 **설계 전환 + happy-path 첫 슬라이스 완료·커밋·push**(`3ca4cf8`, origin/master). 이전 세션 파일럿의 "정책 격리=기능 막힘"은 **confound**(인젝션-모양 프로브 + 수신 프라이밍 부재)로 정정됨 — **프라이밍 깐 에이전트가 자연 메시지를 정상 수용**함이 실측으로 성립(baseline). 사다리를 **happy-path-first**로 재편(ADR-0091/0092).
- **다음 첫 액션:** **답장 왕복 슬라이스**(A→B→A) — 지금은 A→B 도착·수용만 검증됨. 받은 B가 발신 경로(engram-send/MCP)로 A에게 답하는 왕복을 붙인다. 그 다음 = **형식 비교 스파이크**(봉투 후보 몇 개, 판정 수용>토큰) → 그 뒤에 포화 등 비정상.

## 무엇을 했나 (이 세션 후반)
- **설계 전환(사용자 주도):** 파일럿 결론 반박 — 보통 orchestration은 정상 동작(스폰 과제=신뢰 채널)인데, 파일럿은 mid-stream 제3자 인젝션-모양 프로브를 **프라이밍 없는** 에이전트에 던진 것. 즉 기능이 아니라 **수신 절반(수신 계약) 미설계**가 문제. → **happy-path-first**(정상 baseline 먼저, 비정상은 크면 후순위·작으면 병행). 이스케이프(위조 방어)는 Stage 4 유보(우선순위 낮음). ADR-0091(사다리 재배치)·**ADR-0092**(수신 계약·프라이밍 아키텍처·원인 정정) 신규.
- **프라이밍 아키텍처(사용자 결정):** 외부 MD(`prompts/agent-priming.md`, 초안 — **문구 다듬기 사용자 대기**)를 스폰 시 `--append-system-prompt-file`로 시스템프롬프트 주입. **코드엔 경로만**(하드코딩 0), 로더는 seam이라 미래 프롬프트-인젝션/스킬등록 시스템으로 교체. 임시판 전원 주입. 토폴로지 = 그룹챗+메일(1:1·다중·단체) 최종, **1:1 먼저·다중수신 추상** 위에.
- **첫 슬라이스 구현(/implement standard):** `daemon/src/control/priming.rs`(PrimingProvider seam·`discovery::find_install_root` exe-walk-up 경로해석·absolute-or-None·cmd-metachar/non-UTF8 가드) + 스폰 배선(daemon→core `backend/claude.rs`, 코어 격리 유지=경로를 데이터로만) + 다중수신 seam(`ingress.rs` resolve→수신자 목록, 1:1=길이1 동일) + happy-path 스모크 bin(`priming_smoke.rs`, test-harness) + discovery `find_install_root`.

## ★핵심 결과 — baseline 성립★
프라이밍 깐 실 claude(sonnet)에 자연 1:1 메시지("모듈 auth 리뷰 끝, 로그인 경로 이슈 2개") 배달 → **인젝션 취급 없이 팀원 메시지로 수용·협업 반응**("이슈 2개 공유해 주시면 확인하겠습니다"). 신뢰 채널(시스템프롬프트) 프라이밍이 인젝션 반사를 해소함 — 파일럿 confound 확증, 기능 방향 유효.

## 검증 상태 (쌍)
- **한 것(전부 PASS):** `/review code full` 2인 적대(doc-aware reviewer-deep + Codex blind) — R1 doc-aware FIX(1)+Codex BLOCK(5: base-dir cwd·cmd.exe metachar·absolute계약·TOCTOU·non-UTF8) → 하드닝 fix → R2 Codex diff-집중 **PASS**(5 CLOSED·회귀0). `/qa`: 워크스페이스 빌드·daemon 153 lib+44 통합·core 214·discovery 45·fmt·코어격리0. **재실행:** `cargo test -p engram-dashboard-daemon`(+ `--features test-harness`로 priming_smoke 빌드) · `-p engram-dashboard-core` · `-p engram-dashboard-discovery`.
- **안 한 것(정직):** ① **baseline은 1샘플·1모델(sonnet)** — 통계 아님, 다른 모델/문구/긴 컨텍스트 미검증(happy-path-first라 의도적 후순위). ② **답장 왕복 미구현**(다음 슬라이스). ③ **형식 비교 스파이크 미착수**. ④ **프로덕션(WMI-spawn) 프라이밍 실적용**은 exe-walk-up로 코드상 해결됐으나 **패키징/설치 레이아웃 미검증**(dev는 정상). ⑤ **R2 재리뷰는 Codex(cross-family) diff-집중만** — doc-aware 불변식은 R1서 확인됐고 하드닝이 안 건드려 full 2-family 재실행은 생략(기계적 수정·비례 판단). ⑥ 조직 계정에서 수용됐으므로 개인 계정 실측 불요해짐(프라이밍이 조직 반사도 덮음).

## do-not (누적)
- 루트 bare `cargo test` 금지(멤버별 `-p`). 릴리즈 `--all-targets` 금지(test-harness 유니피케이션).
- **프라이밍 내용 코드 하드코딩 금지**(ADR-0092) — 항상 외부 MD, 로더는 seam.
- **배달 경로 = 수신자 목록 추상 유지**(1:1도 길이1) — 단일수신자로 좁히면 그룹 회귀.
- **happy-path baseline 통과 전 비정상(포화·인젝션) 본격 착수 금지** — 성공 모드 없이 실패 모드부터 파지 말 것(이번 세션 교훈, 사용자 지적).
- **회수 측정 프로브를 "기억했다가 보고해" 인젝션-모양으로 만들지 말 것** — 파일럿 confound 원인. 자연 메시지로.
- epoch pinning 금지 · 배달 시맨틱 변경 금지 · `/compact` 유저텍스트로 native 압축 기대 금지.
- **cmd.exe escaping 전반은 선재 이슈**(`console_command`가 모든 path arg를 `cmd /c`로 감쌈, `--mcp-config` 공유) — 이번 슬라이스는 프라이밍 경로에 가드만 걸고 근본 수정은 **별도 추적**(광범위·전 백엔드 영향).

## 정지 조건
- 리뷰어 정면 대립(FIX vs BLOCK)은 자율모드라도 사용자 에스컬레이션 — 이번엔 Codex가 상위집합(실 버그)이라 증거기반 하드닝으로 수렴.
- 굵은 설계(수신 프라이밍 정식화·형식 확정·다중수신 정책)는 사용자 결정 — 임의 확정 금지. **코드 구현 착수 전 사용자에게 조감도 승인**(이 세션 사용자 규칙).
