## 무엇을 했나 (요지)
- **①② `crates/engram-dashboard-core/tests/stdio_physical_pipe.rs`(신규, test-only):** 실 StdioTransport+실 pipe. 무인터리브 = 느린 reader 배압 + 4스레드 동시 send_input + run-length 오라클(`stdin.lock()` 회귀망 — mutation 실험으로 비공허 실증: lock 약화 시 6-run 실패). partial-write = 자식이 정확 64KiB 소비·echo 후 사망 → prefix 물리 배달 실증 + Err 표면화. payload = 시드 고정 xorshift64 + **전 시프트(832만) 무앨리어싱 런타임 기계검증(~90ms)** — wrong-offset write 회귀도 등식 오라클에 걸림(리뷰 대립의 사용자 승인 해소).
- **③ 프로덕션 4파일 + 오라클:** `WriteOutcome.epoch`(착지 세션 by-construction) · `DeliveryObservation.to_epoch`(성공=착지 epoch/실패=None="완결 write 부재") · `ControlRegistry.mid_send_hook`(`#[cfg(feature="test-harness")]` yield seam, resolve↔write 간극 최후 지점 발화, 운영 미컴파일 — self-dev-dep) · 신규 오라클(mid-flight 교체 → to_epoch==Some(1)·신 버퍼만 착지) + 오라클 4/5 강화. **배달 시맨틱 불변**(epoch pinning 없음). 근거·거부 대안 = **ADR-0089**.
