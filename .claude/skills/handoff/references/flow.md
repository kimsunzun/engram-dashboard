# Handoff — 실행 절차

`$ARGUMENTS` = `[save|load]`. 오퍼레이션만(생략 시 load). 그 외 인자는 무시하되 무시했음을 한 줄 보고한다(조용한 흡수 금지).

## 0. 인자 파싱·기본값

**경로 SSOT (baked 기본값 — 프로젝트가 바인딩으로 override 가능):** save·load가 같은 값을 쓴다. 프롬프트·스크립트에 경로를 중복 하드코딩하지 않는다. 프로젝트 override는 소비처 프로젝트 트리 `.claude/skill-bindings/handoff.md`(cwd-상대 Read — ADR-0004, 없으면 아래 기본값).

- `handoffRoot` = 프로젝트 cwd 기준 `.claude/handoff`
- `latestFile` = `<handoffRoot>/latest.md`
- `historyDir` = `<handoffRoot>/history`

**파싱:**
- `op` — 첫 토큰이 `save`/`저장` → save. `load`/`로드` 또는 생략 → load. 그 외 인자는 무시 + 무시 사실 한 줄 보고.
- 파싱 후 "save인지 load인지" 한 줄 명시.

## 1. op = load

- `latestFile`을 **Read 툴로 읽는다** — 로드의 정본은 Read다. (어시스턴트가 출력한 `@경로`는 자동 확장되지 않는다 — 실측 2026-07-07, `@<latestFile>` 출력 방식은 폐기.)
  - 있으면 → 첫 H1 제목 + 수정시각을 **확인 1줄**(`로드: [시각] <제목>`)로 출력하고, 본문의 핵심(한 줄 상태·다음 첫 액션·주의점)을 요약 보고한 뒤 그대로 이어간다.
  - 없으면 → "핸드오프 없음 (먼저 `/handoff save`)" 보고 후 종료.
- **"최신 찾기" 없음** — 늘 `latestFile` 고정으로 읽는다(결정적). 목록·mtime 추정을 하지 않는다.

## 2. op = save

### 2-0. 세션 마무리 먼저 훑기

save = 세션을 닫는 신호다 — 본문을 쓰기 전에 미룬 마무리를 훑는다.

- **남길 것 보존 = 강제.** 날아가면 안 되는 것(작성하다 만 산출물·미커밋 변경·남겨야 할 기록)을 먼저 영속화한다 — 파일로 쓰거나, 게이트 통과분은 커밋하거나, **당장 못 하면 최소한 핸드오프에 "미커밋/미저장" 상태로 명시**한다.
- **정리 = 필요한 것만(린).** 임시 파일·미결 정리는 실제로 필요한 것만 — 없는 일을 만들지 않는다.

### 2-1. 본문 작성 → 기록 → 편의 사본

순서대로:

1. 본문을 작성한다(§2-2). 첫 줄 = H1 제목(`# 핸드오프: …` — load의 확인 1줄이 이 제목을 쓴다).
2. **영구 기록:** `<historyDir>/<YYYYMMDD-HHMMSS>-<슬러그>.md`에 **새 파일**로 쓴다. 기존 history 파일은 **절대 덮거나 지우지 않는다**.
   - 슬러그는 Claude가 H1/주제에서 자동 도출: 짧게, 공백→`-`, **basename만**(하위 디렉토리 생성 금지), 경로구분자·상위경로(`..`)·금지문자 제거. 같은 초 충돌 시 `-2`,`-3` suffix.
3. **편의 사본 갱신:** 같은 내용을 `<latestFile>`에 쓴다(덮어씀 — load가 읽을 고정 파일).
4. "영구 기록 경로 + latest 갱신" 한 줄 보고.

### 2-2. 내용 — 권장 체크리스트 (비강제·린하게)

다음 세션이 그것만 읽고 이어갈 수 있게. 구조는 자유지만 **빠지면 막히는 것**을 환기하고, 불필요한 항목은 넣지 않는다.

- **한 줄 상태 · 다음 첫 액션** — 즉시 시작점.
- **변경/완료 + repo 상태** — 현재 브랜치 · 미커밋·건드린 파일(로컬-온리 맥락 유실 방지).
- **검증 상태(쌍으로)** — 돌린 것(테스트·빌드 + 결과 + **재실행 명령**) ↔ **검증 안 된 항목**(안 돌린 걸 "된 것"처럼 적지 않기).
- **실패한 접근(do-not)** — 다시 시도할 법한 것만(뻔한 dead-end는 생략).
- **정지 조건(stop conditions)** — 다음 세션이 어떤 상태·조건을 만나면 **진행을 멈추고 사용자에게 물어야** 하는지(있을 때만 — 없으면 생략. 외부 선례 차용 2026-07-07).
- 블로커/미결 결정 · (비자명하면) 완료 기준·핵심 가정 · 참조 경로(**읽어야 할 파일만**, 경로 덤프 금지 — 코드를 가리키면 `파일:라인` 포인터 권장).

> AI가 신뢰성 있게 빠뜨리는 필수 = **검증 안 된 항목 · 실패한 접근**. repo 상태·재실행 명령도 자주 누락된다 — 없으면 넣도록 환기한다.

## 3. 완료 보고

- load: 로드한 핸드오프(제목·시각) 또는 "없음".
- save: 영구 기록 경로 + `latest.md` 갱신 확인.
- **피드백 한 줄(의무):** 최종 보고에 "피드백: 없음" 또는 이번 실행에서 발견한 스킬 자체 결함·개선점 한 줄. 발견분만 그 스킬 feedback.md에 누적한다.

## 가드레일

- **history/ append-only** — `historyDir`의 파일은 절대 덮거나 지우지 않는다(기록 보존이 1순위). **단 `latestFile`은 매 save마다 덮어쓴다** — 설계상 편의 사본이라 의도된 동작이다.
- **load는 `latestFile` 고정** — "최신 찾기"·목록·mtime 추정을 하지 않는다.
- **슬러그는 basename만** — 경로구분자·상위경로(`..`)·금지문자를 허용하지 않는다(엉뚱한 위치 생성 차단).
- **환경 가정 주입 금지** — 특정 스택·터미널·세션 매니저 개념을 골격에 박지 않는다. 경로 등 환경 의존은 소비처 프로젝트 트리 `.claude/skill-bindings/handoff.md`(ADR-0004). 멀티스트림·자동선택은 현재 비목표(나중에 바인딩으로 확장).
