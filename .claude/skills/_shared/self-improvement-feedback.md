# 자기개선 피드백 (스킬 공용 규약)

review·qa·research·adr 등 스킬이 공유하는 자기개선 루프의 **단일 출처**다. (구 — 스킬마다 복붙하던 절을 여기로 추출. 각 SKILL.md는 한 줄 포인터만 둔다.)

스킬을 *쓰다가* 결함·개선점을 발견하면:
- **그 자리서 고치지 않는다** — 사용 중 오판으로 멀쩡한 스킬을 되레 망칠 수 있다.
- **작업이 끝나면 즉시** 그 스킬 폴더의 `feedback.md`에 한 줄 기록한다(미루지 않되 작업 흐름은 끊지 않는다). 누적 — 덮어쓰지 않는다.
- **최종 보고에 피드백 한 줄 = 의무** — 발견이 없어도 "피드백: 없음"을 보고한다(각 스킬 flow 최종 보고 절이 실행 훅). 파일에는 발견이 있을 때만 쓴다 — 보고 의무가 조용한 스킵을 관측 가능하게 한다(2026-07-07 사용자 결정).
- 실제 반영은 **관련 주제가 다시 나올 때 사용자 승인 하에** 한다.

`feedback.md`는 **각 스킬 폴더 안**에 둔다(스킬별 피드백 분리). 이 파일은 그 *규약*만 정의한다 — 실제 피드백 항목은 여기 쌓지 않는다.

## 기록 양식 (권장 — 신규 항목부터)

| 날짜 | 발견 | 상태 |
|---|---|---|

- 상태 어휘: **기록**(실측·정보 — 반영할 것 없음) · **미반영**(개선 후보 — 수확 대상) · **반영 (날짜)** · **기각 (사유)**. 애매하면 **미반영**. 수확 = 미반영 행 필터(줄글 구간은 종전대로 통독).
- **append-only — 반영·기각돼도 행을 지우지 않는다.** 날짜·발견 열은 불변, 상태 셀만 전이한다. 지우면 같은 제안이 재등장할 때 이력이 없어 재검토하게 된다. 파일이 비대해지면 반영 완료분을 하단 아카이브 절로 접는다(사용자 승인 하에).
- 전 스킬 feedback.md는 `## 이력` 표 하나로 일괄 개편 완료(2026-07-07 사용자 지시 — 검증 상태 등 prose 절은 보존). 신규 항목은 그 표에 한 행으로 추가한다.

## 학습용 사용 로그 (권장 — 강제 아님)

사용자 학습용 보조 장치다. **권장이되 강제는 아니다** — 여유될 때 남기고, 작업 흐름·산출 품질을 해치면서까지 적지 않는다. 안 남겼다고 규약 위반 아님.

스킬을 한 번 실행하고 나면 그 스킬 폴더의 `usage-log.md`에 한 줄(권장):
- **언제/무엇에**(대상·단계·강도) · **어떻게 흘렀나**(핵심 선택 한둘) · **결과**(PASS/FIX/BLOCK 등). 누적 — 덮어쓰지 않는다.
- 목적 = 사용자가 "이 스킬이 실제로 어떻게 쓰이는지"를 사후에 학습. 결함 로그가 아니다(그건 `feedback.md`) — *정상 사용 기록*이다, 둘을 섞지 않는다.
