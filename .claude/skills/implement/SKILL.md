# Implement

비자명 코드 변경을 **코더 서브에이전트 스폰 → `/review code` 게이트 → `/qa` 게이트 → 커밋 준비**로 한 호출에 오케스트레이션한다. 존재 이유는 **메인이 직접 구현하는 것을 구조로 차단**하는 것 — 위임 우선, 생산자≠리뷰어, 게이트 순서·모델 배정·루프 상한 표준화.

**실행 전 `references/flow.md`를 반드시 Read 한다 — 안 읽고 코더 스폰 금지.** 전체 절차·강도표·루프 상한·가드레일이 거기 있다. `$ARGUMENTS` = 과업 명세(+선택적 강도). 없거나 목표·완료 기준이 모호하면 1~2개만 되묻는다.

## 핵심 설계 (불변)

- **메인은 오케스트레이터 — 구현을 직접 짜지 않는다.** 라우팅·지시서·취합·보고만 메인 토큰이 쓴다. 실제 편집은 fresh 코더 서브에이전트가 한다. 자율 모드에서도 생략 금지.
- **생산자 ≠ 리뷰어 (fresh 분리).** 코더가 짠 산출물은 짠 코더가 아니라 `/review code`가 때리고(standard/critical = cross-family 포함 full/deep · **simple = 단일 family light** — cross-family 아님), `/qa`(기계적 게이트)가 실측한다.
- **게이트 순서 불변 — review → qa.** review("맞나", 적대 판단)가 먼저, qa("실제로 도나", 기계 게이트)가 후행. 역전·생략 금지 — qa는 review의 후행 짝이다(두 게이트 스킬의 계약).
- **강도 = 라우팅(1축).** simple/standard/critical이 코더 슬롯과 두 게이트 강도를 한 번에 정한다. 정량은 `references/flow.md` 강도표가 정본 — 아래 표는 "언제"만.
- **루프 상한 = 무의미 재시도 방어.** 직전 findings를 전부 닫으면 카운터 리셋, 못 닫으면 +1 — 연속 2회 미해소 또는 총 재수정 5회 초과 시 사용자 에스컬레이션. 정량·리셋 정책은 `references/flow.md §3`.
- **커밋 준비까지만 — 커밋은 호출자 몫.** 게이트 통과 후 커밋 메시지 초안 + 스테이징 대상만 메인에 반환한다. 호출자 처리 = mode-aware: **대화 모드 = 사용자에게 확인 / 자율 모드 = 게이트 통과분 커밋 + 내역 명시 보고**(미커밋 이월이 변경 뒤섞임을 만든다). 커밋 실행·기록은 호출자와 프로젝트 관례가 한다 — 스킬은 검증만, 영속화는 밖. 상세 = `references/flow.md §5`.

## 강도 = 라우팅 (핵심)

**정량·라우팅 서술은 `references/flow.md` 강도표가 정본** — 아래 표는 키워드만.

| 강도 | 키워드 |
|---|---|
| **simple** | 국소·저위험 |
| **standard**(기본) | 비자명 일반 변경 |
| **critical** | 동시성·kill·lifetime·보안·공개 API·마이그레이션 |

- **escalation-only:** 코더·리뷰 도중 위험 트리거(동시성·kill·lifetime·보안·공개 API·마이그레이션)를 발견하면 상위 강도로 자동 승격 + 보고. 시작 강도가 하한 — 임의 하향 금지.
- **이 스킬 대상이 아닌 것:** 1~2줄 사소 수정·사소한 문서(오타·서식)·조사/탐색성·스파이크는 인라인 예외 부류다. `references/flow.md §0`에서 "스킬 불필요, 인라인 + QA" 안내 후 종료한다.

## 트리거

`/implement "<과업>" [simple|standard|critical]`. 강도는 옵션 — 미지정이면 위험 영역·범위로 추정하되 **기본 standard**. 파싱·추정은 `references/flow.md` 상단 파싱 절. 호출 시 **"어느 강도·어느 코더 슬롯으로 도는지" 한 줄을 사용자에게 명시**한다.

## 프로젝트 통합 (골격/바인딩 분리)

이 스킬은 **범용 구현 오케스트레이터**다. **자체 바인딩 파일을 만들지 않는다** — 게이트 스킬 바인딩이 이미 정본이라 중복 SSOT가 된다:
- **빌드·테스트 실명령** = `/qa` 스킬 바인딩(소비처 프로젝트 트리 `.claude/skill-bindings/qa.md` — ADR-0004)이 정본. 재수록하지 않고 `/qa`에 위임한다.
- **코드 불변식·전용 체크리스트** = `/review code` 스킬 바인딩이 정본. 코더 지시서의 "제약" 컨텍스트는 프로젝트 통합 지점(CLAUDE.md·ADR)에서 추려 넣는다.
- **커밋·결정 기록(ADR/step-log)** = 프로젝트 관례. 이 스킬은 커밋 준비물만 반환한다.

TDD 구체 명령·모듈 격리 하네스 형태도 프로젝트 바인딩(qa/review)에 위임한다 — 이 스킬은 "테스트 먼저 또는 함께"라는 골격만 코더 지시서에 박는다.
