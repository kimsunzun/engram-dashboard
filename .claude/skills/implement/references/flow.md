# Implement — 실행 절차

`$ARGUMENTS` = 과업 명세 [+ 강도 `simple`|`standard`|`critical`]. **파싱:** 마지막 토큰이 `simple`|`standard`|`critical` 중 하나면 강도, 아니면 전체가 과업 명세다(강도는 강도표로 추정). 목표·완료 기준이 모호하면 1~2개만 되묻는다. 강도 미지정이면 **강도표(아래)로 추정**하되 standard 기본 — 어떤 강도·어느 코더 슬롯으로 도는지 호출 시 한 줄 명시한다.

- **불변 — 스폰 시 모델 명시 강제:** 『코더』 서브에이전트는 `model`을 전역 사전 슬롯대로 명시해 스폰한다. 미명시 스폰 금지 — 조용한 다운그레이드를 막는다.
- **fresh 분리 — 생산자 ≠ 리뷰어:** 코더 슬롯은 리뷰어·메인 오케스트레이터와 다른 fresh 인스턴스다 — 짠 사람이 자기 코드를 통과시키지 않는다. 리뷰어는 `/review`가 소환한다(standard↑ = 다른 family 포함, simple = 단일 family light). `/qa`는 모델 없는 기계 게이트다(family 개념 무관).
- **코더는 서브에이전트를 못 낳는다:** 재귀 폭발 방어 — 코더 지시서에 "서브에이전트 스폰 금지"를 박는다. 팬아웃이 필요하면 메인이 코더를 분할 스폰한다.

## 강도표 (tier별 정량 정본)

강도는 **코더 슬롯 + 두 게이트 범위**를 한 번에 정한다. **정량(코더 슬롯·`/review code` 강도·`/qa` 강도)은 이 표가 유일 정본** — 다른 절은 tier 이름으로 참조만 하고 값을 다시 적지 않는다(rot 방지).

| tier | 언제 | 코더 슬롯 | `/review code` | `/qa` |
|---|---|---|---|---|
| **simple** | 국소·단일 관심사·저위험(위험 영역 사전 배제) | 코더(단순) | `light` | `quick` |
| **standard**(기본) | 비자명 일반 변경 | 코더(복잡) | `full` | `standard` |
| **critical** | 동시성·kill·lifetime·보안·공개 API·마이그레이션 | 코더(복잡) | `deep` | `full` |

- `/review code`·`/qa`의 강도 인자 값은 각 게이트 스킬의 강도 축이다(review = self/light/full/deep · qa = quick/standard/full). 이 표는 tier→게이트 강도 매핑만 고정하고, 게이트 내부 정량은 그쪽 flow가 정본.
- **escalation-only(하향 금지):** tier 선택 트리거 = 위험 영역(동시성·kill·lifetime·보안·공개 API·마이그레이션)·LOC·신규/리팩터·불확실성. 클수록 위 tier. 애매하면 standard. **도중 위험 트리거 발견 시 승격만** — 시작 강도가 하한(§1·§3 참조).
- **승격 후 재실행:** 강도 승격 시 이미 통과한 게이트도 새 강도로 재실행한다(코더 산출은 유지, 검증만 재실행 — 하위 강도 통과가 상위 강도 통과를 함의하지 않는다).
- **게이트 강도는 하한 계약:** 게이트 스킬(`/review`·`/qa`)은 내부 위험 트리거로 self-escalation할 수 있다 — implement가 넘긴 강도는 **하한**이며, 게이트가 승격하면 그 결과 강도를 종결 보고에 반영한다.
- **critical은 review/qa 둘 다 최상위** — 되돌리기 비싼 변경은 적대 리뷰(deep)와 실측(full GUI/실제 동작)을 함께 요구한다.

## 0. 전제 — 과업 명세 확보 + 스킬 적용 여부

- **명세 확보:** 목표·완료 기준(수용 기준)이 모호하면 1~2개만 되묻는다. 명세가 서면 그대로, 아니면 코더에 넘길 만큼 구체화한 뒤 진행한다.
- **스킬 불필요 판정 (인라인 예외 부류):** 과업이 **1~2줄 사소 수정 · 사소한 문서(오타·서식·노트) · 조사/탐색성 · 스파이크(throwaway)** 면 이 스킬 대상이 아니다. **이 스킬을 종료하고 호출자가 인라인 처리(+ `/qa` quick 이상)한다**고 안내한다 — 스킬 내부에서 직접 구현하지 않는다(오케스트레이션 오버헤드가 이득을 넘는다). load-bearing 문서(standing 규약·표준)는 예외 아님 → `/review doc`으로 안내.
- **컨텍스트 준비:** 코더 지시서에 넣을 재료를 추린다 — 관련 ADR · 불변식 · 건드리지 말 것 · 관련 기존 테스트. 정본은 프로젝트 통합 지점(CLAUDE.md·ADR·`/review code` 바인딩)에서 꺼낸다(스킬이 하드코딩하지 않는다). 이 추림은 메인이 하되, 광범위 스윕이 필요하면 조사 서브에이전트에 위임하고 결론만 회수한다(위임 우선). **메인 금지는 *구현 편집*이다** — 컨텍스트 수집(파일 읽기·분석)·지시서 작성·취합은 메인의 일이다.

## 1. 라우팅 확정 (Lead = 메인 오케스트레이터)

- **강도표 + 위험 트리거로 tier 확정.** simple/standard/critical 중 하나. 위험 영역이 하나라도 걸리면 critical.
- **한 줄 명시:** 어떤 강도·어느 코더 슬롯인지 사용자에게 명시한다(예: "critical 강도 / 코더(복잡) 슬롯 — review deep → qa full").
- escalation-only — 여기서 정한 강도가 하한이다. 이후 어느 단계서든 위험 발견 시 승격만 하고 낮추지 않는다.

## 2. 코더 스폰 (fresh · 모델 명시 · 결론만 회수)

- **Agent 도구로 fresh 코더 1명 스폰**(`subagent_type: general-purpose`, `model` = 전역 사전 코더 슬롯 — simple=『코더(단순)』, standard/critical=『코더(복잡)』). 모델 미명시 스폰 금지(위 불변 규칙).
- **지시서 최소 골격:**
  - **역할** — "너는 코더 서브에이전트다. 이 과업만 구현한다."
  - **목표 · 완료 기준** — §0에서 확보한 수용 기준.
  - **제약** — 관련 ADR · 불변식 · 건드리지 말 것(정본은 프로젝트 통합에서 추린 컨텍스트).
  - **TDD** — 테스트를 먼저 또는 함께 작성한다. 새 모듈·외부 의존이 걸린 코드는 seam으로 끊어 **단독 실행 가능한 격리 하네스를 함께 만든다**(생성 의무 — 게이트는 검사만 한다). **단, 프로젝트에 테스트 기반이 없거나 자동 테스트가 구조적으로 안 맞는 영역(에디터·실기기·체감 의존 등)엔 억지 테스트를 만들지 않는다** — 대신 수동 검증 절차(무엇을 어디서 어떻게 확인하나)를 반환물에 작성하고, 게이트는 그것을 검증물로 검사한다. 판정·형태는 프로젝트 바인딩/ADR(= `/qa`·`/review code` 정본)이 있으면 그걸 따른다.
  - **반환 형식** — 변경 요약 + 파일 목록 + 자체 검증 결과만. verbose 도구 출력 덤프 금지(메인 컨텍스트 오염 방지).
  - **금지** — 서브에이전트 스폰 금지(재귀 폭발) · 커밋 금지(커밋 준비는 §5, 실행은 호출자).
- 산출은 결론만 회수한다 — 원시 편집 로그·긴 도구 출력을 메인 컨텍스트에 넣지 않는다.

## 3. 게이트 1 — `/review code` (강도표대로)

- 코더 산출을 **`/review code <강도>`** 로 넘긴다(강도 = 강도표 매핑: simple→light · standard→full · critical→deep). review가 다른 family 적대 리뷰로 "맞나"를 본다.
- **게이트 최소 payload:** 변경 파일 목록 + 코더 변경 요약 + 과업의 완료 기준을 넘긴다(+ §0에서 추린 관련 ADR·불변식은 doc-aware 렌즈용).
- **판정 처리:**
  - **PASS** → §4로.
  - **FIX** → findings를 코더에 반영 재지시(같은 지시서 골격에 FIX 항목 추가). 반영 후 `/review code` 재실행.
  - **BLOCK** → 즉시 사용자 에스컬레이션(임의 진행 금지).
- **재지시 방식:** 코더 세션을 이어갈 수 있는 환경(SendMessage 등)이면 이어가고, 불가하면 **새 fresh 코더를 스폰하되 이전 산출(변경 요약·파일 목록) + FIX findings를 컨텍스트로 준다.**
- **루프 카운터 = 코더 재수정 총 횟수 (무한 리뷰 방어):** review FIX 반영이든 qa 실패 수정이든 **합산**해 센다. 상한 2회. **2회로도 못 닫히면 임의로 계속하지 않고 사용자에게 에스컬레이션**한다(남은 findings + 왜 안 닫히는지 요약). 무한 루프·토큰 폭주 방어.
  - **강도 승격 시 카운터 리셋 + 전 강도 합산 하드캡 3회** — 승격은 검증 대상이 바뀐 것이라 리셋이 정당하고, 하드캡이 남용을 막는다. (비준 2026-07-03)
- 리뷰 도중 위험 트리거 발견 → 강도 승격 + 보고(escalation-only, §1). **승격 시 이미 통과한 게이트도 새 강도로 재실행**(코더 산출 유지, 검증만 재실행 — 강도표 각주).

## 4. 게이트 2 — `/qa` (강도표대로 · review 후행)

- **review PASS 후에만** `/qa <강도>`를 돈다(강도 = 강도표 매핑: simple→quick · standard→standard · critical→full). 순서 역전·생략 금지 — qa는 review의 후행 짝이다(qa 스킬 계약).
- **게이트 최소 payload:** review와 동일 — 변경 파일 목록 + 코더 변경 요약 + 과업의 완료 기준.
- qa가 빌드·테스트·(full이면) GUI/실제 동작 실측으로 "실제로 도나"를 본다. 실명령은 qa 바인딩 정본.
- **바인딩 없는 프로젝트:** 메인이 등가 명령(테스트·빌드)을 도출해 실행하고 "바인딩 부재 — 등가 실행"을 결과에 명시한다(조용한 생략 금지).
- **qa 실패:** 코더에 수정 재지시(§3 재지시 방식) → §3 review부터 다시(코드가 바뀌면 review를 재통과해야 한다 — review를 건너뛰고 qa만 재실행 금지). 이 재수정도 §3 루프 카운터(코더 재수정 총 횟수)에 합산해 무한 왕복을 막는다.
- qa PASS → §5로.

## 5. 종결 — 커밋 준비까지만 (커밋·기록은 호출자)

- **반환물 = 커밋 메시지 초안 + 스테이징 대상 목록**을 메인에 반환한다. 커밋 실행·ADR·step-log 기록은 하지 않는다(review/qa와 같은 경계 — 스킬은 검증만, 영속화는 프로젝트 관례).
- **mode-aware (호출자 행동 지침 — 모드 = 자율 vs 대화):** 아래는 스킬이 반환물을 넘긴 뒤 **호출자(메인)가 어떻게 처리할지**의 지침이다(스킬 자신은 커밋하지 않는다).
  - **대화(interactive) 모드:** 커밋 전 사용자 확인 — "게이트 전부 통과. 커밋할까요?" + 메시지 초안·대상 제시.
  - **자율("진행 쭉해") 모드:** 게이트(review PASS + qa PASS)가 전부 통과했을 때만 커밋하고, **커밋 내역(메시지·대상 파일·통과한 게이트 강도)을 명시 보고**한다(조용히 커밋 X). 미커밋인 채 다음 작업으로 넘어가면 변경이 뒤섞이므로 **게이트 통과분을 미커밋으로 이월하지 않는다.** 게이트 미통과분은 커밋하지 않고 사용자에게 남긴다.
- 커밋 메시지 끝 Co-Authored-By 트레일러 등 프로젝트 커밋 관례는 호출자가 적용한다.
- **자율 모드 교착 해소:** review 불일치·BLOCK·루프 상한 초과는 **자율 모드라도 반드시 사용자 에스컬레이션**한다(자율 ≠ 사람 백스톱 생략).
- **실패 종료 상태 (BLOCK · 루프 초과 · qa 실패로 종료):** 워킹트리 변경은 유지하되 **known-bad로 명시**하고, diff 요약 + 실패 지점 + 남은 findings를 반환한다. 되돌림(revert)은 **사용자 확인 없이 하지 않는다.**

## 가드레일 (앞 절에 없는 금지만)

앞 절에 이미 박힌 규약(fresh 분리·모델 명시 강제·코더 서브에이전트 금지·결론만 회수·루프 상한 2회·escalation-only)은 여기서 반복하지 않는다. 다른 곳에 안 적힌 금지만:

- **메인 직접 구현 금지** — 이 스킬의 존재 이유다. 메인이 편집을 직접 짜면 스킬을 부른 의미가 없다(위임 우선 위반). 자율 모드에서도 동일.
- **게이트 순서 역전·생략 금지** — 반드시 review → qa. 코더 산출을 리뷰 없이 커밋 준비로 넘기지 않는다. review self 강도로 낮춰 리뷰어 렌즈를 끄는 것도 이 스킬 대상(비자명 변경)에선 금지 — 최소 light.
- **바인딩 재수록 금지** — 빌드·테스트 실명령(qa 바인딩)·코드 불변식(review 바인딩)을 이 스킬 문서에 베끼지 않는다(중복 SSOT = rot). 이름으로 참조만 한다.

**비목표(v1):** 병렬 과업 fan-out은 v1 비목표 — 독립 과업은 쪼개서 별도 호출한다.
