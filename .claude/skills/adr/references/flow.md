# ADR — 실행 절차 (범용 골격)

`$ARGUMENTS` = 오퍼레이션 `new`|`supersede`|`lint`|`index` (옵션). 미지정이면 요청으로 추정한다. 어느 오퍼레이션을 도는지 호출 시 사용자에게 한 줄 명시한다.

이 파일은 **스택을 모르는 범용 골격**이다 — 오퍼레이션별 "판단 → 스크립트 호출 → prose → 보고" 흐름과 불변식만 담는다. 실제 스크립트 명령·ADR 폴더·파일명·템플릿·인덱스 위치·상태 범례·앵커 명령 같은 **프로젝트 실체**는 바인딩이 채운다.

> **바인딩(1회 선언 — cwd-상대 Read · ADR-0004):** 서기 스크립트는 스킬 폴더 안 `scripts/adr.mjs`(자족 — 골격과 함께 이동)다. 프로젝트 전용 실값(실명령·경로·플래그·템플릿·인덱스 형식·상태 어휘·앵커 명령)은 **소비처 프로젝트 트리**의 `.claude/skill-bindings/adr.md`가 정본이다. 실행 착수 시 이 파일을 **현재 프로젝트 루트 기준 상대경로 `.claude/skill-bindings/adr.md`로 Read**해 실값을 꺼낸다 — 런타임 cwd(=claude를 띄운 프로젝트)로 절대경로를 조립해 Read하고, 골격에는 절대경로를 박지 않는다. 바인딩 파일이 없으면 실행하지 말고 "이 프로젝트에 adr 바인딩(`.claude/skill-bindings/adr.md`)이 없다"고 보고한다. 아래 절이 "스크립트"·"ADR 폴더"·"템플릿"·"인덱스"라 부르는 자리는 그 바인딩에서 실명령(`<skill>/scripts/adr.mjs` + 플래그)·실체를 꺼낸다. **`<skill>` = 이 스킬이 로드된 base dir**(스킬 호출 시 하네스가 "Base directory for this skill"로 제공 — 다중 사본이 있어도 그 값을 쓰고 추정하지 않는다).

## 역할 분담 — 결정적 스크립트 + 얇은 판단

- **스크립트(결정적)** — 채번(최대+1·쓰기 직전 재스캔) · 파일 스캐폴드 · 인덱스 재생성 · supersede 양방향 링크 · 형식 lint.
- **스킬(LLM)** — ① 입력 수령·검증 ② 전체/부분 폐기 판단 ③ 본문 prose(맥락·결정·거부한 대안·근거·영향) ④ 보고.

## 0. 인자 파싱·기본값 (오퍼레이션 추정)

소문자만 허용한다.

- **정확히 `new`|`supersede`|`lint`|`index` 하나** → 해당 오퍼레이션.
- **인자 없음** → 요청으로 추정: 새 결정 박제=`new` / 기존 결정 번복·"이거 폐기"·"바뀜"=`supersede` / 정합성 확인·인덱스 맞나=`lint` / "인덱스만 다시 만들어"=`index`. 요청도 오퍼레이션도 없는 완전-빈 호출 → 안전 기본값 `lint`(read-only).
- **알 수 없는 토큰·복수 인자·오타·대문자** → **실행하지 말고** 받은 인자와 사용법(`/adr [new|supersede|lint|index]`)을 사용자에게 보고한다.
- **new ↔ supersede 애매**(기존 결정을 *고치는* 건지 *대체*하는 건지 모호) → 사용자에게 한 줄 확인한다. supersede는 옛 결정에 손대므로 임의 추정하지 않는다.

## 1. 입력 수령·검증

스킬은 결정을 *만들지 않는다*. 호출자(메인/사용자)가 다음을 제공해야 한다:

- **new** — 제목 · 맥락 · 결정 · **거부한 대안 + 각 이유** · 근거 · 영향/불변식. 거부한 대안이 비면 되묻는다 — ADR의 핵심이 "거부한 대안 + 이유"라 그게 없으면 다음 세션이 같은 대안을 다시 꺼낸다.
- **supersede** — 위 new 입력 전부 + **어느 기존 ADR**(번호) + 왜 번복 + **전체/부분**(§2 판단).
- **lint · index** — 입력 없음.

내용의 *타당성*은 판단하지 않는다. 빠진 필수 슬롯만 채워 달라고 되묻고, 채워지면 스크립트 호출로 넘어간다(→ 환각 주의).

## 2. 오퍼레이션별 실행

실제 명령·파일명·템플릿·인덱스 형식은 **바인딩이 정본**이다.

### new — 새 결정 박제

1. **스크립트 호출** — 바인딩 `new` 명령에 제목(필요시 상태)을 넘긴다. 스크립트가 채번(재스캔)·스캐폴드·번호 출력을 한다. 번호를 LLM이 지정하지 않는다.
2. **prose 채우기** — 스캐폴드의 빈 슬롯(맥락·결정·거부한 대안·근거·영향)을 §1 내용으로 채운다. **섹션 구조는 건드리지 않는다**(스크립트/템플릿 정본 그대로). 호출자가 안 준 내용을 지어내지 않는다.
3. **인덱스 재생성** — 바인딩 인덱스 재생성 명령(§index)을 돌린다. 인덱스를 손으로 편집하지 않는다.

### supersede — 결정 번복 (전체/부분 + 양방향)

1. **전체 vs 부분 판단 (LLM의 핵심 판단)** — 옛 ADR이 *통째로* 틀렸나(전체), *일부 조항만* 바뀌고 나머지는 살아있나(부분)? **애매하면 사용자에게 확인**한다 — 살아있는 조항을 죽이면 안 된다.
   - **전체(full)** — 옛 status를 `폐기`로 바꾸고 양방향 supersede 링크(기존 status 취소선 보존).
   - **부분(partial)** — **옛 status 유지** + 양방향 `Amends`/`Amended by` 링크에 바뀐 조항을 단서로 박는다.
2. **스크립트 호출** — 바인딩 `supersede` 명령에 옛 번호 · 모드(full/partial) · (partial이면)바뀐 조항 · 새 제목을 넘긴다. 스크립트가 새 스캐폴드 + 옛 ADR 양방향 박기를 한다.
3. **prose 채우기** — 새 스캐폴드를 §1 내용으로(맥락에 "왜 번복"). 옛 ADR의 폐기 사유 TODO도 채운다.
4. **인덱스 재생성** — §index.

<examples>
<bad>옛 ADR-0016의 5개 조항 중 캐시 전략 1개만 바뀌었는데 전체(full)로 처리 → 옛 status가 통째 '폐기'로 덮여 살아있는 4개 조항까지 죽는다.</bad>
<good>1개 조항만 바뀌었으니 부분(partial): 옛 status 유지 + 양방향 Amends 링크에 "캐시 전략" 조항 단서. 나머지 4개 조항은 그대로 유효.</good>
</examples>

> **불변식: 양방향 단방향 금지.** 새 ADR에만 "옛 거 대체함"을 적고 끝내면 *옛 ADR만 읽는* 다음 세션이 폐기·개정된 결정을 멀쩡한 줄 안다. 스크립트가 양쪽을 박지만, 호출 후 lint로 양방향 일치를 확인한다.

### lint — 정합성 점검 (read-only · 보고 전용)

1. **스크립트 호출** — 바인딩 `lint` 명령(read-only). 스크립트가 ① 중복·빠진 번호 ② supersede 양방향 일치(전체·부분) ③ 상태 **어휘만** 유효성(단서절 무시) ④ 코드 앵커 고아(코드 경로 한정, 문서 제외)를 검사해 JSON으로 낸다.
2. **보고** — 결과를 사람 말로 옮긴다. **error(진짜 결함)와 advisory(권고·레거시 마이그레이션)를 구분**한다.

**lint은 자동수정하지 않는다.** 파일에 쓰는 오퍼레이션은 `index`(아래)뿐이다.

### index — 인덱스 재생성 (본문 파생)

new/supersede의 마지막 하위단계이자, "인덱스만 다시 만들어" 요청 시 단독 오퍼레이션이다. lint과 별개 명령이다.

- 바인딩 인덱스 재생성 명령이 본문 H1·상태줄을 스캔해 표를 다시 만든다(본문 = 단일 출처 → 제목/상태 drift 차단).
- **보존 규칙** — 인덱스 셀이 본문보다 정보가 많으면(수작업 큐레이션·레거시 부분폐기 단서) 스크립트가 기존 셀을 보존하고 경고만 낸다(자동 손실 금지). 어느 쪽이 정본인지(본문 vs 인덱스)는 맥락 판단이라 자동 손실을 금한다.
- 의미 있는 drift(양방향 누락·고아 앵커·제목 정본 충돌)는 사용자/메인 판단을 받는다.

## 3. 결과 보고

메인이 스크립트 JSON 출력을 사람 말로 옮겨 사용자에게 보고한다.

- **new** — 채번 번호 · 만든 파일 · prose 채운 섹션 · 인덱스 재생성 결과.
- **supersede** — 모드(전체/부분) · 새 번호 · 번복한 번호 · 양방향 박기 완료 · lint로 양방향 확인.
- **lint** — error/advisory 구분한 발견 목록 · 사용자 판단 필요분. 깨끗하면 "정합".
- 흐름(언제/무엇) 기록·커밋은 **메인이 처리**한다(이 스킬이 직접 쓰지 않는다 — 바인딩 §연동).
- **피드백 한 줄(의무):** 최종 보고에 "피드백: 없음" 또는 이번 실행에서 발견한 스킬 자체 결함·개선점 한 줄. 발견분만 그 스킬 feedback.md에 누적한다.

## LLM 환각 주의

LLM이 본문(맥락·근거·거부한 대안)을 채울 때 없는 API·가짜 근거·지어낸 대안을 환각하는 게 검증된 실패 모드다(리서치: Equal Experts "kernel of truth").

- **거부한 대안·근거·맥락은 호출자가 준 것만 박제한다.** 빈 슬롯을 그럴듯하게 메우지 않고 되묻는다.
- 채운 본문은 호출자가 **fact-check**하도록 보고한다 — 스킬은 결정이 "옳다"고 보장하지 않는다.

## 가드레일 (앞 절에 없는 금지만)

- **결정 날조 금지** — 결정 내용·거부한 대안·근거를 만들지 않는다. 빈 필수 슬롯은 되묻는다.
- **ADR 덮어쓰기 금지(누적)** — 기존 ADR 본문을 고쳐 결정을 바꾸지 않는다. 번복은 항상 새 번호 + 폐기/Amends(supersede)다. 덮어쓰면 이력이 사라진다.
- **스크립트 우회 금지** — 채번·양방향 링크·인덱스 재생성은 스크립트가 한다. LLM이 손으로 번호를 매기거나 인덱스 표를 편집하지 않는다.
- **경로·명령 하드코딩 금지** — 골격에 폴더·파일명·스크립트 명령·앵커 명령·템플릿 실체를 박지 않는다. 전부 바인딩이 채운다. 바인딩 파일 자체도 절대경로를 박지 않는다(cwd-상대).
