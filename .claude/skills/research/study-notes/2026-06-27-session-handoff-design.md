# Study Note — 세션 핸드오프 설계 패턴 리서치 (2026-06-27)

## 쟁점과 결론 도출 과정

### 쟁점 1: "하이브리드가 최선"이 두 family 합의인데 조건이 뭔가?
- Claude 갈래 C: Amp 폐기 사례 — 자동 compaction 왜곡. "하이브리드"가 아니라 "무감독 덮어쓰기"가 문제.
- Codex: 하이브리드 권장, append log 분리.
- opus 적대검증: 반증 시도 — "동기화 규율 없으면 worse"가 성립하는가? → **현 CCB가 반증 사례**. 과거 파일 동결(append-only)이라 모순이 구조적으로 불가능.
- 결론: "하이브리드 최선"은 살아남되 **수정형** — "append-only 동결형"이어야 한다는 조건 추가.

### 쟁점 2: "핸드오프 = 별도 레이어" — 왜 프로젝트 CLAUDE.md에 그냥 안 넣나?
- opus 적대검증: 반증 시도 — "그냥 CLAUDE.md에 넣으면 자동 로드됨, 레이어 줄임 가능하지 않나?"
- 반박: 수명·쓰기주체·주입시점이 기조 문서와 직교. CLAUDE.md = 불변 기조 (사람이 드물게 갱신, 항상 상주). 핸드오프 = 매 세션 휘발성 (에이전트가 매번 쓰고, resume 시점에만 1회 주입).
- 결론: **강화됨**. 현 engram CLAUDE.md 자체가 "기조 vs 상태 분리"를 명문화.

### 쟁점 3: ETH Zurich 수치
- Claude 팬아웃이 가져온 수치 (arXiv 2602.14690, 성공률 3% 감소, 비용 20%↑).
- opus 적대검증: arXiv ID 2602 = 2026년 2월, 미래 날짜 형식. 직접 확인 불가.
- 결론: **기각**. 방향성("상주 오염 위험")만 위험으로 생존.
- 교훈: Claude 팬아웃이 환각성 수치를 가져올 수 있음. opus 적대검증이 실제로 걸러냄 — cross-family 가치 실증.

## Medium tier가 실제로 어떻게 달랐나

- 갈래 4개 병렬(Claude 4명 + Codex 1명) — 각자 독립 조사.
- Codex가 Devin을 미언급(상업 서비스 접근 한계), Claude는 공식 문서 직접 확인. → 팬아웃 family 특성 차이 실증.
- Codex가 ETH Zurich 수치를 미언급 → 양쪽 합의가 없어 opus가 검증 → 기각. Light tier였으면 불일치 표기 후 그냥 넘어갔을 것.
