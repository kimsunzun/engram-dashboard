# Research — 실행 절차

`$ARGUMENTS` = 조사 주제 [+ 강도 `light`|`medium`|`deep`]. 주제가 모호하면 1~2개만 되묻는다. 강도 미지정이면 **라우팅 가이드(아래)로 추정**하되 medium 기본 — 어떤 강도로 도는지 호출 시 사용자에게 한 줄 명시한다.

## 0. Codex 가용성 확인 (medium+ 리뷰 전 체크)

`mcp__codex__codex` MCP가 이 세션에 연결됐는지 확인한다. light는 Codex 적대 리뷰가 없으므로 미연결이어도 무방하다 — **medium 이상에서만 필요**하다.

**medium+인데 미연결이면 즉시 사용자에게 알리고 진행 여부를 확인한다:**
> "Codex MCP가 연결되지 않았습니다 — cross-family 적대 리뷰 없이 주계열 수집 + grounding만 가능합니다 (confident-wrong 적출 불성립). light로 낮춰 진행할까요, 리뷰 없이 계속할까요, 아니면 Codex 연결 후 재실행할까요?"

사용자 확인 없이 **조용히 적대 리뷰를 건너뛰고 완료로 처리하지 않는다** — 그렇게 하면 confident-wrong 검증이 없었음에도 medium이 완료된 것처럼 보인다.

**사용자가 "리뷰 없이 계속"을 고르면 산출물을 열화(degraded)로 명시 라벨한다** — 보고서 메타헤더·상태 줄에 `medium (적대 리뷰 생략 — cross-family 검증 없음)`으로 박는다. 조용한 "medium"으로 내보내면 cross-family 검증이 없었다는 품질 신호가 사라진다(사용자가 나중에 "medium이었으니 적대 리뷰를 거쳤다"고 오신뢰).

## 역할 → 모델 배정표 (모델명은 여기 한 곳에만)

본문은 **역할(슬롯)** 으로만 말한다. 모델 바인딩은 이 표가 유일한 정본이다 — 신모델(Fable 등)이 나오면 **본문·절차를 안 건드리고 이 표만 교체**한다(프로젝트 원칙: 특정 모델에 코드를 묶지 않는다).

| 슬롯(역할) | 모델 | 하는 일 |
|---|---|---|
| 주계열 수집자 | Claude Sonnet | 팬아웃 조사 · 출처 회수 · calibration(기권/확신도) |
| 주계열 합성·grounding | Claude Opus  ← 신모델 나오면 여기만 교체 | 클레임 종합 · 클레임↔출처 함의 검증(메인이 소유) |
| cross-family 리뷰어 | Codex (gpt) | 합성 산출물 적대 리뷰(medium+) |
| 메인 오케스트레이터 | Claude Opus (현재 세션) | 분해·라우팅·취합·판정 집계·보고 |
| (deep 옵션) cross-family 수집자 | Codex (gpt) | deep에서만 병렬 수집(BLIND) |

- **불변 — 리뷰어는 주계열과 다른 family:** cross-family 리뷰어(Codex)는 주계열(Claude)과 반드시 **다른 family**여야 confident-wrong·학습 편향이 갈린다. 같은 family로는 자기 산출을 self-consistency로 무르게 봐 적출이 안 된다.
- **fresh 분리 — 생산자 ≠ 리뷰어:** 리뷰어 슬롯은 수집자·합성자·메인 오케스트레이터와 다른 **fresh 인스턴스**다(자기 산출을 자기가 통과시키지 않는다).
- **tiebreaker는 조건부(불변식 아님):** deep에서 cross-family 병렬 수집을 켰고 두 답이 정면 충돌할 때, **두 family 어느 쪽도 아닌 제3 family**가 가용하면 tiebreak 리뷰 옵션을 쓴다. **제3 family 미보유가 기본**이므로 없으면 contested로 남긴다(§5) — same-family tiebreak는 편향이 안 갈려 금지.
- (실측: fresh 독자는 역할 슬롯 + 이 배정표 조합을 무리 없이 이해했다 — 모델 매핑은 이 표가 짊어진다.)

## 강도표 (tier별 정량 정본)

강도는 **라우팅 + 정량**을 한 번에 정한다. **정량(수집자 수·검색량·effort·grounding 범위·리뷰 범위)은 이 표가 유일 정본**이다 — 다른 절 본문은 tier 이름으로 참조만 하고 숫자를 다시 적지 않는다(rot 방지).

| tier | 주계열 팬아웃 / 검색 | grounding (메인 외부·상시) | cross-family 적대 리뷰 | 종료 | 산출 |
|---|---|---|---|---|---|
| **light** | 1명 / 각 3~5회 | 핵심 클레임 스팟체크 | ✕ (없음) | 단일 패스 | 출처 포함 요약 |
| **medium**(기본) | 갈래별 1명 / 각 8~12회 | load-bearing 클레임 전수 | Codex 1회 (effort=medium · **레벨 2~3**) | 단일 패스 | 출처 단 보고서 + 리뷰 결과 |
| **deep** | 갈래별 1~2명 / 각 15~25회 + WebFetch | 전수 | Codex (effort=high · **레벨 4~5**) + 필요시 재리뷰 | loop-until-dry | 인용 보고서 + 쟁점 판정 |

- depth(재귀 깊이) × breadth(팬아웃 너비)는 직교. 깊이를 올릴수록 breadth는 절반으로 줄여 비용 폭발을 막는다.
- deep은 비용·시간이 크다 — 호출 시 경고하고 비동기/긴 대기를 전제한다.
- **light엔 Codex 리뷰가 없다** — findable-fact는 단일 수집 + grounding + 기권으로 충분하다(교차 이중화는 같은 웹→같은 오차라 이득 0). grounding은 light에도 켜진다(상시).
- **적대 강도 레벨(위 리뷰 열)** — 리뷰어가 수집자 답에서 독립하는 정도를 올리는 레벨 2~5 축, 각 레벨 정의는 §4 사다리. tier별 레벨은 위 표가 정본(레벨↑ = 독립도↑ = 적출력↑·비용↑).
- **deep 옵션 — cross-family 병렬 수집(누락 백스톱):** 후보·관점 공간이 넓거나 논쟁적이라 **무엇을 안 다뤘나가 결론을 가르는** omission-critical 주제 — 정착된 단일 사실이 아닌 것 — 는 deep에서 cross-family 수집자를 갈래별 병렬로 켠다(BLIND). 이게 §4 리뷰어의 누락 렌즈(⑥)로는 못 메우는 **잔여 frame lock-in의 백스톱**이다(리뷰어는 보고서에 앵커돼 있어 독립 수집보다 약하다 — §4). 이땐 abstention≠contradiction·만장일치≠정답이 수집 비교에도 적용된다(§5). cross-family 수집자는 자기 서브에이전트를 못 낳으니 **메인이 갈래마다 호출을 병렬로 띄운다**(한 메시지에 N개 동시).
- **deep 비용 하드 백스톱(강제 · 정량 정본):** loop·팬아웃·재리뷰가 폭주하지 않게 상한을 박는다 — **최대 iteration 4회 · 총 검색/도구호출 예산 ≤120회 · 벽시계/토큰 상한 = 세션 한도의 ~40%**. 상한에 닿으면 "dry"가 아니어도 loop를 멈추고 **남은 미커버 갈래/공백을 보고서 한계 섹션에 명시**한다(§5). 상한은 안전판이지 목표가 아니다 — 보통 그 전에 dry로 끝난다.
- tier 선택 트리거 = 되돌리기 비용·불확실성·confident-wrong 가능성·출처 분쟁 가능성(클수록 위). 애매하면 medium. escalation-only(하향 금지 — 도중 위험 발견 시 승격만).

## 라우팅 가이드 (강도 추정 — 급소)

강도 미지정이면 질의 유형으로 tier를 고른다(오분류가 비용/누락을 가른다):

| 질의 유형 | tier | 왜 |
|---|---|---|
| 단발 사실 · 웹서 바로 나옴 · 저위험 | **light** | 단일 수집이 ~100%. 리뷰는 낭비(둘 다 같은 웹) |
| 종합·비교·해석 · 논쟁적 · 1차자료 희소 · confident-wrong 위험 | **medium+** | 적대 리뷰가 confident-wrong·과장을 적출 |
| 비가역 · 고비용 · 안전치명 · 출처 분쟁 | **deep** | 사람 백스톱 + loop + (옵션)병렬 수집 |

- **confident-wrong 위험 판별:** 모델이 *확신에 차 답할 법한데 틀리면 치명적*인 것(널리 퍼진 오해, 버전·API 세부, 통계 수치, 인용 귀속) → medium+. 모델이 이미 기권/저확신하는 obscure 사실은 light로도 calibration이 잡는다(→ 리뷰 불필요).
- **"light처럼 보이지만 medium+"(오라우팅 = 치명 · 급소):** 단일 findable-fact로 *보이지만* 확신-오류 위험이 높아 **light 금지**인 부류 — **라이브러리/API 버전 · 법·규정 · 통계·수치 · 인용 귀속 · 최신 가격·정책 · 시간민감하거나 흔히 오기억되는 것 전반**. 웹서 한 방으로 답이 나올 듯 보여도 medium+로 라우팅한다(단일 수집 + grounding만으론 confident-wrong을 못 갈라서 — 여기서 오분류하면 확신에 찬 오답이 그대로 나간다).

## 1. Scope 분해 + 라우팅 확정 (Lead = 메인 오케스트레이터)
- 주제를 N개 sub-question(갈래)로 쪼갠다. **복잡도별 갈래·수집자 수를 규제**한다(단순=1 / 비교=2~4 / 복잡=다수). 막연히 늘리지 않는다(과도 스폰 = 토큰 낭비). **light는 단일 갈래로 돈다(분해 생략, 팬아웃 수집자 1).**
- **라우팅을 확정**한다(강도표·라우팅 가이드) — findable-fact면 light, 종합·confident-wrong 위험이면 medium+, 비가역·고위험이면 deep. 어떤 tier인지 사용자에게 한 줄 명시.
- 갈래마다 "무엇을·어디서 찾나"를 한 줄로 명시한다(작업 경계 — 중복 조사 방지).
- 대상 코드베이스/도메인에 제약 문서가 있으면 먼저 읽어 제약을 반영한다.

## 2. 주계열 팬아웃 수집 (병렬 · calibration 강제)
- 갈래별 1명을 **Agent 도구로 병렬 스폰**한다(`subagent_type: general-purpose`, `model` = 배정표의 주계열 수집자). 한 메시지에 여러 Agent 호출 = 동시 실행. 팬아웃 수집자끼리 **갈래가 겹치지 않게** 경계를 준다(같은 걸 두 번 긁지 않게).
- broad-first → narrow(처음부터 핀포인트 쿼리 금지).
- 각 수집자는 **결론 + 출처 URL + honest 확신도(확실/가능성 높음/불확실)만 구조화 반환**한다 — 결론만 회수(verbose 도구 출력을 메인 컨텍스트에 넣지 않는다).
- **calibration = 1급 규칙(강제):** 수집자는 **확신 없으면 "모름/기권"** 을 명시하고 지어내지 않는다. 저확신·근거(출처) 없는 주장은 §3 grounding에서 불신·강등된다. 이 규칙이 사실조회 환각의 대부분을 막는다(모델은 obscure 사실에 이미 self-calibrate — 확신-환각 대신 기권). *이 확신도는 자기보고이며, 최종 확신도 태그는 §5에서 grounding + 리뷰로 파생한다(자기보고 채택 X).*
- 소스 품질: 1차·공식·논문 우선, SEO 콘텐츠팜 배제. 수치·통계는 의심되면 출처 재확인.

## 3. 합성 + grounding 검증 (메인 외부 체크 · 상시 ON)

수집 결과를 합성하고(주계열 합성 슬롯), **메인이 클레임↔출처 함의를 외부에서 검증**한다 — 수집자 셀프 체크가 아니다(자기 인용을 자기가 통과시키면 환각을 못 거른다). 이 pass는 모든 tier에 켜진다(light 포함).

- **중복 정리 → 합성:** 갈래별 결론을 병합하고 중복·상충을 정리한다.
- **grounding (claim↔source 함의 — 환각 직격 · 상시):** load-bearing 클레임마다 **인용한 출처가 실제로 그 주장을 뒷받침하는지** 확인한다(ALCE/CitationAgent식 NLI/함의 판정). 뒷받침 안 되거나 잘못 귀속된 인용은 **플래그·강등**한다. **범위는 강도표대로**.
- 이건 "죽은 링크·오인용 거르기"(링크 생사·인용문 문구 대조)와 **다르다** — 링크가 살아 있고 인용문이 정확해도 그 출처가 그 주장을 실제로 *뒷받침하지 않을* 수 있다(claim↔source 함의는 별개 축).
- grounding 판정 = 클레임별 `{지지 · 부분지지 · 미지지 · 불확실}`. 이 판정이 §5 확신도 태그의 1차 근거다(수집자 자기보고 % 무시).
- **grounding은 medium+에서 단일 family가 아니다:** 메인(주계열)이 소스를 오독하면 **같은 family라 스스로 못 잡는다**(v2/v3 잔여 결함). 그래서 medium↑에선 §4 cross-family 리뷰어가 load-bearing 클레임의 claim↔source 함의를 **스팟 재검증**한다 — grounding이 Claude-메인 단독이 아니게. 이 이중 grounding이 메인 오독 환각의 2차 방어선이다.
- **light는 여기서 끝** — grounding 통과분을 출처 포함 요약으로 보고한다(§6). medium↑는 §4 적대 리뷰로 넘긴다.

## 4. cross-family 적대 리뷰 (medium+ · Codex fresh)

grounding을 통과한 **합성 산출물**을 다른 family(Codex)가 적대적으로 때린다. 이게 cross-family의 진짜 값어치 — 수집 이중화가 아니라 **confident-wrong 적출**이다. (`/review` 스킬의 Codex-blind-adversary 패턴을 리서치 산출물에 적용.)

- **리뷰어 = fresh Codex 호출** — 수집자·합성자·메인이 아니다(생산자 ≠ 리뷰어). 자기 산출을 자기가 통과시키지 않는다.
- **리뷰 렌즈(공격 표면):** ① 근거 없는 주장(출처 없이 단언) · ② 과장(overclaim — 출처가 말하는 것보다 세게 주장) · ③ 오귀속 인용(출처가 그 주장을 안 함) · ④ 논리 공백(결론이 근거에서 안 따라옴) · ⑤ **confident-wrong**(널리 퍼진 오해·틀린 세부를 확신조로) · ⑥ **완전성/누락**(무엇이 빠졌나 — 안 다룬 후보·반대 학파·최신 변경·반례·vendor caveat). ⑥은 리뷰어가 **자기 web_search로 누락을 능동 탐침**한다 — *있는 것을 비판*만이 아니라 *없는 것을 찾는다*(단일 수집자가 애초에 안 꺼낸 각도는 산출물만 봐선 안 보인다).
- **누락 렌즈(⑥)는 독립 수집보다 약하다 — 명시:** 리뷰어는 **보고서에 앵커돼 있어** 잔여 frame lock-in(omission blindness)을 완전히 못 벗는다 — 산출물 리뷰는 "안 꺼낸 프레임"을 부분적으로만 되찾는다. 그래서 후보·관점 공간이 넓거나 논쟁적인 **omission-critical 주제는 deep의 독립 cross-family 병렬 수집**을 백스톱으로 남긴다(강도표 deep 옵션). ⑥은 그 백스톱을 medium에서 근사할 뿐, 대체하지 않는다.
- **주관 판정 금지:** "어느 보고서가 더 잘 읽히나"(주관·self-preference)가 아니라 **"이 주장이 근거로 지지되나 / 어디가 틀렸나"**(적대·근거)만 본다. 근거기반 렌즈라야 same-family 자기선호가 죽고 적출력이 산다.
- **grounding 스팟 재검증(medium+ · cross-family 이중 grounding):** 리뷰어는 적대 렌즈에 더해 **load-bearing 클레임 몇 개의 claim↔source 함의를 직접 재확인**한다 — grounding이 주계열 메인 단독이 아니게(§3). 메인이 소스를 오독했으면 여기서 갈린다(같은 family면 못 잡을 오독).
- **호출 파라미터:** `sandbox: read-only`, `approval-policy: never`, `config: { "model_reasoning_effort": <강도표대로>, "tools": { "web_search": true } }`. web_search는 리뷰어가 **반증(counter-evidence)과 누락(⑥ — 안 다룬 후보·반례·caveat)** 을 찾게 켠다. effort는 강도표대로 tier로 스케일한다(숫자는 여기 다시 적지 않는다).
- **입력 = 합성 산출물 + 출처 + 수집 시 드러난 공백·버려진 후보·수집자 불확실 노트.** 리뷰어는 산출물을 봐야 때린다(수집 BLIND와 반대 축 — BLIND는 *병렬 수집* 에만 건다, §강도표 deep 옵션). 단 **"결론만 회수"(§2)는 *수집 결과의 메인 컨텍스트 복원*에만 적용**된다 — 리뷰어의 누락/커버리지 렌즈(⑥)는 *무엇이 빠졌나*를 봐야 하므로 **정제된 결론만 주면 굶는다**. 원시 검색 로그 덤프는 여전히 피하되, 공백·버려진 후보·수집자 기권 노트는 리뷰어에게 함께 넘긴다.
- **출력 = 판정 + findings[].** 각 finding = `{클레임(또는 누락) · 결함타입(위 렌즈) · 근거/반증 · 심각도}`. verbose 덤프 회수 금지 — 메인은 결론만 회수.
- **반박은 반증 출처 강제(evidence-grounded · 강제):** 반박(refute) verdict는 **반증(counter-evidence) 출처를 반드시 동반**한다 — 근거 없는 트집·의심만으로는 반박으로 치지 않는다(출처 없는 의심은 finding까진 되나 verdict는 안 된다). 적대를 과하게 하면 멀쩡한 답을 약한 근거로 반박하는 **오경보**가 늘고 비용도 폭발한다(debate가 정확도를 되레 떨어뜨린 연구 다수). 이 규칙이 "세되 헛되지 않게" 고삐를 잡는다. (§5 contested도 반증 있을 때만 — 정합.)
- **과신 금지 — 만장일치 ≠ 정답:** 리뷰가 아무것도 못 잡아도 정답 증명이 아니다(리뷰어도 일부 주제엔 같은 편향·blind spot). 리뷰 통과는 신뢰도를 *올리는 신호*지 *증명*이 아니다 — grounding + 사람 백스톱이 앵커.

### 적대 강도 레벨 사다리 (축 = 리뷰어가 수집자 답으로부터 독립하는 정도)

리뷰를 얼마나 세게 때리나는 **리뷰어가 수집자 답에 얼마나 안 묶이나**로 정한다 — 독립도가 높을수록 앵커링이 걷혀 적출력이 세지고 비용도 커진다. **레벨→tier 매핑은 강도표 정본**이라 여기 다시 적지 않는다(본문은 레벨을 이름으로만 참조 — rot 방지). **레벨 1(고무도장 — "이거 맞아?"식 확인)은 수집자 답에 앵커돼 적출력이 사실상 0이라 쓰지 않는다.** 쓰는 레벨은 2~5:

- **레벨 2 — 검산(읽고 검증):** 답 + 추론체인을 **읽고**, "틀렸다 가정"으로 각 홉을 출처로 되짚어 검증한다. 틀리면 **반증 출처와 함께** 고친다. (수집자 답을 보므로 앵커링 잔존.)
- **레벨 3 — 홉 독립 재도출:** 체인을 **안 읽고** 각 홉을 처음부터 리뷰어가 직접 재계산해 수집자 답과 대조한다. (앵커링 제거.)
- **레벨 4 — 다중 렌즈 + 대안답 랭킹:** 공격을 전문 렌즈로 분리(숫자정밀 · 개체혼동 · 날짜/시점 · 출처권위)해 각각 사냥하고, 답 후보 top-K를 **근거로 랭킹**한다(예/아니오 이진이 아니라 순위).
- **레벨 5 — blind 재해결:** 리뷰어에게 **질문만** 주고(답·체인·갈래분해 숨김) 통째로 다시 연구시켜 두 완성 답을 대조한다. 누락·틀린 프레임까지 잡는다 — 사실상 cross-family 독립 수집(§강도표 deep 옵션과 같은 축), 비용 ~2배.

## 5. 종결 + 판정 집계 (메인 · 승자 강요 X)

메인은 grounding 판정(§3)과 (medium+) 리뷰 판정(§4)을 **기계적으로 집계**한다 — 판결이 아니라 규칙 적용이다.

**확신도 = grounding + 리뷰에서 파생(자기보고·합의도 금지):** 태그 근거는 ① grounding `{지지/미지지/불확실}` ② 리뷰가 그 클레임을 때렸나. **"수집자 합의도 = 확신"은 폐기**(단일 웹을 둘이 긁으면 합의는 독립 신호가 아니라 연출). 자기보고 %도 무시(과신 편향 — 실증).

| grounding | 리뷰(medium+) | 독립 교차확증 | 확신도 태그 |
|---|---|---|---|
| 지지 | 통과(못 때림) or 단지 기권 | **있음**(제2 1차자료 / deep 독립 cross-family 수집 합의) | **확실** |
| 지지 | 통과(못 때림) or 단지 기권 | 없음(단일 출처) | **가능성 높음** |
| 지지 | 근거로 반박(counter-evidence) | (무관) | **contested** → 아래 |
| 부분지지 | 통과/약한 지적 | (무관) | **가능성 높음** |
| 부분지지 | 근거로 반박(counter-evidence) | (무관) | **contested** → 아래 |
| 미지지 / 불확실 | (무관) | (무관) | **불확실** → 보류 |

- **확신도 상한 — 만장일치≠정답 존중 (급소):** grounding 지지 + 리뷰 통과만으로는 **가능성 높음**까지다. **확실**은 **독립 교차확증**(서로 다른 제2 1차자료, 또는 deep의 독립 cross-family 수집이 같은 답)이 있을 때만 준다. 단일 출처가 grounding됐고 리뷰가 안 때렸다는 건 *반증이 안 나왔다*는 뜻이지 *독립 확인*이 아니다 — 리뷰가 공격 안 한 lone-source 클레임은 확실이 아니라 **가능성 높음**. (만장일치≠정답과 정합: 아무도 못 때렸다 ≠ 증명.)
- **abstention ≠ contradiction (v2 결함 수정 · 급소):** 리뷰어(또는 deep의 cross-family 수집자)가 클레임을 **단지 확인 못 함/기권**한 것은 반박이 아니다 — grounding이 지지하면 채택한다(정답을 모순으로 강등하지 않는다). **반증(서로 다른 답·counter-evidence)을 낼 때만** contested.
- **contested 처리 — 메인은 승자를 안 고른다:** load-bearing contested → mode-aware 에스컬레이션(아래). (deep에서 cross-family 병렬 수집을 켰고 제3 family가 가용하면 tiebreak 리뷰 옵션 — 없으면 contested 유지.)
- **전체 종료 조건 (loop-until-dry — deep, 범위는 강도표대로):** (a) 모든 sub-question 커버 (b) 마지막 iteration이 **새 load-bearing 공백/불일치를 못 찾음** (c) 모든 클레임이 판정(불확실 포함)을 가짐. **deep만 loop, light/medium은 단일 패스.** **+ 하드 백스톱(강제):** iteration·검색/도구호출·토큰 상한(수치 = 강도표 정본)에 닿으면 dry가 아니어도 loop를 멈추고 **남은 미커버 갈래/공백을 한계 섹션에 명시**한다 — 상한은 폭주 안전판이지 목표가 아니다.
- **금지 안티패턴:** 합의로 몰아가는 강요 · 충돌을 어중간한 중간값으로 평균내기 · same-family tiebreak · 만장일치=증명.

### mode-aware 에스컬레이션 (모드 = 자율 vs 대화 감지, 문자적 "재석" 아님)

contested/미지지는 위 집계가 처리한다 — **메인이 판결하지 않는다.**

- **대화(interactive) 모드:** load-bearing `contested`/미지지 → **사용자에게 질문**.
- **자율("진행 쭉해") 모드:** 진행하되 **contested/불확실 태그를 유지**(조용히 해소 X) + **보고서의 쟁점/한계 섹션에 쟁점→판정불가 경위를 로그**한다(사후감사 sink — 보고서가 프로젝트 산출 경로에 영속되므로 이것으로 충분, 스킬 폴더에 별도 로그를 쓰지 않는다). contested load-bearing 클레임에 **의존하는 하류 결정은 플래그**한다(조용히 확정하지 않는다). **light는 보고서가 없으므로, 자율 모드에서 load-bearing 미지지/불확실이 나오면 medium으로 승격한다**(escalation-only — 승격이 보고서=sink를 만든다). *(쟁점 로그 ≠ `feedback.md` — feedback.md는 스킬 결함용, 리서치 쟁점은 보고서.)*
- **마이너 contested**(결론에 영향 없음) → 메인 자율(양 모드 공통).
- **마이너/굵음 게이트 = tier 트리거 재활용:** 되돌리기 비용·불확실성·confident-wrong 가능성·출처 분쟁 가능성이 크면 굵은 쟁점(질문 또는 로그), 결론에 영향 없으면 마이너(메인 자율).

## 6. 산출 분기 (결정권 = 사용자)
- **인용 검증 (별도 pass — 링크 생사·오인용):** load-bearing 인용의 링크 생사·인용문 문구를 대조해 죽은 링크·오인용을 거른다(§3 grounding = claim↔source 함의는 별개 축 — 여기선 링크·문구 위생). 범위는 강도표대로.
- **light**면 보고서 없이 **출처 포함 요약**(grounding 통과 결론 + 확신도 + 출처 + 기권/공백)을 메인이 보고한다.
- **medium↑**면 **출처 단 보고서**: 메타헤더(상태·방법·날짜·확신도 범례) · 발견마다 확신도(§5 태그) · 출처 · **grounding 결과(뒷받침/미뒷받침·오귀속 플래그)** · **적대 리뷰 결과(적출 결함·반증)** · **쟁점/한계(contested 클레임 포함)** · 공백.
- 결정·선택이 필요하면 `AskUserQuestion`으로 선택지를 제시한다(임의 채택 금지). 추천 1개는 맨 앞 "(추천)", 각 옵션에 한 줄 트레이드오프.
- 저장 위치/포맷·결정 기록(ADR 등)·후속 작업은 프로젝트 관례에 위임한다(리서치 문서 폴더가 있으면 거기, 없으면 사용자에게 보고).

## 7. 설계-결정 모드 (OSS 서베이 → 접근법 선택) — 옛 prior-art 흡수
조사가 *순수 사실*이 아니라 *"이 기능을 어떻게 만들까"*(설계 착수 전 OSS·표준 서베이)면, 위 절차에 **출력 4개**를 더한다(엔진은 그대로 — 라우팅·grounding·적대 리뷰·AskUserQuestion 유지):
- **제약 먼저(§1 보강):** 프로젝트 제약(기술스택·교체성·규모·LLM제어·라이선스)을 CLAUDE.md/ADR에서 추출해 적는다 — 적합도 판정 기준.
- **by-candidate 팬아웃(§2 변형):** 갈래 대신 *후보별*로 팬아웃하고 스키마에 `{핵심메커니즘·장점·단점·라이선스·성숙도/채택도}`를 추가(로컬 클론 references 있으면 코드까지).
- **제약 적합도 표(§6 보강):** 종합에 **후보 × 제약 적합도 표**(행=후보, 열=제약+장단)를 만든다 — 결정용 핵심 아티팩트. (설계 서베이는 종합형이라 **기본 medium+** — 적대 리뷰가 후보 과장·오귀속을 적출.)
- **거부 후보 → ADR 거부 대안(§6 연결):** 명백히 안 맞는 후보를 *사유와 함께* 추려, `/adr`로 박을 때 **거부한 대안**으로 넘긴다(ADR의 핵심).

## 가드레일 (앞 절에 없는 금지만)

앞 절에 이미 박힌 규약(calibration 강제·grounding 상시·abstention≠contradiction·확신도=grounding+리뷰·mode-aware 에스컬레이션·만장일치≠정답)은 여기서 반복하지 않는다. 이 절은 **다른 곳에 안 적힌 금지**만 모은다:

- **리뷰어 same-family 금지** — cross-family 리뷰어는 주계열(Claude)과 다른 family(Codex). Claude로 대체하면 confident-wrong이 안 갈린다.
- **생산자 = 리뷰어 금지** — 리뷰어는 수집자·합성자·메인과 다른 **fresh 인스턴스**(§4). 자기 산출을 자기가 통과시키지 않는다.
- **근거 없는 반박 금지(evidence-grounded)** — 반박 verdict는 반증(counter-evidence) 출처 동반 필수(§4). 출처 없는 의심만으론 반박 아님 — 과적대 오경보·비용폭발 방지.
- **수집 이중화로 회귀 금지** — cross-family를 *수집자*로 기본 투입하지 않는다(같은 웹→같은 오차, 비용 2배, 오경보). 병렬 수집은 **deep 옵션 한정**(강도표).
- **collector-agreement = confidence 금지** — 확신도는 grounding + 리뷰에서만 파생(§5). "둘이 같은 답 = 확실"은 단일 웹 위 연출이다.
- **grounding 셀프 체크 금지** — claim↔source 함의는 수집자가 아니라 **메인이 외부에서**(§3).
- **메인 판결 금지** — contested에서 메인은 승자를 고르지 않는다. mode-aware 에스컬레이션(§5). same-family tiebreak도 금지(제3 family 가용 시에만 tiebreak, 없으면 contested).
- **완전 협업 금지(병렬 수집 시)** — deep에서 cross-family 병렬 수집을 켜면 두 family는 결과를 공유하지 않는다(앵커링·sycophancy로 교차 무력화). 적대 리뷰는 반대로 산출물을 본다 — BLIND는 *병렬 수집* 축에만.
- **과도 스폰 금지** — 복잡도별 수 규제. 단순 주제에 대군 스폰 X.
- **결론만 회수** — verbose 검색/리뷰 출력으로 메인 컨텍스트를 오염시키지 않는다(에이전트가 압축).
- **환각 금지** — 사실/추정 구분, 불확실은 "불확실", 출처 없으면 "지식 기반" 표기.
- **검색 미필터링 금지** — 콘텐츠팜·낮은 신뢰 소스 배제.
- **peer groupchat 금지** — 오케스트레이터(메인) 없는 자유 토론 구조는 오류를 증폭시킨다.
