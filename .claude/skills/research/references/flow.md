# Research — 실행 절차

`$ARGUMENTS` = 조사 주제 [+ 강도 `light`|`medium`|`deep`]. 주제가 모호하면 1~2개만 되묻는다. 강도 미지정이면 복잡도로 추정하되 medium 기본 — 어떤 강도로 도는지 호출 시 사용자에게 한 줄 명시한다.

## 0. Codex 가용성 확인 (필수 — 스폰 전 체크)

`mcp__codex__codex` MCP가 이 세션에 연결됐는지 확인한다.

**미연결이면 즉시 사용자에게 알리고 진행 여부를 확인한다:**
> "Codex MCP가 연결되지 않았습니다 — Codex blind 교차 없이 Claude 단독 조사만 가능합니다 (cross-family 교차검증 불성립). 계속 진행하시겠습니까, 아니면 Codex 연결 후 재실행하시겠습니까?"

사용자 확인 없이 **조용히 Claude 단독으로 대체하지 않는다** — 그렇게 하면 교차검증이 없었음에도 완료된 것처럼 보인다.

---

## 강도(intensity) 정하기

| tier | Claude 팬아웃 / 검색 | Codex 교차 | 적대 검증(opus) | 산출 |
|---|---|---|---|---|
| **light** | 1명 / 각 3~5회 | blind 1명 | ✕ | 출처 포함 요약 (보고서 X) |
| **medium (기본)** | 갈래별 1명 / 각 8~12회 | blind 1회 | 핵심 주장 2~3개 | 출처 단 보고서 + 교차표 |
| **deep** | 갈래별 1~2명 / 각 15~25회 + WebFetch | blind | 다중 주장 + 불일치 전수 | 인용 보고서 + 쟁점 판정 (+옵션 loop-until-dry) |

- depth(재귀 깊이) × breadth(팬아웃 너비)는 직교. 깊이를 올릴수록 breadth는 절반으로 줄여 비용 폭발을 막는다.
- deep은 비용·시간이 크다 — 호출 시 경고하고 비동기/긴 대기를 전제한다.
- light도 cross-family 교차(Claude 1 + Codex 1 독립)는 유지한다 — 적대검증만 생략하는 경량판이다. 교차 자체가 불필요한 단순 즉답은 이 스킬이 아니라 일반 조사로.
- tier 선택 트리거: 결정의 되돌리기 비용·불확실성·환각 시 피해·출처 분쟁 가능성이 클수록 위 tier. 애매하면 medium.

## 1. Scope 분해 (Lead = 메인 오케스트레이터)
- 주제를 N개 sub-question(갈래)로 쪼갠다. **복잡도별 갈래·에이전트 수를 규제**한다(단순=1 / 비교=2~4 / 복잡=다수). 막연히 늘리지 않는다(과도 스폰 = 토큰 낭비). **light는 단일 갈래로 돈다(분해 생략, Claude 1 + Codex 1).**
- 갈래마다 "무엇을·어디서 찾나"를 한 줄로 명시한다(작업 경계 — 중복 조사 방지).
- 대상 코드베이스/도메인에 제약 문서가 있으면 먼저 읽어 제약을 반영한다.

## 2. Claude 팬아웃 검색 (Sonnet 서브에이전트, 병렬 · BLIND)
- 갈래별 1명을 **Agent 도구로 병렬 스폰**한다(`subagent_type: general-purpose`, `model: sonnet`). 한 메시지에 여러 Agent 호출 = 동시 실행.
- broad-first → narrow(처음부터 핀포인트 쿼리 금지).
- 각 에이전트는 **결론 + 출처 URL + 확신도(확실/가능성 높음/불확실)만 구조화 반환**한다 — 결론만 회수(verbose 도구 출력을 메인 컨텍스트에 넣지 않는다).
- 소스 품질: 1차·공식·논문 우선, SEO 콘텐츠팜 배제. 수치·통계는 의심되면 출처 재확인.
- **상대(Codex)의 결과를 주지 않는다**(독립 유지).

## 3. Codex 독립 교차 (병렬 · BLIND)
- 모든 tier에서 같은 갈래(또는 핵심 갈래)를 `mcp__codex__codex`로 독립 조사한다(light=1명, medium 이상=갈래별). 호출 파라미터: `sandbox: read-only`, `approval-policy: never`, `config: { "model_reasoning_effort": "medium", "tools": { "web_search": true } }`.
- Codex에게도 Claude 결과를 주지 않는다. **cross-family = 독립 편향**이 교차검증의 근거다 — 같은 family 둘로는 효과가 없으므로 Codex는 필수다.
- Codex가 web 불가면 지식 기반으로 답하되 "지식 기반"으로 표기시킨다.

## 4. 교차 대조 + 적대 검증 (opus, aggregator 단일 패스)
- 갈래별 Claude ↔ Codex 결과를 **클레임 단위로 대조**한다: 합의 = 확신도↑(단 **"만장일치 ≠ 정답"** — 둘 다 같은 학습 편향에 수렴했을 수 있음), 불일치 = 쟁점 플래그. **light는 여기까지(교차 대조)만 하고 적대 검증은 생략**한다.
- 핵심 주장·불일치 클레임을 **적대 검증**한다(반증 시도 + 웹 재확인). 검증은 opus 서브에이전트(또는 메인 opus). deep은 다중/전수.
- 불일치가 안 풀리면 **사용자에게 에스컬레이션**한다(판정을 떠넘김 — 메인이 한쪽 family면 자기편 편향 방지). **light는 적대검증이 없으므로 불일치를 풀려 하지 않고 양쪽을 병기**해 보고하고, 그 불일치가 결정에 영향을 주면 medium 승격 또는 사용자 선택을 요청한다.
- 섞는 것은 **이 단계의 불일치 클레임에 한정**한다. 수집(2·3) 단계에서 모델끼리 결과를 공유하지 않는다.

## 5. 종합 + 인용 검증 (별도 pass)
- 중복 정리 → 확신도 순위 → 출처 검증(죽은 링크·오인용 거르기).
- light면 보고서 없이 **출처 포함 요약**(합의/불일치 + 확신도 + 출처)을 메인이 보고한다. medium 이상이면 **출처 단 보고서**를 만든다:
  - 메타헤더(상태·방법·날짜·확신도 범례) · 발견마다 확신도 · 출처 · **교차검증표(Claude↔Codex 수렴/발산)** · 공백 · 한계.
- 저장 위치/포맷은 프로젝트 관례를 따른다(리서치 문서 폴더가 있으면 거기, 없으면 사용자에게 보고).

## 6. 산출 분기 (결정권 = 사용자)
- 결정·선택이 필요하면 `AskUserQuestion`으로 선택지를 제시한다(임의 채택 금지). 추천 1개는 맨 앞 "(추천)", 각 옵션에 한 줄 트레이드오프.
- 결정 기록(ADR 등)·후속 작업은 프로젝트 관례에 위임한다.

## 7. 설계-결정 모드 (OSS 서베이 → 접근법 선택) — 옛 prior-art 흡수
조사가 *순수 사실*이 아니라 *"이 기능을 어떻게 만들까"*(설계 착수 전 OSS·표준 서베이)면, 위 절차에 **출력 4개**를 더한다(엔진은 그대로 — cross-family·적대검증·AskUserQuestion 유지):
- **제약 먼저(§1 보강):** 프로젝트 제약(기술스택·교체성·규모·LLM제어·라이선스)을 CLAUDE.md/ADR에서 추출해 적는다 — 적합도 판정 기준.
- **by-candidate 팬아웃(§2 변형):** 갈래 대신 *후보별*로 팬아웃하고 스키마에 `{핵심메커니즘·장점·단점·라이선스·성숙도/채택도}`를 추가(로컬 클론 references 있으면 코드까지).
- **제약 적합도 표(§5 보강):** 종합에 **후보 × 제약 적합도 표**(행=후보, 열=제약+장단)를 만든다 — 결정용 핵심 아티팩트.
- **거부 후보 → ADR 거부 대안(§6 연결):** 명백히 안 맞는 후보를 *사유와 함께* 추려, `/adr`로 박을 때 **거부한 대안**으로 넘긴다(ADR의 핵심).

## 가드레일 (안티패턴 차단)
- **same-family 교차 금지** — 반드시 Codex를 포함한다(독립 편향). Claude 둘로 대체하지 않는다.
- **완전 협업 금지** — 수집 단계에서 모델끼리 결과를 공유하지 않는다(앵커링·sycophancy로 교차검증 무력화). 섞는 건 4단계 불일치 클레임 한정.
- **과도 스폰 금지** — 복잡도별 수 규제. 단순 주제에 대군 스폰 X.
- **결론만 회수** — verbose 검색 출력으로 메인 컨텍스트를 오염시키지 않는다(서브에이전트가 압축).
- **만장일치 ≠ 정답** — 합의해도 공통 편향을 의심한다. deep은 반증 1회 이상.
- **환각 금지** — 사실/추정 구분, 불확실은 "불확실", 출처 없으면 "지식 기반" 표기.
- **검색 미필터링 금지** — 콘텐츠팜·낮은 신뢰 소스 배제.
- **peer groupchat 금지** — 오케스트레이터(opus aggregator) 없는 자유 토론 구조는 오류를 증폭시킨다.
