`$ARGUMENTS` = [서브클래스] [조사 주제] [강도 `light`|`medium`|`deep`] — 파싱 규칙은 SKILL.md "인자 파싱"이 정본. 주제 생략 시 대화 컨텍스트에서 추론(없거나 애매하면 1~2개만 되묻기). 강도 미지정이면 **라우팅 가이드(아래)로 추정**하되 medium 기본 — 어떤 주제·강도로 도는지 시작 시 한 줄 명시한다.

> **조합 마커 (서브클래스 규약):** 🔒SEALED = 서브클래스가 못 덮는 불변식(약화·생략 시 무시+보고). 🕳HOLE = 서브클래스가 프로젝트 특화로 채우는 지점(없으면 Base 기본값). 조합 절차는 SKILL.md "서브클래스 조합".

## 0. 『cross-family(blind) 리뷰어』 가용성 확인 (medium+ 전 체크)

🔒SEALED — 『cross-family(blind) 리뷰어』의 바인딩(전역 사전 "호출" 열의 MCP 툴)이 이 세션에 연결됐는지 확인한다. light는 적대 리뷰가 없어 미연결이어도 무방 — **medium 이상에서만 필요**하다. (서브클래스가 이 게이트를 끄지 못한다.)

**medium+인데 미연결이면 즉시 사용자에게 알리고 진행 여부를 확인한다:**
> "cross-family(blind) 리뷰어가 연결되지 않았습니다 — 적대 리뷰 없이 주계열 수집 + grounding만 가능합니다 (confident-wrong 적출 불성립). light로 낮출까요, 리뷰 없이 계속할까요, 연결 후 재실행할까요?"

사용자 확인 없이 **조용히 적대 리뷰를 건너뛰고 완료로 처리하지 않는다.** 사용자가 "리뷰 없이 계속"을 고르면 산출물을 **열화(degraded)로 명시 라벨**한다 — 메타헤더·상태 줄에 `medium (적대 리뷰 생략 — cross-family 검증 없음)`으로 박는다(조용한 "medium"으로 내보내면 검증 없었다는 신호가 사라진다).

## 강도표 (tier별 정량 정본)

**정량(수집자 수·검색량·effort·grounding·리뷰 범위)은 이 표가 유일 정본** — 다른 절은 tier 이름으로 참조만 하고 숫자를 다시 적지 않는다(rot 방지).

| tier | 주계열 팬아웃 / 검색 | grounding (메인 외부·상시) | cross-family 적대 리뷰 | 종료 | 산출 |
|---|---|---|---|---|---|
| **light** | 1명 / 각 3~5회 | 핵심 클레임 스팟체크 | ✕ (없음) | 단일 패스 | 출처 포함 요약 |
| **medium**(기본) | 갈래별 1명 / 각 8~12회 | load-bearing 클레임 전수 | 1회 (effort=medium · **레벨 2~3**) | 단일 패스 | 출처 단 보고서 + 리뷰 결과 |
| **deep** | 갈래별 1~2명 / 각 15~25회 + WebFetch | 전수 | effort=high · **레벨 4~5** + 필요시 재리뷰 | loop-until-dry | 인용 보고서 + 쟁점 판정 |

- effort → 모델 = 전역 사전 참조.
- depth(재귀 깊이) × breadth(팬아웃 너비)는 직교 — 깊이를 올리면 breadth를 절반으로 줄인다.
- deep은 비용·시간이 크다 — 호출 시 경고하고 비동기/긴 대기를 전제한다.
- **적대 강도 레벨(리뷰 열의 2~5)** = 리뷰어가 수집자 답에서 독립하는 정도 사다리 — 정의는 §4, tier 매핑은 이 표가 정본(레벨↑ = 독립도↑ = 적출력↑·비용↑).
- **deep 옵션 — cross-family 병렬 수집(누락 백스톱):** 후보·관점 공간이 넓거나 논쟁적이라 **무엇을 안 다뤘나가 결론을 가르는** omission-critical 주제는 deep에서 『cross-family(blind) 리뷰어 · 수집자』(수집자 슬롯)를 갈래별 병렬로 켠다(BLIND). §4 리뷰어 누락 렌즈(⑥)로 못 메우는 잔여 frame lock-in의 백스톱이다. 이땐 abstention≠contradiction·만장일치≠정답이 수집 비교에도 적용(§5). **cross-family 수집자는 서브에이전트를 못 낳으니 메인이 갈래마다 병렬 호출한다**(한 메시지에 N개 동시).
- **deep 비용 하드 백스톱(강제 · 정량 정본):** **최대 iteration 4회 · 총 검색/도구호출 ≤120회 · 벽시계/토큰 ≤ 세션 한도 ~40%**. 상한에 닿으면 dry가 아니어도 loop를 멈추고 **남은 미커버 갈래/공백을 보고서 한계 섹션에 명시**한다(§5). 안전판이지 목표가 아니다.
- tier 선택 트리거 = 되돌리기 비용·불확실성·confident-wrong 가능성·출처 분쟁 가능성(클수록 위). 애매하면 medium. **escalation-only**(하향 금지 — 도중 위험 발견 시 승격만).

## 라우팅 가이드 (강도 추정)

강도 미지정이면 질의 유형으로 tier를 고른다:

| 질의 유형 | tier | 왜 |
|---|---|---|
| 단발 사실 · 웹서 바로 나옴 · 저위험 | **light** | 단일 수집이 ~100%. 리뷰는 낭비(둘 다 같은 웹) |
| 종합·비교·해석 · 논쟁적 · 1차자료 희소 · confident-wrong 위험 | **medium+** | 적대 리뷰가 confident-wrong·과장을 적출 |
| 비가역 · 고비용 · 안전치명 · 출처 분쟁 | **deep** | 사람 백스톱 + loop + (옵션)병렬 수집 |

- **confident-wrong 위험 판별:** 모델이 확신에 차 답할 법한데 틀리면 치명적인 것(널리 퍼진 오해, 버전·API 세부, 통계 수치, 인용 귀속) → medium+. 이미 기권/저확신하는 obscure 사실은 light calibration이 잡는다.
- **"light처럼 보이지만 medium+"(오라우팅 = 치명):** 단일 findable-fact로 보이지만 확신-오류 위험이 높아 **light 금지**인 부류 — **라이브러리/API 버전 · 법·규정 · 통계·수치 · 인용 귀속 · 최신 가격·정책 · 시간민감하거나 흔히 오기억되는 것 전반**. 웹서 한 방으로 답이 나올 듯 보여도 medium+로 라우팅한다(단일 수집 + grounding만으론 confident-wrong을 못 갈라서).

## 1. Scope 분해 + 라우팅 확정 (Lead = 메인 오케스트레이터)

- 주제를 N개 sub-question(갈래)로 쪼갠다. **복잡도별 규제**(단순=1 / 비교=2~4 / 복잡=다수). **light는 단일 갈래로 돈다(분해 생략, 수집자 1).**
- **라우팅 확정**(강도표·라우팅 가이드). 어떤 tier인지 사용자에게 한 줄 명시.
- 갈래마다 "무엇을·어디서 찾나"를 한 줄로 명시(작업 경계).
- 🕳HOLE(제약 문서) — 대상 코드베이스/도메인에 제약 문서가 있으면 먼저 읽어 반영한다. **어느 문서를 읽나는 서브클래스가 지정**(없으면 스킵).

## 2. 주계열 팬아웃 수집 (병렬 · calibration 강제)

- 갈래별 『조사 수집자』 1명을 **Agent 도구로 병렬 스폰**한다(`subagent_type: general-purpose`, `model` = 『조사 수집자』 배정). 한 메시지에 여러 Agent 호출 = 동시 실행. 갈래가 겹치지 않게 경계를 준다.
- broad-first → narrow.
- 각 수집자는 **결론 + 출처 URL + honest 확신도(확실/가능성 높음/불확실)만 구조화 반환** — 결론만 회수(verbose 도구 출력을 메인 컨텍스트에 넣지 않는다).
- 🔒SEALED **calibration = 1급 규칙(강제):** 수집자는 **확신 없으면 "모름/기권"** 을 명시하고 지어내지 않는다. 저확신·근거 없는 주장은 §3 grounding에서 강등된다. 이 확신도는 자기보고 — 최종 태그는 §5에서 grounding + 리뷰로 파생(자기보고 채택 X). (서브클래스가 못 끈다.)
- 🕳HOLE(출처 우선순위) — 소스 품질: 1차·공식·논문 우선, SEO 콘텐츠팜 배제. 수치·통계는 의심되면 출처 재확인. **프로젝트별 우선 출처(내부 문서·특정 레지스트리 등)는 서브클래스가 더한다.**

## 3. 합성 + grounding 검증 (메인 외부 체크 · 상시 ON)

수집 결과를 합성하고, **메인이 클레임↔출처 함의를 외부에서 검증**한다 — 수집자 셀프 체크가 아니다. 모든 tier(light 포함).

- **중복 정리 → 합성:** 갈래별 결론을 병합하고 중복·상충을 정리한다.
- 🔒SEALED **grounding (claim↔source 함의 — 상시):** load-bearing 클레임마다 **인용 출처가 실제로 그 주장을 뒷받침하는지** 확인한다(ALCE/CitationAgent식 함의 판정). 안 되거나 오귀속이면 **플래그·강등**한다. 범위는 강도표대로. (모든 tier·light 포함 — 서브클래스가 못 끈다.)
- 이건 "죽은 링크·오인용 거르기"(§6)와 **다르다** — 링크가 살고 인용문이 정확해도 그 출처가 주장을 뒷받침 안 할 수 있다.
- grounding 판정 = 클레임별 `{지지 · 부분지지 · 미지지 · 불확실}` — §5 확신도 태그의 1차 근거.
- **medium+에선 단일 family가 아니다:** 메인이 소스를 오독하면 같은 family라 스스로 못 잡는다 → medium↑에선 §4 cross-family(blind) 리뷰어가 load-bearing 클레임 함의를 **스팟 재검증**한다(2차 방어선).
- **light는 여기서 끝** — grounding 통과분을 출처 포함 요약으로 보고한다(§6). medium↑는 §4 적대 리뷰로 넘긴다.

## 4. cross-family 적대 리뷰 (medium+ · fresh 리뷰어)

grounding을 통과한 **합성 산출물**을 『cross-family(blind) 리뷰어』가 적대적으로 때린다 — 수집 이중화가 아니라 confident-wrong 적출.

- 🔒SEALED **리뷰어 = fresh cross-family(blind) 리뷰어** — 수집자·합성자·메인이 아니다(생산자 ≠ 리뷰어). 주계열과 다른 family여야 confident-wrong·학습 편향이 갈린다(같은 family는 self-consistency로 자기 산출을 무르게 봐 적출 실패). (서브클래스가 same-family로 못 바꾼다.)
- **리뷰 렌즈(공격 표면):** ① 근거 없는 주장 · ② 과장(overclaim) · ③ 오귀속 인용 · ④ 논리 공백 · ⑤ **confident-wrong**(널리 퍼진 오해·틀린 세부를 확신조로) · ⑥ **완전성/누락**(안 다룬 후보·반대 학파·최신 변경·반례·vendor caveat). ⑥은 리뷰어가 **자기 web_search로 누락을 능동 탐침** — 있는 것 비판만이 아니라 없는 것 찾기.
- **누락 렌즈(⑥)는 독립 수집보다 약하다:** 리뷰어는 **보고서에 앵커돼** 있어 잔여 frame lock-in을 완전히 못 벗는다 → omission-critical 주제는 deep의 독립 cross-family 병렬 수집을 백스톱으로 남긴다(강도표 deep 옵션). ⑥은 medium에서 근사할 뿐 대체하지 않는다.
- **주관 판정 금지:** "어느 보고서가 더 잘 읽히나"가 아니라 **"이 주장이 근거로 지지되나 / 어디가 틀렸나"** 만 본다.
- **grounding 스팟 재검증(medium+ · 이중 grounding):** 리뷰어는 적대 렌즈에 더해 load-bearing 클레임 몇 개의 claim↔source 함의를 직접 재확인한다(§3).
- **호출 파라미터:** `sandbox: read-only`, `approval-policy: never`, `config: { "model_reasoning_effort": <강도표대로>, "tools": { "web_search": true } }`.
- **입력 = 합성 산출물 + 출처 + 수집 시 드러난 공백·버려진 후보·불확실 노트.** 리뷰어는 산출물을 봐야 때린다(BLIND는 병렬 수집 축에만). "결론만 회수"(§2)는 수집 결과의 메인 컨텍스트 복원에만 적용 — 리뷰어 누락 렌즈(⑥)는 공백·버려진 후보·기권 노트를 함께 넘긴다(원시 검색 로그 덤프는 회피).
- **출력 = 판정 + findings[]** — 각 = `{클레임/누락 · 결함타입 · 근거/반증 · 심각도}`. verbose 덤프 회수 금지.
- **반박은 반증 출처 강제(evidence-grounded):** 반박(refute) verdict는 **반증 출처를 반드시 동반**한다 — 근거 없는 트집·의심만으론 반박이 아니다(finding까진 되나 verdict는 안 된다). 적대를 과하게 하면 오경보가 늘고 비용도 폭발한다.
- **과신 금지 — 만장일치 ≠ 정답:** 리뷰가 아무것도 못 잡아도 정답 증명이 아니다. 통과는 신뢰도를 올리는 신호지 증명이 아니다 — grounding + 사람 백스톱이 앵커.

### 적대 강도 레벨 사다리 (축 = 리뷰어가 수집자 답에서 독립하는 정도)

독립도가 높을수록 앵커링이 걷혀 적출력·비용이 오른다. **레벨→tier 매핑은 강도표 정본**. **레벨 1(고무도장)은 앵커돼 적출력이 사실상 0이라 쓰지 않는다.** 쓰는 레벨은 2~5:

- **레벨 2 — 검산(읽고 검증):** 답 + 추론체인을 **읽고** "틀렸다 가정"으로 각 홉을 출처로 되짚어 검증, 틀리면 **반증 출처와 함께** 고친다.
- **레벨 3 — 홉 독립 재도출:** 체인을 **안 읽고** 각 홉을 처음부터 재계산해 수집자 답과 대조한다.
- **레벨 4 — 다중 렌즈 + 대안답 랭킹:** 공격을 전문 렌즈(숫자정밀 · 개체혼동 · 날짜/시점 · 출처권위)로 분리해 사냥하고, 답 후보 top-K를 **근거로 랭킹**한다.
- **레벨 5 — blind 재해결:** 리뷰어에게 **질문만** 주고 통째로 다시 연구시켜 두 완성 답을 대조한다 — 사실상 cross-family 독립 수집, 비용 ~2배.

## 5. 종결 + 판정 집계 (메인 · 승자 강요 X)

메인은 grounding 판정(§3)과 (medium+) 리뷰 판정(§4)을 **기계적으로 집계**한다 — 판결이 아니라 규칙 적용이다.

**확신도 = grounding + 리뷰에서 파생(자기보고·합의도 금지):** 근거는 ① grounding `{지지/미지지/불확실}` ② 리뷰가 그 클레임을 때렸나. "수집자 합의도 = 확신"은 폐기(단일 웹을 둘이 긁으면 합의는 연출). 자기보고 %도 무시.

| grounding | 리뷰(medium+) | 독립 교차확증 | 확신도 태그 |
|---|---|---|---|
| 지지 | 통과(못 때림) or 단지 기권 | **있음**(제2 1차자료 / deep 독립 cross-family 수집 합의) | **확실** |
| 지지 | 통과(못 때림) or 단지 기권 | 없음(단일 출처) | **가능성 높음** |
| 지지 | 근거로 반박(counter-evidence) | (무관) | **contested** → 아래 |
| 부분지지 | 통과/약한 지적 | (무관) | **가능성 높음** |
| 부분지지 | 근거로 반박(counter-evidence) | (무관) | **contested** → 아래 |
| 미지지 / 불확실 | (무관) | (무관) | **불확실** → 보류 |

- **확신도 상한 — 만장일치≠정답:** grounding 지지 + 리뷰 통과만으론 **가능성 높음**까지. **확실**은 **독립 교차확증**이 있을 때만. lone-source가 grounding됐고 리뷰가 안 때렸다는 건 반증이 안 나왔다는 뜻이지 독립 확인이 아니다.
- **abstention ≠ contradiction:** 리뷰어(또는 deep의 cross-family 수집자)가 **단지 확인 못 함/기권**한 것은 반박이 아니다 — grounding이 지지하면 채택한다. **반증을 낼 때만** contested.
- **contested 처리 — 메인은 승자를 안 고른다:** load-bearing contested → mode-aware 에스컬레이션(아래). deep에서 병렬 수집을 켰고 두 답이 정면 충돌할 때 **두 family 어느 쪽도 아닌 제3 family가 가용하면** tiebreak 옵션을 쓴다 — 미보유가 기본이라 없으면 contested로 남긴다(same-family tiebreak는 편향이 안 갈려 금지).
- **종료 조건 (loop-until-dry — deep, 범위는 강도표):** (a) 모든 sub-question 커버 (b) 마지막 iteration이 새 load-bearing 공백/불일치를 못 찾음 (c) 모든 클레임이 판정을 가짐. **deep만 loop, light/medium은 단일 패스.** **하드 백스톱(강제):** iteration·검색/도구호출·토큰 상한(수치 = 강도표)에 닿으면 dry가 아니어도 멈추고 **남은 미커버 갈래/공백을 한계 섹션에 명시**한다.
- **금지 안티패턴:** 합의 강요 · 충돌을 어중간한 중간값으로 평균 · same-family tiebreak · 만장일치=증명.

### mode-aware 에스컬레이션 (모드 = 자율 vs 대화 감지)

contested/미지지는 위 집계가 처리한다 — **메인이 판결하지 않는다.**

- **대화(interactive) 모드:** load-bearing `contested`/미지지 → **사용자에게 질문**.
- **자율("진행 쭉해") 모드:** 진행하되 **contested/불확실 태그 유지** + **보고서 쟁점/한계 섹션에 쟁점→판정불가 경위를 로그**(사후감사 sink — 스킬 폴더에 별도 로그 X). contested load-bearing에 의존하는 하류 결정은 플래그. **light는 보고서가 없으므로, 자율 모드에서 load-bearing 미지지/불확실이 나오면 medium으로 승격**한다(escalation-only). (쟁점 로그 ≠ `feedback.md` — feedback.md는 스킬 결함용, 리서치 쟁점은 보고서.)
- **마이너 contested**(결론 영향 없음) → 메인 자율(양 모드 공통).
- **마이너/굵음 게이트 = tier 트리거 재활용**(강도표).

## 6. 산출 분기 (결정권 = 사용자)

- **인용 검증 (별도 pass — 링크 생사·오인용):** load-bearing 인용의 링크 생사·인용문 문구를 대조해 죽은 링크·오인용을 거른다(§3 grounding = claim↔source 함의는 별개 축). 범위는 강도표대로.
- **light** → 보고서 없이 **출처 포함 요약**(grounding 통과 결론 + 확신도 + 출처 + 기권/공백)을 메인이 보고.
- **medium↑** → **출처 단 보고서:** 메타헤더(상태·방법·날짜·확신도 범례) · 발견마다 확신도(§5) · 출처 · **grounding 결과** · **적대 리뷰 결과(적출 결함·반증)** · **쟁점/한계(contested 포함)** · 공백.
- 결정·선택이 필요하면 `AskUserQuestion`으로 선택지 제시(임의 채택 금지). 추천 1개는 맨 앞 "(추천)", 각 옵션에 한 줄 트레이드오프.
- 🕳HOLE(산출 저장) — 저장 위치/포맷·결정 기록(ADR)·후속 작업은 프로젝트 관례에 위임한다. **보고서 저장 경로·포맷은 서브클래스가 지정**(없으면 메인이 사용자에게 위치를 묻는다).

## 7. 설계-결정 모드 (OSS 서베이 → 접근법 선택)

조사가 순수 사실이 아니라 "이 기능을 어떻게 만들까"면 위 절차에 **출력 4개**를 더한다(엔진은 그대로):
- 🕳HOLE(제약 추출원) **제약 먼저(§1 보강):** 프로젝트 제약(기술스택·교체성·규모·LLM제어·라이선스)을 추출해 적는다. **어느 파일에서 추출하나는 서브클래스가 지정**(없으면 사용자에게 제약을 묻는다).
- **by-candidate 팬아웃(§2 변형):** 갈래 대신 후보별로 팬아웃하고 스키마에 `{핵심메커니즘·장점·단점·라이선스·성숙도}`를 추가.
- **제약 적합도 표(§6 보강):** 후보 × 제약 적합도 표를 만든다. (설계 서베이는 종합형 → **기본 medium+**.)
- **거부 후보 → ADR 거부 대안(§6 연결):** 명백히 안 맞는 후보를 사유와 함께 추려 `/adr` 거부 대안으로 넘긴다.

## 가드레일 (앞 절에 없는 금지만)

앞 절에 이미 박힌 규약(same-family 금지·생산자≠리뷰어·근거 없는 반박 금지·grounding 셀프 체크 금지·collector-agreement=confidence 금지·메인 판결 금지·BLIND·과도 스폰 금지·결론만 회수)은 반복하지 않는다. 다른 곳에 안 적힌 금지만:

- **수집 이중화로 회귀 금지** — cross-family를 수집자로 기본 투입하지 않는다. 병렬 수집은 **deep 옵션 한정**(강도표).
- **환각 금지** — 사실/추정 구분, 불확실은 "불확실", 출처 없으면 "지식 기반" 표기.
- **peer groupchat 금지** — 오케스트레이터(메인) 없는 자유 토론 구조는 오류를 증폭시킨다.
