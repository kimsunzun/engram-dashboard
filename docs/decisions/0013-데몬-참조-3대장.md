# ADR-0013: 데몬 참조 3대장 — tmux / Zellij / Mosh

- 상태: 확정 (2026-06, S12 데몬화 대비 참조 선정)
- 관련: CLAUDE.md 참조 구현 · `docs/process/S12-daemonization/` · 클론: `I:\Engram_Workspace\references\`

## 맥락
S12 데몬화(서버가 세션 보유, 클라가 attach = tmux 모델)를 설계·구현할 때 바닥부터 짜지 않고 성숙한 오픈소스를 참조한다. 후보가 여럿(tmux/Zellij/Mosh/ttyd/gotty/wezterm/Screen/ET…)이라 1급 참조의 우선순위를 고정한다.

## 결정
데몬 설계의 1급 참조를 3개로 고정한다 (읽는 순서):
1. **tmux** — 데몬 모델 교본. 세션 수명 분리 · attach/detach · control mode 프로토콜.
2. **Zellij (Rust)** — 구현 참조. 같은 스택이라 패턴을 직접 이식 가능. 자체 IPC · client-server · Windows I/O 코드 포함.
3. **Mosh** — 원격/재연결 견고성(SSP state-sync). 끊김·IP 변경 생존 → 원격·모바일 단계 대비.

## 거부한 대안
- **ttyd / gotty** — PTY→WebSocket 브리지. 전송 스니펫엔 유용하나 *데몬 자체* 참조로는 얕음(전송 참고용 보조로만).
- **wezterm mux** — 이미 운영에 사용 중(Claude Orchestra 기반). 별도 학습 참조 대상이 아님.
- **GNU Screen / Eternal Terminal / abduco·dtach** — tmux와 중복되거나 더 단순. 추가 가치 적음.

## 근거
engram 제약 = Rust + tmux 모델 + 원격/모바일 미래. tmux(모델)·Zellij(Rust 구현)·Mosh(원격 견고성)가 각 축을 **비중복**으로 커버한다.

## 영향 / 불변식
- 데몬 관련 설계 시 이 3개를 우선 참조한다. **차용 = 패턴이지 코드 복붙 아님**(복사 시 라이선스 법무 확인).
- 참조를 추가/교체하면 이 ADR을 갱신한다.
