# ADR-0016: 에이전트 수명 모델 — sid 인스턴스, 저장=살림·삭제=끔, restart 기본 Always, 단순 가드

- 상태: 확정 (2026-06-16, 근거: 사용자 개념 정정 + `/consult` 교차검증 job `20260616-001941`) — **단, restart_policy=Always의 "런타임 자동재시작" 해석은 폐기 (Superseded in part by ADR-0019). 런타임 자동재시작은 없다(크래시=예약 수동 재실행). 부팅 복원·가드 카운터·Failed 영속은 유효(추후 재검토).**
- 관련: ADR-0015(데몬 수명) · ADR-0008(세션 복원 sid) · `agent/profile.rs` · `agent/manager.rs` · `docs/process/S12-daemonization/lifecycle-policy-prd.md`
- 범위: persist 데몬이 든 claude 에이전트의 수명(부팅 복원·런타임 재시작·크래시 가드). **이번 단계=모델+변수 확정, 동작 구현은 TODO.**

## 맥락
claude 에이전트의 "언제 살리고 언제 죽이고 언제 포기하나"를 정해야 한다. 직전 chat에서 개념을 여러 번 잘못 잡아(프로필을 인스턴스와 분리, pause 상태 가정, 안정가동 리셋 등) 정정이 누적됐고, consult 교차검증으로 다듬었다.

## 결정
1. **에이전트 = sid 기반 인스턴스, 영속 저장.** spawn은 sid로 인스턴스를 만들고 그 자체가 저장된다. **저장 존재 = 살려둘 의도.** 되살리기 = 저장된 sid로 `--resume` 재spawn. **삭제가 유일한 "안 살림" 스위치**(저장 제거).
2. **pause/수동정지 상태 없음.** claude는 **살아있거나 죽거나** 둘뿐 — 안 쓰면 idle로 두면 되고 그게 곧 pause다. "잠깐 끔"·"수동 정지" 같은 별도 상태를 만들지 않는다. 끄려면 삭제.
3. **restart_policy 기본 = `Always`.** 죽으면 되살린다(Failed·삭제 제외). 잠깐 멈추려고 kill하는 개념이 없으므로(§2), 멈추려면 삭제한다. ⚠️ **이 "런타임 자동재시작" 해석은 ADR-0019로 폐기** — 런타임에선 크래시=예약(대기) 복귀+수동 재실행(자동 재시작 없음), "죽으면 되살린다"는 **부팅 복원** 맥락으로만 유효.
4. **크래시 가드 = per-agent 단순 카운터.** `consecutive_failures`를 재시작 시도마다 +1, 한도(`restart_max`, 예 5) 초과 시 Failed로 정지. **리셋은 수동 재시작만** — 자동 리셋·안정가동 리셋·슬라이딩 윈도 전부 안 쓴다. 자동 리셋을 안 하므로 "35초 살다 죽는" 폭주 루프도 그냥 5번에 멈춘다. **수동 재시작 = 사용자/LLM의 명시적 spawn 요청**(UI 버튼·커맨드).
5. **Failed = 저장 유지 + 자동 깨우기 suspend.** 콜드부팅을 넘어 유지(자동복원 제외), 수동 깨우기 전까지. **데이터 삭제 아님.** 기존 `auto_restore` 필드를 **"자동 깨우기 on/off"로 재정의** — 정상 true, 가드가 막으면 false(=suspended), 수동 깨우면 true. (별도 의도/관측 분리 변수 불필요 — §1)
6. **부팅 복원 = 저장된 비-suspend 인스턴스 전부 되살림**(데몬 콜드부팅 1회, UI 무관). "UI 열림 → 전체 자동복원"은 금지(ADR-0015 detachable viewer). 단 **명시적 사용자 요청**(특정 에이전트 깨우기·데몬 확보)은 정상.
7. **데몬 폭주 가드 = app-side, 한도 `daemon_respawn_max`(예 3) < 에이전트(5).** 데몬 1회 재기동 = 전체 복원이라 더 빨리 포기. **데몬 수동 재시작 = UI 실행(ensure-daemon 요청)**, 가드 리셋도 UI 실행으로.
8. **전역 튜닝값 = 신규 `daemon-config.json`**(restart_max, daemon_respawn_max 등). `last_restore` → `last_start_at` rename(기록·디버깅용, 리셋 판정엔 안 씀). 가드 카운터·Failed는 **영속**(agents.json).

## 거부한 대안
- **pause/수동정지 상태** — claude는 alive-or-dead, idle이 곧 pause라 불필요.
- **restart 기본 Never** — "항상 살아있게"라는 제품 방향과 어긋남(사용자가 Always 선택). (consult는 마이그레이션 안전성으로 Never 권고했으나 사용자 override.)
- **안정가동 리셋(T초 생존 시 카운터 0)·슬라이딩 윈도** — 과방어. 수동-리셋-only면 폭주가 자연히 멈춘다. claude 워크로드(정상 1프롬프트 1~2분)엔 stable_secs=30s가 오히려 위험(35초 루프가 매번 리셋돼 안 멈춤 — consult Gemini 적출).
- **전역 가드 카운터** — 고장난 1개가 멀쩡한 에이전트까지 멈춤. per-agent라야.
- **was_running 관측 추적 / auto_restore를 "의도 vs 관측"으로 분리** — sid-instance 모델에선 불필요(저장=살림, 삭제=끔). 초기에 이 분리를 검토했으나 개념 정정으로 폐기.
- **UI 열림 = 복원 트리거** — 안티패턴(데몬=두뇌가 뷰어에 종속, consult 6개 시스템 중 0개 채택). 명시적 사용자 요청만 허용.
- **데몬 유휴 자동종료** — persist-until-kill(ADR-0015).

## 근거
- 사용자 개념 정정: 에이전트=sid 인스턴스, 저장=인스턴스 자체, 되살리기=재spawn, 삭제=저장 제거, pause 없음(alive/dead).
- consult 교차검증(GPT·Gemini·Claude opus 블라인드 + judge): 의도≠관측 분리·Failed 영속·데몬가드×에이전트가드 카운터 휘발·daemon-config.json·exit-kind(Killed 재시작 제외) 합의. E(가드 리셋) 함정은 Gemini 단독 정량 적출.

## 영향 / 불변식
- **끄기 = 삭제뿐.** kill은 잠깐 멈춤이 아니다(Always면 되살아남). 잠깐 안 쓸 거면 idle로 둔다.
- **Failed·가드 카운터는 영속** — 데몬 재부팅이 카운터를 0으로 날리면 무한루프(데몬 재기동→복원→즉사→…)가 가드를 우회한다. 그래서 agents.json에 저장.
- **수동 kill/shutdown은 Always여도 재시작 안 함** — 안 그러면 끌 수 없는 에이전트가 된다(exit-kind로 `Killed` 구분 필요, 동작 단계).
- **런타임 죽음 관측 훅**(재시작 트리거 경로)은 코드에 없음 → 별도 결정: **제안 ADR-0017 = `done_rx` supervisor**(기존 완료 채널 재활용, ADR-0001/0005 무손상). 승인 대기.
