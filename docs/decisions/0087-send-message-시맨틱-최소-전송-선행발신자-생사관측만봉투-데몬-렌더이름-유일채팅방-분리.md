# ADR-0087: send_message 시맨틱 — 최소 전송 선행·발신자 생사=관측만·봉투 데몬 렌더·이름 유일·채팅방 분리

- 상태: 확정 (2026-07-19, 근거: OSS 서베이 8종 + Orca 실소스 검증 + 사용자 단계별 승인 + 리뷰 2인 적대 3라운드 양측 PASS + QA standard 9게이트)
- 관련: Amends ADR-0086 (스텝 사다리 ②③④ 분할 순서 → 2-min 최소 전송 일괄 선행 + SQLite 메일박스 보류(사용자 학습 후 재개)) · ADR-0086(듀얼 입구·토큰 인증 — 본체 유효) · ADR-0007(epoch — 이름은 AgentId에 묶임) · ADR-0006(락 규율 — 생존 게이트 기각 근거) · ADR-0004(백엔드 지식 격리 — env 주입 위치) · 구현 커밋 `e05bd83` · Amended by ADR-0088 (사다리 순서(포맷 스파이크 → 배달 정확성 검증 선행으로 재편) + 봉투 판정 축(위조내성 → 이스케이프로 이관, 포맷은 가독성·준수 한정))

## 맥락
스텝 2(send_message) 착수 전 설계 논의를 확장했다. 쟁점: ① 에이전트끼리 "서로 알아먹는" 층(주소·봉투·답장 규약)이 ADR-0086에 미설계 ② SQLite 메일박스는 사용자가 학습 후 결정하길 원함 ③ 스텝을 잘게 나누면 실측(발화 테스트)이 늦어짐. OSS 조사 2회(웹 서베이: mcp_agent_mail·Claude Code Agent Teams·Claude Swarm·AutoGen·CrewAI·A2A·OpenAI Agents SDK·LangGraph / Orca(stablyai, 21.4k★)는 커밋 고정 permalink로 실소스 검증)를 근거로 사용자가 아래를 단계별 확정했다.

## 결정

1. **사다리 재편(0086 부분 개정):** 스텝 ②③④(send 툴 / 메일박스 / 배달 분할)를 **"2-min 최소 전송" 일괄 선행**으로 교체 — 접수(툴+Validator+ACK)·즉시 배달(write_stdin)·CLI 입구까지 최소형으로 한 번에(장부·idle 게이트·봉투 확정·이름 suffix·그룹은 스코프아웃, 자리만 예약). SQLite 메일박스는 **사용자 학습 후 재개**. 구현 완료 = `e05bd83`. 다음 = **봉투 포맷 실측 스파이크**(XML 태그 vs Orca식 배너, 모델 사다리 haiku→sonnet→sonnet 불량 시 opus, 합격 기준선 = sonnet — haiku는 하한 탐지용).
2. **발신자 생사 ≠ 배달 게이트 (관측만):** 메시지 유효성은 **작성 시점 인증**(입구 토큰 검증)으로 성립하며, 발신자가 이후 죽거나 epoch 회전해도 배달을 막지 않는다. `is_identity_live` 조회는 남기되 **기록용 warn 로그만**(제품 고도화 때 재고 여지 — 사용자 지시 "기록용으로 남겨"). 근거: "결과 보내고 셧다운"은 핵심 패턴(Orca worker_done 동형 — 유언이 가장 중요한 메시지)이고, 미래 메일박스 시맨틱(장부 기록=커밋, 이후 발신자 사망은 unsend 아님)과 정합.
3. **봉투 비대칭 — 생산은 기계 강제, 소비는 모델 친화:** 에이전트는 typed 송신(`{to, body}` 툴콜/CLI 인자)만 하고 **봉투는 데몬 렌더러가 조립**한다(오타 0 + 본문 속 가짜 봉투 태그 이스케이프로 위조 차단 가능). 봉투 방향 = **XML 태그**(claude가 하네스 `<system-reminder>`류로 훈련된 네이티브 구획 관례 + 여러 통 배치 시 경계 견고) — **최종 확정은 스파이크 실측**. 답장 줄은 **조건부 어투**("답장이 필요하면: …" — 기계적 방법은 항상 내장하되 답장 의무를 암시하지 않음)로 확인-답장 폭풍을 막는다.
4. **이름 규칙:** 주소 = 사람이 읽는 에이전트 이름(조사 전 OSS에서 UUID 주소 채택 0건). 라이브 에이전트 간 **유일성 강제**(충돌 시 `-2`,`-3` 자동 suffix), **데몬 가동 중 suffix 번호 재사용 금지**(죽은 이름 기억에 의한 오배달 차단 — 죽은 이름은 "수신자 없음" 에러가 정답), 이름은 **AgentId에 묶여 epoch 교체에도 유지**. (미구현 — 현 최소판은 정확 매칭 + AMBIGUOUS 에러, suffix 기계는 후속.)
5. **배달 정책 = 데몬 스위치:** idle 게이트(StreamJson `MessageDone` 기반 턴 상태) + **기본 = 배치 주입**(밀린 봉투 일괄, Orca 동형) / **직렬 모드 옵션**(에이전트별 전환) — 아키텍처가 아니라 정책 노브로 두어 실트래픽 A/B로 판정. (미구현 — 현 최소판은 즉시 push.)
6. **교정형 에러 ACK:** 검증 실패는 선언이 아니라 교정 정보를 싣는다(예: RECIPIENT_NOT_FOUND에 live 이름 목록 hint) — 같은 턴 자기교정 유도, 별도 로스터 툴 없이 부분 해소.
7. **채팅방 = 메일과 별도 시스템(완전 추후):** 공유 append-only 텍스트가 곧 방이고, **멤버별 읽음 커서**(카톡식 안읽음 수 파생), 메시지마다 깨우지 않고 **트리거 단위 캐치업**(멘션 즉시/누적·침묵/idle), 슬롯에서 사용자 관전·참여(추후 원격 참여자). 용법 구분: **메일 = 단발 용건, 방 = 왕복 예상 대화**(사람의 "메일 답답하면 메신저" 전환 개념 포함 — 이 판단 기준을 에이전트 규약에도 실음). 착수 시 별도 PRD/조사 라운드(발화 폭주 규율이 핵심 난제 — 성공 OSS 부재).
8. **브로드캐스트/그룹 = 추후, 자리만:** `to`의 `@…`(그룹군·all — 데몬이 해석해 수신자별 fan-out, Orca 그룹 주소 동형)는 GROUPS_NOT_SUPPORTED 에러로 예약.
9. **(결정 아님 — 고민 중 기록)** 멀티 데몬 확장 시 MCP = 원격/데몬 간 후보, CLI = 로컬 내부 후보 — `ControlIngress` seam 위 정책 배정이라 아키텍처 변경 없이 후일 결정 가능.

## 거부한 대안
- **봉투를 에이전트가 직접 작성** — 마커 스파이크 컴플라이언스 66%(ADR-0086)가 보여준 생산측 실패(따옴표·nonce 오기재). 구조는 읽기는 잘하나 만들기에서 깨진다 → 생산은 typed 스키마 강제.
- **pull 방식 수신(mcp_agent_mail식 fetch_inbox)** — 에이전트가 확인을 잊거나 폴링 루프를 돎(Orca가 프리앰블에 폴링 금지문을 둘 정도). push(입력 주입)는 수신 협조 0.
- **UUID 주소** — LLM 오기재 표면 + 조사 전 OSS 채택 0건.
- **발신자 생존 거부 게이트(SENDER_REVOKED)** — 유언 메시지 유실. 완전 봉쇄는 락 쥔 채 I/O(ADR-0006 위반)라 불가능하고, 리뷰어 대립(Codex "잔여 race" vs deep "수용")을 사용자가 "생사는 게이트가 아니다"로 프레임 전환해 해소.
- **답장 줄 명령형 어투** — 모든 봉투에 답장을 유발해 확인-답장 폭풍(턴 낭비).
- **이름 suffix 번호 재사용** — 다른 에이전트 컨텍스트에 남은 죽은 이름이 새 에이전트에게 오배달.
- **배치/직렬을 지금 확정** — 인공 시나리오 실측은 대표성이 없고, 스위치 비용이 0이라 실트래픽 A/B로 미룸.

## 근거
- **OSS 수렴 2건:** "완성형 답장 명령 내장" 봉투(Orca `[Reply: orca orchestration reply --id … --from …]` 실소스 + mcp_agent_mail ack_required/frontmatter)가 독립 수렴 — 조사 최고 확신 패턴. 주소=이름·역할설명 1줄 노출도 전 OSS 공통.
- **Orca 실소스 검증(커밋 고정):** formatter(배치 배너)·preamble(규칙을 CLI 예시 바로 위 배치 — "LLM은 예시에 앵커링") ·messages 스키마(sequence/delivered_at/thread_id 실재, `read`는 **불리언** — 기존 조사보고서의 read_at은 오류로 정정) · 그룹 fan-out = 수신자별 row.
- **리서치 프로세스 교훈(별건 기록):** 웹 수집자가 Orca를 못 찾은 원인 = 앵커(repo URL) 미전달 + blind 오적용(조회형 과제) — "검증형 조사엔 보유 앵커 동봉, blind는 판단 수집에만" 룰 채택. UNVERIFIED 정직 신고는 설계대로 작동.
- **게이트:** 리뷰 2인 적대(deep=reviewer-deep + blind=Codex) 3라운드 → 양측 PASS · QA standard 9게이트 PASS(core 224·daemon 164·protocol 50·discovery 44·vitest 621·격리 0).

## 영향 / 불변식
- **메시지 유효성 = 작성 시점 인증** — 배달 시점 발신자 생사로 거부하는 코드를 다시 넣지 않는다(`ingress.rs`의 사용자 결정 주석이 앵커). 관측 로그(warn)는 유지.
- **봉투 렌더링은 데몬 단일 지점**(`wrap_message` 1함수) — 포맷 변경 = 렌더러 수정 + 발화 재실측이지 에이전트 마이그레이션이 아니다. 에이전트 계약은 `{to, body}`뿐.
- **least-privilege·from=토큰 파생·토큰 수명(AgentId, epoch)** — ADR-0086 불변식 그대로 승계.
- **`to` 파싱은 확장 지점** — `@` 접두 예약(그룹), 이름/ID 이중 수용(ID 정확 매칭 우선 — 이름이 남의 UUID인 가로채기 차단).
- 미구현 결정(이름 suffix·idle 게이트·배치 스위치)은 이 ADR이 사양 정본 — 구현 시 재론하지 않는다.
