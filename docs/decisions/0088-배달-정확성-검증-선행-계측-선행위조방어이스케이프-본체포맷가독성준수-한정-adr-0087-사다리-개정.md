# ADR-0088: 배달 정확성 검증 선행 — 계측 선행·위조방어=이스케이프 본체·포맷=가독성/준수 한정 (ADR-0087 사다리 개정)

- 상태: 확정 (2026-07-19, 근거: /research(OSS·문헌 서베이 3갈래 + grounding) + Codex cross-family 적대 리뷰 2회(테스트군 BLOCK·계획 BLOCK, findings 20+) + 사용자 단계별 승인)
- 관련: Amends ADR-0087 (사다리 순서(포맷 스파이크 → 배달 정확성 검증 선행으로 재편) + 봉투 판정 축(위조내성 → 이스케이프로 이관, 포맷은 가독성·준수 한정)) · ADR-0086(듀얼 입구·즉시 push relay — 본체 유효) · ADR-0006(락 규율 — 즉시 push의 근거) · ADR-0012(모듈 격리 하네스 — 스파이크 하네스가 이 결을 탐) · Amended by ADR-0091 (사다리 순서 + 단계4 분리 (포화를 포맷-수용부 뒤로 재배치; 이스케이프 유보))

## 맥락
ADR-0087은 스텝 2-min(듀얼 입구 최소 전송) 완료 후 다음을 **봉투 포맷 실측 스파이크**(XML vs Orca식 배너, sonnet 기준선)로 지목했다. 그 스파이크의 테스트군을 설계하려고 /research(입구 비교 / 봉투 포맷 / 행동 케이스 3갈래)를 돌리고, 그 산출물을 Codex(비주도 family)에 두 차례 적대 리뷰 걸었더니 **둘 다 BLOCK**이 나왔다. 핵심 지적:

1. **순서가 틀렸다.** 현재 배달은 **즉시 push**(`write_stdin` = 1 write=완결 유저턴 1개)이고 **idle 게이트·배치·큐가 미구현**이다(ADR-0087 사다리 ③④가 아직 자리만 예약). 그래서 수신 에이전트가 뭔가 하는 도중 메시지가 꽂힐 때 유실·잘림·중복·타이밍 오류가 나는지 **아무도 검증 안 했다**. 이 파이프 건전성이 미증명인 채 "어느 봉투가 잘 읽히나" 판정을 내면 결과 해석이 불가능하다.
2. **관측 가능성 공백.** relay 로그 + 어시스턴트 트랜스크립트만으로는 "에이전트가 메시지를 놓쳤다"가 *전송 실패*인지 *모델 무시*인지 구별할 수 없다 — write 경계 계측 없이는 모든 배달 판정이 모호하다.
3. **위조 방어를 봉투 문법으로 재는 건 착각이다.** 봉투 문법(XML/배너)이 무엇이든 발신 에이전트가 **본문 텍스트로 그 문법을 흉내 내면** 뚫린다(ChatInject: 본문 속 역할/시스템 태그 위장이 순진한 주입보다 ASR가 3배↑). 진짜 방어 레버는 문법이 아니라 **데몬이 본문 속 봉투 경계 흡사 문자열을 무력화(이스케이프)**하는 것이다.

이 세 지적은 ADR-0087의 스텝 사다리 순서와 봉투 스파이크의 판정 축을 바꾼다 — 나머지 결정(발신자 생사=관측만·봉투 비대칭·이름 규칙·채팅방 분리 등)은 유효하므로 **부분 개정**한다.

## 결정

1. **사다리 재편(0087 부분 개정) — 배달 정확성 검증을 포맷 스파이크 앞에 삽입.** 기존 ③(포맷 스파이크)·④(배달 고도화=idle 게이트·배치) 사이에 **"배달 정확성 검증"을 선행 스텝**으로 넣는다. 즉시-push·idle 게이트 부재 하에서 메시지가 유실·중복·잘림·타이밍 오류 없이 배달되는지, 컨텍스트 포화 시 오동작이 없는지를 계측+하네스로 실측한 **뒤에야** 포맷 스파이크로 간다. 이 검증 결과가 곧 idle 게이트(구 ④) 필요성의 실측 근거다(③↔④가 엉켜 있었음을 인정하고 순서를 바로잡음).
2. **b 슬라이스로 증분(사용자 결정).** 한 번에 다 짓지 않는다: **(0) 계측 → (1) 배달 정확성 core**(상태 매트릭스: 텍스트/툴인자 생성 중·툴 실행 중·권한/결과 대기·수신자 자체 send 중·self-send + exact-once 버스트 2/10/100통 + 수명 경쟁 kill/재시작 + PTY 경계 64KiB±1·멀티바이트) **→ (2) 컨텍스트 포화 축소 요인설계**(길이 × 필러종류 × 봉투위치 × 지연사용 × compaction 전후, 판정=지속 처리). 이 1차가 통과해야 (3) 입구 비교 + (4) 포맷 스파이크로 넘어간다. 파이프가 안 건전하면 비싼 포맷 런(50+)을 낭비하지 않기 위함.
3. **위조 방어 = 이스케이프(봉투 문법 아님).** 봉투 문법 선택은 위조 내성을 만들지 못한다 — 진짜 방어는 `wrap_message`에서 본문을 감싸기 전에 봉투 경계 흡사 문자열을 무력화하는 것. 이스케이프를 **포맷 스파이크에 방어 본체로 함께 실측**하되(포맷별 이스케이프 규칙 + breakout 코퍼스: 닫는 태그·구분자·bidi·zero-width·homoglyph 발신자명), **포맷 대회 자체의 판정 축은 "가독성·준수"로 한정**한다. 위조 내성을 포맷 우열로 주장하지 않는다.
4. **계측 선행(관측 가능성 전제조건).** 어떤 배달 판정도 write 경계 계측 없이는 성립 못 하므로, `write_stdin` 경계 계측(write 시작/완료·바이트수/에러 + msg_id 상관)을 **모든 배달 테스트의 전제조건으로 먼저 구현**한다. "놓침" 분류 전에 전송 완결 여부를 데몬이 증거로 남긴다.
5. **하네스 기본 요건 — 재현성·안전·통계 사전등록.** 스파이크 하네스는 (a) 재현성 핀(해석된 모델 ID·CLI 커밋·데몬 커밋·설정·시스템프롬프트 고정), (b) 안전 샌드박스(격리 워크스페이스·비밀 없음·네트워크 차단·최소 권한 툴·스폰/메시지 하드 상한 — 1i의 `$()`·백틱·전파 케이스가 실제 명령 실행/전파 가능하므로 강제), (c) 통계 사전등록(셀 배분·최소효과·신뢰구간·반복실패 임계·holdout)을 갖춘다.

## 거부한 대안
- **포맷 스파이크 선행(우리가 하려던 것, ADR-0087 원안)** — Codex BLOCK: 즉시-push 파이프 건전성이 미증명인 상태의 포맷 판정은 해석 불가. 배달이 잘리거나 뒤섞이면 포맷 런을 통째 다시 짜야 해 비싼 실 스폰이 낭비된다.
- **봉투 문법으로 위조 방어(F1/F2 우열로 스푸핑 내성 판정)** — 본문이 봉투 문법을 흉내낼 수 있어(ChatInject 실측 ASR 상승) 문법은 방어 레버가 아니다. 방어는 이스케이프(배치·인코딩)로만 성립.
- **풀 리거 일괄 구현(a안)** — 파이프 미검증 상태에서 전 단계(입구·포맷·주입내성 전부)를 한 번에 지으면 1차 결과에 따라 재작업 위험이 크다. 최소(c안)는 재구성 취지(검증 먼저)를 반만 살려 배달 core를 못 덮는다. → 중간 b 슬라이스.
- **즉시 ack = 배달 성공 판정** — Codex: 즉시 수신 표시는 지속 처리를 증명 못 한다. 판정은 지연 회수(나중 turn에서 sender/id/body 회상)·지연 답장 정확성·원과제 완주·compaction 생존으로 한다.
- **relay 로그 순서 = 배달 순서 오라클** — relay 로그 순서가 실제 PTY 수용 순서에 선행할 수 있어 무효. 데몬 admission sequence를 오라클로 부여하고 수신 입력 확인을 포착한다.

## 근거
- **리서치 grounding(확정/가능성 높음):** long-context instability(arXiv 2512.02445 — ~100K 토큰 넘어가면 주입 처리 저하·불안정 거부) · ChatInject(arXiv 2509.22830 — 본문 내 chat-template 위장 ASR 15.13%→52.33%) · Anthropic 권고(untrusted 콘텐츠는 tool_result 배치 + JSON 인코딩). Orca formatter·mcp_agent_mail 답장 구조는 1차 소스(핀 커밋) 직접 검증.
- **적대 리뷰(BLOCK×2):** 테스트군 리서치 산출물 리뷰에서 오귀속 3건 적발(자기교정 85% 수치 = 무관 논문 / OSS 수렴 2건 = mcp_agent_mail은 별도 툴로 답장, 내장 명령은 Orca 단독 / Spotlighting 압축 수치 = datamarking 것). 계획 리뷰에서 순서 오류·관측 가능성 공백·교락·안전 공백 적출. 리뷰어 = Codex(GPT family, effort high, codex exec 백그라운드).
- **리서치 프로세스 정정:** ADR-0087 근거 절의 "OSS 수렴 2건" 문구는 위 오귀속 ②에 해당 — 내장 답장 명령의 독립 수렴은 성립하지 않고 Orca 단독 선례다(이 ADR이 그 사실을 정정 기록; 0087 결정 자체는 봉투를 스파이크가 확정하므로 영향 없음).

## 영향 / 불변식
- **배달 건전성 미증명 상태에서 포맷 판정 금지** — 봉투 포맷 스파이크(구 스텝 3)는 배달 정확성 검증(신규 선행 스텝) 통과가 게이트다. 다음 세션이 순서를 되돌려 포맷부터 재려 하면 이 ADR이 앵커.
- **위조 방어는 `wrap_message` 이스케이프에 있다** — 봉투 문법 변경은 가독성·준수 실험이지 위조 방어 변경이 아니다. 이스케이프 제거·문법으로 방어 대체 금지.
- **계측은 배달 테스트의 전제** — write 경계 계측 없이 "놓침"을 분류하지 않는다.
- **하네스 안전 요건은 강제** — 격리 워크스페이스·네트워크 차단·스폰 상한 없이 주입/셸 확장 케이스(1i·전파)를 실 에이전트에 돌리지 않는다.
- **즉시 push relay(ADR-0086)·발신자 생사=관측만(ADR-0087)·봉투 비대칭·이름 규칙·채팅방 분리 = 그대로 유효** — 이 개정이 건드리지 않는다(부분 개정 범위 = 사다리 순서 + 봉투 판정 축 2개 조항뿐).
