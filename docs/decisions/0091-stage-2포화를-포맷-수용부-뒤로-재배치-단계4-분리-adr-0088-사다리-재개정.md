# ADR-0091: Stage 2(포화)를 포맷-수용부 뒤로 재배치 + 단계4 분리 — ADR-0088 사다리 재개정

- 상태: 확정 (2026-07-20, 근거: ADR-0090 Stage 2 파일럿 발견(정책 격리) + 결과보고서 + 사용자 재배치 결정)
- 관련: Amends ADR-0088 (사다리 순서 + 단계4 분리 (포화를 포맷-수용부 뒤로 재배치; 이스케이프 유보)) · ADR-0090(파일럿 실행 설계 — 이 개정의 근거 발견) · ADR-0086(듀얼 입구·즉시 push — 본체 유효)

## 맥락
ADR-0088 사다리는 **배달 정확성(포화 포함) → 그 다음 포맷**이었다. 그런데 Stage 2(컨텍스트 포화) 파일럿(ADR-0090)에서 정책-준수 claude가 **인스트럭션을 담은 에이전트 간 메시지(`[message from ...]` 봉투)를 프롬프트 인젝션으로 판정·격리**했다(트랜스크립트 thinking 트레이스 직접 확인, 다수 런 재현). 핵심: 배달(바이트)은 정상인데 **수용**이 안 된다 — 에이전트가 메시지를 아예 처리하지 않으니, 포화든 아니든 잴 대상이 없다.

즉 포화를 측정하려면 **"어떤 프레이밍이면 정당한 메시지가 수용·처리되나"(포맷-수용부)가 선행조건**이다. 사다리상 포화(구 단계2)가 포맷 작업(구 단계4의 일부)에 의존한다 — ADR-0088이 가정한 순서가 뒤집혔다. 이 발견은 사다리 순서와 단계4 정의를 바꾸므로 ADR-0088을 **부분 개정**한다(나머지 조항은 유효).

## 결정
1. **포화 재배치.** 포화(구 단계2)를 포맷-수용부 뒤로 옮긴다. 새 순서: **포맷-수용부 → 포화.** 포화 본런은 "수용되는 메시지" 기준선이 선 뒤에만 의미가 있다.
2. **구 단계4(봉투 포맷 + 이스케이프)를 두 갈래로 분리:**
   - **(4a) 포맷-수용부** — 정당한 에이전트 간 메시지가 인젝션으로 격리되지 않고 수용·처리되는 프레이밍/배치(가독성·준수·**수용**). **critical path로 승격 — 다음 착수.**
   - **(4b) 이스케이프(위조 방어)** — 악의적 본문의 봉투 경계 탈출 차단. **유보(우선순위 낮음, 사용자 결정 2026-07-20).**
3. **입구 비교(구 단계3)는 순서 유연** — 블로킹 아님. 아무 때나.
4. **ADR-0088 나머지 조항 유효** — 판정 기준(지속 처리·즉시 ack 아님)·배달 시맨틱·이스케이프=위조방어 본체·하네스 안전 요건·계측 선행은 그대로.

## 거부한 대안
- **ADR-0088 원 순서 유지(포화 먼저)** — 수용되는 메시지가 없어 포화 측정 대상 자체가 부재. 배달은 정상이나 에이전트가 처리를 안 한다.
- **지표를 태스크 채널 회수로 바꿔 포화를 지금 측정(갈림길 a)** — 유효하고 즉시 가능하나, "진짜 에이전트 간 메시지"가 아니라 **태스크 채널**의 포화를 재게 된다(측정 대상이 다름). 사용자가 진짜 메시지 거동 측정을 원해 재배치를 택함.
- **이스케이프(4b)까지 지금 앞당김** — 포맷 미확정 상태에서 이스케이프 규칙은 봉투 구분자에 의존해 재작업 위험(CLAUDE.md §0 "고비용·불확실 = 껍데기만"). 유보.

## 근거
- **ADR-0090 Stage 2 파일럿 발견(확실):** 정책-준수 claude(`claude-sonnet-4-6`)가 인스트럭션 담은 봉투를 인젝션으로 격리 — codeword/sender/id 회수 전부 실패, 사전정당화 문구는 역효과. 상세 = `docs/process/S17-llm-control-surface/stage2-context-saturation-pilot-findings.md`.
- **사용자 재배치 결정(2026-07-20):** "포화 이상현상은 나중, 포맷 먼저" + 이스케이프 유보 + "진짜 메시지 거동을 재고 싶다".

## 영향 / 불변식
- **다음 착수 = 포맷-수용부(4a) 스파이크.** 포화 본런은 그 뒤. 다음 세션이 순서를 되돌려 포화부터 재려 하면 이 ADR이 앵커.
- **포맷-수용부 실측은 개인계정 의존** — 조직 managed-settings 보안 규칙(도구 콘텐츠 속 지시=데이터)이 조직 계정 claude의 수용 판정을 오염한다. 조직 계정에선 모델-레벨 수용을 깨끗이 못 잰다(집/개인 계정 필요). 이 격리가 조직 규칙 때문인지 모델 훈련 때문인지도 그 실측에서 갈린다.
- **이스케이프(4b)는 유보이나 로컬 모델 도입 시 필수로 승격** — 로컬 모델은 인젝션 저항 훈련이 약해 시스템-층(데몬 `wrap_message`) 이스케이프 없이는 뚫린다. 유보 = 지금 안 함이지 불필요가 아님.
- **판정 = 지속 처리 유지**(ADR-0088). 즉시 ack를 수용 성공으로 쓰지 않는다.
