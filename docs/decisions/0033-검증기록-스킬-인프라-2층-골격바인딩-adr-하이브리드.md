# ADR-0033: 검증·기록 스킬 인프라 — 2층 골격+바인딩 + adr 하이브리드

- 상태: 확정 (2026-06-26, 근거: research(adr-tooling) + dogfood review/qa)
- 관련: CLAUDE.md "★ 구현 실행 규약 ★"·"★ 설계 결정 기록 (ADR) ★" · `.claude/skills/{research,review,qa,adr}/` · `scripts/adr.mjs` · ADR-0031(검수 체계) · step-log

## 맥락
개발 스텝(PRD→TRD→코드→검증)을 굴릴 재사용 스킬 셋이 필요했다. 과거 `tp`/`tr`(통짜 오케스트레이터)는 묻어온 AutoFlow로 못 써서 폐기됐고, 검수 체계는 ADR-0031로 "opus + Codex 2자 적대 리뷰"로 이미 정해졌다. 남은 결정 = 이 스킬들을 **어떻게 구조화**하고, 특히 ADR 기록처럼 기계적 비중이 큰 스킬을 어떤 형태로 짤지였다.

## 결정
research/review/qa/adr 4종을 다음으로 고정한다:
- **2층 구조** — 범용 골격(`references/flow.md`, 스택 흔적 0) + 프로젝트 바인딩(`references/bindings/<project>.md`). 단 **research는 범용-only**(프로젝트 실명령이 없어 바인딩 불필요 — 통합은 CLAUDE.md에서). **review/qa/adr만 바인딩을 가지며**, 다른 프로젝트는 그 바인딩만 추가해 골격 재사용.
- **축(axis)** — research=강도(light/medium/deep) · review=단계(prd/trd/code/doc)×강도 · qa=강도(quick/standard/full) · adr=**오퍼레이션**(new/supersede/lint).
- **adr만 결정적 스크립트 하이브리드** — 서기 작업(채번·인덱스 재생성·supersede 양방향·lint)은 `scripts/adr.mjs`(결정적), 전체/부분 폐기 판단·본문 prose는 스킬(LLM).
- **자기개선 피드백 루프** — 스킬 사용 중 발견한 결함은 그 자리서 안 고치고(오판 위험) `feedback.md`에 누적 → 관련 주제 재등장 시 사용자 승인 하에 반영.
- **research가 prior-art 흡수** — 설계-결정 모드(OSS 서베이 → 제약 적합도 표 + 거부후보→ADR)를 research에 더하고 prior-art 스킬은 폐기.

## 거부한 대안
- **통짜 오케스트레이터(tp/tr식)** — 거대·불투명, 이미 한 번 실패한 전철. 스킬 조합 + CLAUDE.md 규약으로 대체.
- **단일층(스킬에 스택 하드코딩)** — rot + 타 프로젝트 재사용 불가. 2층으로 골격을 스택-중립 유지.
- **adr 순수 스킬(LLM이 채번·인덱스·supersede 전부)** — `/review code`에서 **데이터 손상 실증**(양방향 한쪽 누락·번호 충돌·복합 상태 거짓 lint). 그래서 결정적 부분을 스크립트로 내림.
- **adr 순수 스크립트** — 전체 vs 부분 폐기 판단·본문 prose는 기계가 못 한다. 그래서 하이브리드.
- **adr에 "강도" 축(다른 스킬과 평행)** — ADR 기록은 결정적이라 "대충/철저히" 깊이 트레이드오프가 없다. 의미 있는 축은 오퍼레이션.

## 근거
- **research(`docs/research/adr-tooling-hybrid-research-2026-06-26.md`)** — adr-tools가 thin 하이브리드 레퍼런스, "기계(서기)/판단 경계"가 업계 전반(2015~2026) 일관(Claude+Codex 수렴). 부분 폐기는 Amends 양방향 링크 관습.
- **dogfood** — review/qa로 각 스킬 검증. 특히 adr은 `/review code` FIX(데이터 손상 6+건) → 수정 → qa실측 PASS(실데이터 32 ADR 무손실, index --write idempotent).

## 영향 / 불변식
- 스킬 = **범용 골격(스택 0)** + (review/qa/adr) **바인딩**. **research는 바인딩 없는 범용-only**(프로젝트 실명령 없음). 새 프로젝트는 바인딩 파일만 추가한다(골격 수정 금지).
- adr 기계 작업의 단일 출처 = `scripts/adr.mjs`. LLM이 손으로 번호·인덱스·양방향 링크를 만들지 않는다(1차 초안이 그래서 깨졌다).
- **검증 상태:** 이 스킬들이 단일 모델 대비 실제로 나은지는 아직 대조검증 전 = "근거 있는 가설". 실사용 backtest로 입증 필요.
