# ADR-0006: 락 순서 규율 (sessions → 내부)

- 상태: 확정 (S1 LLD §10, dr26 리뷰 포착)
- 관련: CLAUDE.md 핵심 불변식 · `manager.rs::get_session` · `output_core.rs::emit`

## 맥락
`sessions` 맵 lock과 session 내부 lock(core/status)을 동시에 보유하면 다른 경로와 lock 순서가 엇갈려 데드락이 난다.

## 결정
- **sessions RwLock은 조회 전용** — `Arc<AgentSession>`을 clone하고 lock을 **즉시 해제**한 뒤에야 내부 lock(core/transport)을 취득한다.
- **status lock 보유 중 외부 호출 금지** — `finish`/`enter_exiting`는 status lock을 해제한 뒤 status_changed를 발행.
- **emit의 send** — subscribers를 clone한 뒤 lock 미보유 상태로 send.

## 거부한 대안
- **lock 중첩 보유** (sessions lock 잡은 채 내부 접근) — 경로마다 순서가 달라지면 데드락. 가장 흔한 동시성 함정.

## 근거
dr26(fable) LLD 리뷰가 transition race·중첩 lock 위험을 실결함으로 포착. 단방향 락 순서로 교착 가능성 제거.

## 영향 / 불변식
- 모든 manager 메서드는 `get_session`(Arc clone + 즉시 해제) 패턴을 따른다.
- 신규 코드가 sessions lock 보유 중 session 내부를 만지면 이 ADR 위반.
