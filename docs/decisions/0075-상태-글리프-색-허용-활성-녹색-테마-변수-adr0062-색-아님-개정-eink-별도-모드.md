# ADR-0075: 상태 글리프 색 허용(활성=녹색, 테마 변수) — ADR-0062 "색 아님" 개정 (e-ink 별도 모드 분리 전제)

- 상태: 확정 (2026-07-13)
- 관련: ADR-0062(5-glyph 어휘 — 색 아닌 모양) · `src/styles/theme.css`(--status-running) · `src/components/agent/statusGlyph.ts`(statusGlyph/statusGlyphColor) · `AgentList.tsx`

## 맥락

ADR-0062 는 상태를 색이 아닌 "모양"으로 표현하기로 결정했고, 색을 배제한 **유일한 근거는 e-ink(흑백)에서 색이 소실**되는 것이었다(§32). 사용자가 활성(Running) 에이전트 글리프를 녹색으로 원하며, e-ink 를 별도 모드로 분리할 예정이라고 결정.

## 결정

색을 모양에 **부가**(대체 아님)한다 — 활성(Running)만 녹색(theme var `--status-running`), 그 외는 muted(현행). 색 리터럴 금지·변수-only 유지(ADR-0062 §44): 녹색 값은 theme.css 가 소유하고, e-ink 블록은 `--status-running: var(--text-muted)` 로 중립화해 e-ink 에선 모양만 남긴다. statusGlyph/statusGlyphColor 는 pure 함수로 `statusGlyph.ts` 공용 모듈에 두고 AgentList 가 re-export.

## 거부한 대안

- **순수 모양-only 유지(ADR-0062 원안)** — 색 배제의 유일 근거(e-ink)가 별도 모드 분리로 소멸하므로, dark/light 에서까지 색을 포기할 이유가 없다.
- **하드코딩 hex** — 변수-only 불변식(ADR-0062 §44) 위반이고 e-ink 중립화가 불가능하다.

## 근거

모양이 여전히 1차 신호(색은 보조)라 e-ink 에서도 무손실. dark/light 에선 색이 활성 상태 인지 부하를 줄인다. statusGlyphColor 는 pure + 전분기 단위테스트(statusGlyph 규율 동형).

## 영향 / 불변식

- ADR-0062 §32(색으로 상태 표시 거부)·§44(색 리터럴 금지)는 이 ADR 로 **부분 개정** — 색은 허용하되 변수-only + e-ink 중립화로 §44 정신은 유지.
- e-ink 별도 모드가 실제로 분리되기 전까지 e-ink 테마는 var 중립화로 모양-only 를 보존한다.
