# ADR-0020: 클라이언트/백엔드 경로 통합 — 단일 프로토콜 + transport-중립 dispatch core

- 상태: 확정 (2026-06-17, 근거: `/consult` 3-way 교차검증 job `20260617-021239-consult-client-unification` + judge 판정)
- 관련: ADR-0011(agentClient 제어표면) · ADR-0002/0003(OutputSink seam) · ADR-0001(kill 2동사) · ADR-0005(finalize 1회) · ADR-0006(락 순서) · `crates/engram-dashboard-daemon/src/ws.rs` · `crates/engram-dashboard-protocol/messages.rs` · `src-tauri/src/commands/` · `src/api/{agentClient,embeddedClient,daemonClient}.ts` · Amended by ADR-0037 (결정3: 프로토콜 의미론 위치 — JS ProtocolClient → Rust(DaemonClient/protocol_state))
- 범위: 로컬(embedded)/원격(daemon) **이중 경로**를 단일 프로토콜 + 단일 dispatch core로 통합하고 transport만 교체. 이 ADR=결정+경계, 구현 단계는 TRD(`docs/process/S12-daemonization/client-unification-trd.md`).

## 맥락

프론트는 `AgentClient` 인터페이스 1개 + 구현 2개(`EmbeddedClient`=Tauri invoke/Channel, `DaemonClient`=WS+프로토콜). 백엔드도 같은 `AgentManager`를 감싸는 진입점 2개(`src-tauri/commands/*.rs` thin wrapper, `daemon/ws.rs` dispatch). 프로토콜(`AgentCommand`/`AgentEvent`)은 이미 AgentClient 전 메서드와 1:1 대응하도록 설계됐으나, **embedded가 이 프로토콜을 우회**(Tauri invoke 직접)하고 daemon만 사용한다. 결과: 새 기능마다 양쪽 2벌 배선(2x 유지비). 코어(AgentManager) 이하는 이미 1벌 공유.

핵심 관찰(consult B=claude 적출): `dispatch(cmd, conn_tx, manager, registry, multiview, ...)`의 10개 인자 중 순수 비즈니스는 `manager`뿐이고, dispatch 본체는 이미 `conn_tx`(WsOutbound mpsc)·`reply()` 추상으로만 응답해 **TcpStream/tungstenite/frame codec을 직접 만지지 않는다**. 즉 transport 절단선이 이미 거의 그어져 있다.

## 결정

**통합한다. carrier(전송)만 교체 가능하게 만들고, 프로토콜+dispatch는 단일 공유.**

1. **(결정1) message-level 추출.** handshake/auth/Origin/frame codec/keepalive는 WS 어댑터에 잔류, 그 뒤 `AgentCommand 입력 + Outbound(AgentEvent/binary) sink 출력`을 받는 **transport-중립 ConnectionCore**(dispatch + per-conn 세션 상태)를 추출. 미래 carrier(gRPC/모바일) 대비해 입출력을 trait(`OutboundSink`)으로 박는다(ADR §0 저위험+장기 → over-eng 허용).
2. **(결정2) embedded carrier = generic `agent_command(AgentCommand)` invoke 1개 + Channel(AgentEvent/출력 수신).** WebView2 IPC 경계상 직렬화는 어차피 발생 → invoke 20개를 1개로 합치는 것(직렬화 횟수 동일/감소). **출력 인코딩은 carrier별로**: InProc=base64 Channel(기존 PtyEvent 유지), WS=binary frame. `OutputSink`는 "frame을 sink로"까지만, 인코딩은 sink 구현 소유(ADR-0003 정합).
3. **(결정2 보강 — GPT 단독 적출, 필수)** **embedded 동시 invoke racing 직렬화.** Tauri command는 각 invoke가 독립 async task로 spawn돼 **병렬 도착 시 명령 순서가 비결정**이다(WS는 단일 read_task라 자연 직렬). embedded 어댑터도 **per-connection inbound mpsc + 단일 command loop**로 명령을 직렬화해 WS와 의미 동등성을 보장한다.
4. **(결정3) 프론트 `ProtocolClient`를 `Transport` trait 위로 일반화.** request_id 매칭·seq dedup·epoch 가드·재연결은 carrier 무관 프로토콜 의미론 → 한 곳에 모은다. `WsTransport`(재연결 있음) / `InProcTransport`(invoke+Channel, 항상 connected → 재연결/dedup no-op 자연수렴). 일관성 위해 invoke도 ack-only로 통일(request_id 매칭이 ProtocolClient 한 곳에 산다).
5. **lease/viewport/registry — 우회 금지.** embedded도 동일 ConnectionCore path를 타되 single-client에서 자연 무력화: conn_id 1개 → `check_input` 항상 Allow, viewport 1개 → smallest=자기크기. `if embedded { skip }` 분기 금지(미래 embedded+원격 동시 attach 시 인터리브). `ConnRegistry`(status fanout)는 WS 어댑터 내부 디테일로 잔류, status broadcast는 carrier가 구현하는 `StatusSink`로(이미 TauriStatusSink vs DaemonStatusSink).
6. **crate 위치.** ConnectionCore는 protocol+core 양쪽 의존 → **코어 crate에 못 둔다**(코어 tauri/protocol import 0 격리, ADR §4 유지). daemon crate(또는 신설 `connection-core` crate)에 둔다. Tauri 어댑터(src-tauri)가 그걸 호출.

## 거부한 대안

- **(결정1b) byte-stream generic** — `handle_connection`을 `S: AsyncRead+AsyncWrite`로 일반화, 로컬은 `tokio::io::duplex`로 WS 스택 통째 재사용. **거부:** 로컬에 WS handshake/frame masking/Origin/토큰비교/keepalive를 영구 부담 + 디버깅 콜스택이 tungstenite 경유로 길어짐 + headless 단위테스트가 duplex+handshake를 세워야 함. 로컬을 "가짜 WebSocket"으로 만드는 안티패턴. (WS 프로토콜 로컬 회귀검증 필요성은 이미 `ws_e2e.rs` in-proc 서버가 커버.)
- **(결정2b) duplex stream carrier** — 결정1b와 같은 이유로 거부.
- **(결정3b) 얇은 EmbeddedClient 새로 작성** — 통합 후 embedded도 AgentCommand 직렬화+Channel 디코드+(방어적) dedup이 필요해 더 이상 얇지 않고 DaemonClient 로직 부분집합을 재구현 → "2벌 배선"을 다른 형태로 부활. 리팩터 목적과 정면 충돌.
- **codegen으로 2벌 자동생성** — 증상(2벌 배선)만 자동화하고 단일 dispatch를 안 만듦. §5(단일 진입점) 미달성, 두 경로의 미묘한 동작 차이(R1~R3) 여지 잔존, 매크로 디버깅 비용. 10년/교체성 원칙 부적합.
- **always-loopback-daemon(embedded 폐기, 로컬도 WS loopback)** — 클라 1벌로 가장 단순하나 항상 TCP 포트 점유(방화벽/충돌 표면), 로컬 hot path가 WS framing+masking+loopback TCP를 영구 지불(결정1b 비용을 앱 전체로 확대), 부팅 핸드셰이크 지연. consult A(gemini)가 "진지 고려"로 격상했으나 자기 결정1a 근거와 모순. 거부(미래 재론 시 ttyd/zellij prior-art 참조).
- **request_id를 invoke return으로(Tauri 한정 하이브리드)** — embedded 단순화되나 request_id 매칭이 carrier별로 갈려 결정3(a) 단일 ProtocolClient를 깸(2벌 부활). 거부, 단 embedded 성능 문제 시 탈출구로 기록.

## 근거

- consult 3-way 블라인드(GPT·Gemini·Claude-opus) + judge: **셋 다 1a/2a/3a 합의.** judge 최신뢰=Claude(코드사실·ADR 불변식 정밀, 사실오류 0), 필수병합=GPT(결정3 racing 단독 적출), 참고만=Gemini(seq 유실 사실오류 — seq는 OutputCore 단일 pump 불변식이라 carrier가 못 바꿈; always-loopback 과대평가).
- 코드가 이미 절단선(dispatch가 carrier 무지)에 도달 → 1a는 최소변경·최대효과. 확신: 결정1·2 **확실**, 결정3 **가능성 높음**(DaemonClient 580줄 분해 난이도가 변수).

## 영향 / 불변식 (회귀 위험 — 리팩터 중 깨지면 안 됨)

- **R1 Ack→replay→ReplayComplete FIFO** (subscribers lock 보유 중 on_ready로 Ack를 replay binary보다 먼저, ADR-0006/C4). OutboundSink mock으로 [Ack,Binary,ReplayComplete] 순서 단언(carrier 양쪽).
- **R2 seq dedup/high-water** (`lastDeliveredSeq` 기준, replay_from으로 기준 안 건드림 = 기존 버그B 교훈). ProtocolClient 단위(transport mock).
- **R3 epoch 가드/Resume** (resubscribe epoch=null이면 FromOldest 중복 = 기존 버그A). **baseline 커버 확인됨**: `ws_e2e.rs` case06/case10 + 프론트 vitest 재연결 회귀.
- **R4 finalize 1회/terminal 분담** (terminal=pump 단독, 프론트는 agent-list-updated 판정, ADR-0005). headless StatusSink mock 1회.
- **R5 kill 2동사** (ADR-0001) — Kill arm은 manager 위임이라 core 미변경, 안전.
- **R6 close_signal**(WS 큐 포화 out-of-band) — **WS-특정, 어댑터 잔류**(추출 금지). OutboundSink는 SinkError만 반환, 어댑터가 carrier별 해석.
- **R7 코어 tauri import 0 격리**(ADR §4) — ConnectionCore는 daemon/신설 crate에. `rg "use tauri" crates/engram-dashboard-core/src/` → 0줄 유지.
- **테스트 순서:** phase4 green baseline 고정 → 순수 이동(기능변화 0, 매 단계 WS 재실행) → Tauri 어댑터 추가 → WS/embedded conformance(동등성) 테스트 → cdp.mjs 실 WebView2 실측. 단위테스트는 전 단계 갱신/누적(사용자 지시).
