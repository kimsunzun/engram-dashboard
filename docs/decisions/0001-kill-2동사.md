# ADR-0001: kill = 2동사 (shutdown + join_pump)

- 상태: 확정 (2026-06-11 spike 검증, S2/S3)
- 관련: CLAUDE.md 핵심 불변식 · `session.rs:82` · `manager.rs::kill_agent` · `transport/pty.rs`

## 맥락
PTY 프로세스와 출력 pump 스레드를 안전하게 종료해야 한다. 자원 폐쇄(master/child/job)와 pump 스레드 합류를 어떻게 묶을지 결정 필요.

## 결정
`session.kill` = `transport.shutdown()` **THEN** `core.join_pump(timeout)` — 두 동사를 분리한다.
- shutdown: shutdown flag Release → child.kill+wait → TerminateJobObject → master.take/drop. **반환 전 master drop 보장.**
- join_pump: 그 뒤 pump 종료를 timeout(5s) 대기.

## 거부한 대안
- **1동사(shutdown 내부에서 join)** — pump가 자기 종료를 기다리는 self-join → hang. master를 아직 안 놓은 채 join하면 영구 대기.
- **옛 kill 6단계** — 동치이나 transport/core 책임이 분리되지 않음(S10 재편 전 형태). 2동사가 소유권 분할과 맞물려 더 깨끗.

## 근거
shutdown이 master를 drop → reader가 EOF → pump break → `core.finish(reason)` → done_tx. 인과가 단방향이라 hang 없음. spike에서 master.take 후 reader EOF까지 **17~23ms** 실측.

## 영향 / 불변식
- **순서 역전 금지** — join_pump를 shutdown보다 먼저 하면 살아있는 pump를 기다려 hang.
- terminal 전이(`Killed`)는 pump가 finish로 단독 발행(→ ADR-0005). manager는 `agent-list-updated`만.
