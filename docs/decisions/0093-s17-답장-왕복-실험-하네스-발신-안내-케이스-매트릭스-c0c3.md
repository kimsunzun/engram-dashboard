# ADR-0093: S17 답장 왕복 실험 하네스 — 발신-안내 케이스 매트릭스 (C0~C3)

- 상태: 확정 (2026-07-20, 근거: 사용자 결정 — 조감도 승인 + 발신경로 비교 요구)
- 관련: ADR-0092(수신 계약·프라이밍) · ADR-0091(happy-path 사다리) · ADR-0086(듀얼 입구·토큰 파생 from) · ADR-0088(write-boundary 관측) · `crates/engram-dashboard-daemon/src/bin/roundtrip_smoke.rs` · `prompts/experiments/agent-priming-send-{both,mcp,cli}.md` · step-log S17

## 맥락
baseline(ADR-0092)에서 프라이밍된 에이전트가 A→B 자연 메시지를 인젝션 취급 없이 수용함이 성립했다. 다음 단계는 B→A **답장 왕복**과 "B가 발신법을 어떻게 아는가"의 실증이다. 발신은 듀얼 입구(MCP `send_message` / `engram-send` CLI, ADR-0086)로 이미 배선돼 있으나, **실 에이전트가 발신 절반을 스스로 호출한 적은 아직 없다**(baseline 스모크는 데몬이 가짜 발신자로 `handle_send`를 직접 부른 것). 현재 프라이밍(`prompts/agent-priming.md`)은 수신만 명시하고 발신은 "가능·세부 추후"로만 둔다. happy-path-first 사다리(ADR-0091) 위의 슬라이스다.

## 결정
1. **왕복 슬라이스 형태 = seed→B, B가 실 A에게 답장.** A·B 둘 다 실 primed 에이전트로 스폰, 데몬이 "A발" 자연 메시지를 B에 주입(baseline 경로 재사용), B가 자기 발신 경로로 A에게 답, A가 수신·처리. baseline을 딛고 미검증 두 조각(실 에이전트의 발신 절반 실사용 · A의 답장 자연수용)만 새로 검증 — 결정적.
2. **"B가 발신법을 아는 경로"는 단일 채택이 아니라 C0~C3 매트릭스로 실증 비교 후 결정.** C0=프라이밍 무(MCP 툴 자동발견만) · C1=수신+발신(MCP·CLI 둘 다 안내) · C2=발신 MCP만 · C3=발신 CLI만. 판정축 = 수용>토큰, `DeliveryObservation.entrance`로 B의 자연 선택(MCP vs CLI) 관측.
3. **유력 가설(미확정) = 프라이밍이 발신·수신 양방향을 일관되게 커버** — 단 툴 자동발견만으로 성립하는지도 병행 확인(그래서 C0 포함).
4. **후속 형식 스파이크에 "무양식/자연문 본문" 후보 추가.** 데몬이 봉투(`[message from <sender> id:<id>] <body>`, ingress.rs `wrap_message`)를 씌우므로 발신자는 raw 본문만 내도 된다 — 가설: 양식↓ → 에이전트 산출 오차↓.

## 거부한 대안
- **완전 자율(A가 먼저 발신 결정)** — A가 보낼지 말지가 프롬프트 운에 좌우돼 테스트가 비결정적. happy-path 첫 슬라이스엔 과함 → 후순위.
- **스텁 A(받은 것만 기록하는 껍데기)** — 제일 싸지만 "A가 답장을 자연수용"이라는 왕복의 핵심을 못 봐 무의미.
- **단일 발신경로만 테스트** — 듀얼 입구(ADR-0086) 중 하나만 봐 커버리지 손실. 둘 다 성립해야 함.
- **지금 프라이밍 본체(agent-priming.md)에 발신 절 확정 주입** — 문구가 형식 스파이크서 바뀔 예정이라 조기 고정 회피. 실험 변형은 `prompts/experiments/`에 격리 draft로 둔다.

## 근거
사용자 결정(조감도 승인 + "MCP·CLI 둘 다 케이스 비교해 최적 결정" 요구, 2026-07-20). baseline confound 교훈(ADR-0090→0092): 성공 모드 없이 실패 모드부터 파지 않는다 — 왕복 happy-path를 먼저 세운다. 실 에이전트 발신·A 수용은 실측으로만 확인 가능하므로 매트릭스 실행이 결정의 뒷받침(실행은 후속).

## 영향 / 불변식
- `roundtrip_smoke.rs`(test-harness bin, required-features 게이팅 — 프로덕션 빌드 제외)로 C0~C3 파라미터 실행. 최적 발신 구성·프라이밍 양방향화 여부의 **최종 결정은 매트릭스 실측 후로 유보**.
- 하네스 불변: baseline 이후 from=B·to=A 레코드만 답장으로 판정(pre-seed 오탐 방지) · A·B 둘 다 생존해야 valid negative(A 사망=setup 실패) · CLI-요구 프라이밍인데 engram-send 부재=SETUP-SKIP(정상 negative 아님) · 발신자 신원은 토큰 파생(ADR-0086, 본문 문자열 아님) · 관측은 write-boundary 싱크(ADR-0088).
- 프라이밍 실험 변형(`agent-priming-send-{both,mcp,cli}.md`)은 **프로덕션 아님** — 형식 스파이크서 문구 확정 시 정리.
