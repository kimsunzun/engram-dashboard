# ADR-0017: 세션/슬롯 구조 — 슬롯=한 모드의 한 세션(끝나면 슬롯도 끝), "죽음"=백엔드별 Run 종료+이유

- 상태: 확정 (2026-06-16, 사용자 결정 — consult (나) 권고를 도메인 직관 (A)로 덮음)
- 근거: `/consult` job `20260616-011502-consult-death-abstraction`(GPT·Gemini·Claude+judge, OSS 조사) · ADR-0016(에이전트 수명 모델) · ADR-0002/0004(백엔드 추상) · ADR-0001(kill 2동사)
- 대체: `docs/process/S12-daemonization/session-death-structure-options.md`의 (나) 권고 부분

## 맥락
"에이전트 죽음"을 백엔드 무관하게 정의해야 하는데, consult는 **Slot≠Run≠Transport로 슬롯을 persist시키는 (나) 모델**(tmux remain-on-exit/respawn-pane식)을 권고했다. 그러나 그 모델은 "슬롯 안에서 백엔드 갈아끼우기(claude→codex)·zombie(frozen) 슬롯·dead pane vs live terminal" 혼란을 유발한다. 사용자 도메인 직관은 더 단순한 **(A)**: 슬롯은 한 종류의 한 세션이고, 그 세션이 끝나면 슬롯도 끝난다.

## 결정
1. **슬롯 = 한 모드(터미널/claude/codex/api)의 한 세션.** 그 세션이 끝나면 **슬롯도 끝난다**(현행 유지). **슬롯 안에서 모드/백엔드를 갈아끼우지 않는다**(claude→codex 인플레이스 전환 없음). 다른 모드가 필요하면 **새 슬롯.**
2. **zombie/frozen 슬롯 없음** — 종료된 슬롯을 멈춘 채 남기지 않는다(tmux remain-on-exit식 (나) 거부). `cmd /k`로 셸 살리기도 거부(raw-mode 오염·EOF 안 옴·Job 충돌).
3. **복원 = 되살리기(새 세션), 백엔드 무관하게.** 세션 가진 에이전트(`needs_session=true`: claude/codex)는 sid로 `--resume`, 그 외는 fresh. 복원은 "그 종류 그대로" 다시 띄움. (사용자의 "터미널·codex도 복원" 요구 = 모든 종류에 복원이 동작 = 충족. 슬롯 내 백엔드 교체와는 무관.)
4. **터미널(셸)은 저장/복원 대상 아님.** `needs_session=false` → 이어받을 세션(sid)·대화가 없음. 되살려봤자 빈 새 셸이라 무의미 → **persist/auto_restore에서 제외.** 저장+복원되는 건 세션 가진 claude/codex뿐.
5. **"죽음"은 백엔드별 정의 + 이유 분류(consult에서 살린 부분):** Run 종료 + reason(Completed/Failed/Signaled/KilledByUser/Interrupted/StartupFailed/...). API는 stream-end=Completed(≠death). **KilledByUser/Interrupt는 자동재시작·(미래)respawn 비대상**(사람이 끈 건 안 살아남). kill(프로세스/Job 레벨)과 interrupt(Ctrl-C)는 분리 유지(ADR-0001).

## 거부한 대안
- **(나) Slot≠Run persist 컨테이너** (tmux remain-on-exit/respawn-pane, GNU screen zombie, Zellij command pane, VS Code relaunch): 같은 자리 respawn·모드 교체 유연성이 있으나 도메인 혼란(zombie 슬롯·in-place 백엔드 교체·dead pane을 live terminal로 오인) 유발. **consult 권고였으나 사용자 도메인 직관으로 기각** — 혼란 회피 > tmux식 유연성. (Gemini가 짚은 "Rust Slot 수명 격상" 구현법은 (나)를 택할 때만 의미 있어 함께 보류.)
- **cmd /k로 셸 살리기** — TUI raw-mode 오염·PTY EOF 안 옴(child watcher 필요)·JobObject가 셸까지 잡음. 거부.
- **터미널 persist/복원** — 이어받을 세션 상태가 없어 무의미.

## 근거
consult 교차검증(3종+judge)의 합의 중 **"백엔드별 죽음 정의 + exit-reason 분류 + KilledByUser 제외 + API stream≠death"**는 채택(백엔드 무관성에 필요). 그러나 핵심 구조 선택(슬롯 persist 컨테이너 (나) vs 슬롯=세션 (A))은 OSS 선례보다 **사용자 도메인 직관**을 우선 — claude가 곧 슬롯이고 종료=슬롯 종료가 사용자 멘탈모델에 맞고 혼란이 없다.

## 영향 / 불변식
- **슬롯 수명 = 세션 수명**(끝나면 같이 끝). 현행 동작 유지 → ADR-0001(kill 2동사)·ADR-0005(finalize 1회) 불변식 **무변경.**
- **터미널/셸 슬롯은 agents.json persist·auto_restore에서 제외**(`needs_session=false`는 복원 대상 아님). 저장·복원은 `needs_session=true`(claude/codex)에 한정.
- **슬롯 내 백엔드 교체 경로를 만들지 말 것** — 모드는 슬롯 생성 시 고정.
- ADR-0016과 정합: ADR-0016의 "sid 인스턴스"가 곧 이 ADR의 "한 세션", "저장=살림"은 **세션 가진 에이전트 한정**(터미널 제외)으로 구체화.
- "죽음" 관측은 여전히 PTY EOF(현행, ADR-0016 §런타임 죽음 관측 훅 TODO) — (나)를 안 가므로 child watcher 불필요.
