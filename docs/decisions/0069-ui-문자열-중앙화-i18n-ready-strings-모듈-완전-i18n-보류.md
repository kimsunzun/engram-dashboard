# ADR-0069: UI 문자열 중앙화 — i18n-ready strings 모듈 (완전 i18n 보류)

- 상태: 확정 (2026-07-12, 근거: 사용자 결정 — "일단 중앙화, 나중에 그 중앙 소스 기반 로컬화")
- 관련: CLAUDE.md §5(LLM 제어)·§0(저위험·장기 seam) · `src/i18n/`(신설) · step-log "UI 문자열 중앙화" 항목 · 백로그 "data-driven 우클릭 메뉴"

## 맥락
대시보드 UI 문자열(command 제목·우클릭 메뉴 라벨·컴포넌트 텍스트·aria-label·기본 탭명 `View 1`·빈상태 `- empty -` 등)이 코드 전반에 한국어로 **인라인 하드코딩**돼 있다. i18n/locale 인프라는 없다(package.json·`src/` 확인 — 라이브러리·`t()`·locale 파일 0). `src/commands/`만 해도 ~59곳. 사용자가 "메뉴 이름 같은 걸 한글로 하드코딩 중인데 로컬 테이블로 빼야 하지 않나"라고 지적.

두 목표를 구분해야 규모가 갈린다: **중앙화**(단일 소스 — drift 방지 + §5에서 LLM이 라벨을 한 곳에서 읽기/수정 + 튜토리얼 문자열 소스, 저비용) vs **완전 i18n**(react-i18next + locale + `t()` 런타임 다국어, 무거움). 현재 내부 한국어 전용 툴이라 다국어 계획은 없다. (게임 UE5의 Perforce 로컬라이징 데이터 테이블과는 별개 — 대시보드는 독립 JS 툴.)

## 결정
사용자 결정 = **일단 중앙화, 나중에 그 중앙 소스를 로컬화 베이스로**. 이에 따라:
- `src/i18n/` 모듈 신설 — `ko.ts`(네임스페이스드 `key → 한국어 문자열` 테이블) + `index.ts`의 `t(key, params?)` accessor.
- **key는 `ko` 테이블에서 파생한 union 타입**(타입세이프) — 오타·누락 키를 tsc가 컴파일 타임에 잡는다.
- `t(key, params?)`는 **보간 지원**(예: `t('tab.close', {name})` → `'탭 닫기: {name}'`의 `{name}` 치환). 지금은 `ko`만 반환(단일 언어).
- 사용자 노출 UI 문자열을 이 `t()`로 **점진 마이그레이션**(command → 컴포넌트). 새 문자열은 인라인 금지, `t()` 경유.
- 나중 로컬화 = `t()` 백엔드를 locale-aware로 교체(호출부·key 불변). 위치를 `src/strings/`가 아니라 `src/i18n/`로 둔 것이 그 미래 신호.

## 거부한 대안
- **완전 i18n 지금(react-i18next + locale + `t()` 런타임)** — 다국어 계획 없는 내부 툴엔 YAGNI·런타임/계층 과중. **폐기 아니라 보류**: 이 중앙 소스 위에 나중에 얹는다(호출부 불변).
- **인라인 문자열 유지** — drift(aria-label ↔ 표시 텍스트 불일치)·§5 단일 소스 부재·튜토리얼 문자열이 코드 전반에 흩어짐.
- **무타입 flat map(`Record<string,string>`)** — 마이그레이션 중 오타·존재하지 않는 키를 tsc가 못 잡아 런타임에 깨진 라벨(빈/undefined)로 샌다. 타입세이프 key가 대량 치환의 안전망.
- **`src/strings/` 네이밍** — 로컬화 의도를 안 드러냄. `src/i18n/`가 "나중에 로컬화" 신호를 코드 지도에 박는다.

## 근거
- **CLAUDE.md §0(저위험 + 장기 → seam 미리 깐다):** 문자열 접근 seam은 저위험이고 나중에 바꾸면 비싸다(전 호출부 재작업) → 지금 깐다(over-engineering 허용 구간).
- **§5(LLM 제어):** 중앙 테이블 = LLM이 모든 UI 라벨을 한 곳에서 읽고(정보화) 바꾸는(제어) 미래 표면 — 백로그 "data-driven 우클릭 메뉴"와 정합.
- **로드맵 "튜토리얼 엔그람":** 문자열 다수 예정 → 중앙 소스가 이득.
- **타입세이프 key:** 대량 마이그레이션의 컴파일 안전망(누락·오타 즉시 발견).

## 영향 / 불변식
- **key = 안정 API** — 키 변경은 전 호출부에 영향(신중히). 문자열 값 변경은 자유.
- **마이그레이션 범위 = 사용자 노출 UI 문자열**(command 제목·메뉴 라벨·컴포넌트 텍스트·aria-label·기본명·빈상태). 내부 `console.warn`/에러 로그는 **범위 밖**(UI 아님 — 별도 판단).
- **나중 locale 추가 = `t()` 백엔드 교체만** — 호출부·key 불변(그래서 지금 형태가 로컬화 베이스).
- **§5 LLM 노출**(테이블 read/override command)은 **후속 슬라이스**(MVP는 중앙화까지).
- **새 UI 문자열 인라인 금지** — `t()` 경유(리뷰 체크 항목).
